140  <p>Pygame currently supports Linux (V4L2) and Windows (MSMF) cameras natively,
141  with wider platform support available via an integrated OpenCV backend.</p>
142  <div class="versionadded">
143  <p><span class="versionmodified added">New in pygame 2.0.2: </span>Windows native camera support</p>
144  </div>

147  </div>

176  <dt class="sig sig-object py title" id="pygame.camera.init">
177  <span class="sig-prename descclassname"><span class="pre">pygame.camera.</span></span><span class="sig-name descname"><span class="pre">init</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.camera.init" title="Permalink to this definition">¶</a></dt>
178  <dd><div class="line-block">
179  <div class="line"><span class="summaryline">Module init</span></div>
180  <div class="line"><span class="signature">init(backend = None) -&gt; None</span></div>
181  </div>
182  <p>This function starts up the camera module, choosing the best webcam backend
183  it can find for your system. This is not guaranteed to succeed, and may even
184  attempt to import third party modules, like <cite>OpenCV</cite>. If you want to
185  override its backend choice, you can call pass the name of the backend you
186  want into this function. More about backends in
187  <a class="reference internal" href="#pygame.camera.get_backends" title="pygame.camera.get_backends"><code class="xref py py-func docutils literal notranslate"><span class="pre">get_backends()</span></code></a>.</p>
188  <div class="versionchanged">
189  <p><span class="versionmodified changed">Changed in pygame 2.0.3: </span>Option to explicitly select backend</p>
190  </div>
191  </dd></dl>

193  <dl class="py function definition">
194  <dt class="sig sig-object py title" id="pygame.camera.get_backends">
195  <span class="sig-prename descclassname"><span class="pre">pygame.camera.</span></span><span class="sig-name descname"><span class="pre">get_backends</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.camera.get_backends" title="Permalink to this definition">¶</a></dt>
196  <dd><div class="line-block">
197  <div class="line"><span class="summaryline">Get the backends supported on this system</span></div>
198  <div class="line"><span class="signature">get_backends() -&gt; [str]</span></div>
199  </div>
200  <p>This function returns every backend it thinks has a possibility of working
201  on your system, in order of priority.</p>
202  <p>pygame.camera Backends:</p>
203  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span>Backend           OS        Description
204  ---------------------------------------------------------------------------------
205  _camera (MSMF)    Windows   Builtin, works on Windows 8+ Python3
206  _camera (V4L2)    Linux     Builtin
207  OpenCV            Any       Uses `opencv-python` module, can&#39;t enumerate cameras
208  OpenCV-Mac        Mac       Same as OpenCV, but has camera enumeration
209  VideoCapture      Windows   Uses abandoned `VideoCapture` module, can&#39;t enumerate
210                              cameras, may be removed in the future
211  </pre></div>
212  </div>
213  <p>There are two main differences among backends.</p>
214  <p>The _camera backends are built in to pygame itself, and require no third
215  party imports. All the other backends do. For the OpenCV and VideoCapture
216  backends, those modules need to be installed on your system.</p>
217  <p>The other big difference is &quot;camera enumeration.&quot; Some backends don't have
218  a way to list out camera names, or even the number of cameras on the
219  system. In these cases, <a class="reference internal" href="#pygame.camera.list_cameras" title="pygame.camera.list_cameras"><code class="xref py py-func docutils literal notranslate"><span class="pre">list_cameras()</span></code></a> will return
220  something like <code class="docutils literal notranslate"><span class="pre">[0]</span></code>. If you know you have multiple cameras on the
221  system, these backend ports will pass through a &quot;camera index number&quot;
222  through if you use that as the <code class="docutils literal notranslate"><span class="pre">device</span></code> parameter.</p>
223  <div class="versionadded">
224  <p><span class="versionmodified added">New in pygame 2.0.3.</span></p>
225  </div>
226  </dd></dl>

229  <dt class="sig sig-object py title" id="pygame.camera.colorspace">
230  <span class="sig-prename descclassname"><span class="pre">pygame.camera.</span></span><span class="sig-name descname"><span class="pre">colorspace</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.camera.colorspace" title="Permalink to this definition">¶</a></dt>
231  <dd><div class="line-block">
232  <div class="line"><span class="summaryline">Surface colorspace conversion</span></div>
233  <div class="line"><span class="signature">colorspace(Surface, format, DestSurface = None) -&gt; Surface</span></div>
234  </div>
235  <p>Allows for conversion from &quot;RGB&quot; to a destination colorspace of &quot;HSV&quot; or
236  &quot;YUV&quot;. The source and destination surfaces must be the same size and pixel
237  depth. This is useful for computer vision on devices with limited processing
238  power. Capture as small of an image as possible, <code class="docutils literal notranslate"><span class="pre">transform.scale()</span></code> it
239  even smaller, and then convert the colorspace to <code class="docutils literal notranslate"><span class="pre">YUV</span></code> or <code class="docutils literal notranslate"><span class="pre">HSV</span></code> before
240  doing any processing on it.</p>
241  </dd></dl>

313  <p>Loads a camera. On Linux, the device is typically something like
314  &quot;/dev/video0&quot;. Default width and height are 640 by 480.
315  Format is the desired colorspace of the output.
316  This is useful for computer vision purposes. The default is
317  <code class="docutils literal notranslate"><span class="pre">RGB</span></code>. The following are supported:</p>
318  <blockquote>
319  <div><ul class="simple">
320  <li><p><code class="docutils literal notranslate"><span class="pre">RGB</span></code> - Red, Green, Blue</p></li>
321  <li><p><code class="docutils literal notranslate"><span class="pre">YUV</span></code> - Luma, Blue Chrominance, Red Chrominance</p></li>
322  <li><p><code class="docutils literal notranslate"><span class="pre">HSV</span></code> - Hue, Saturation, Value</p></li>
323  </ul>
324  </div></blockquote>

349  <dt class="sig sig-object py title" id="pygame.camera.Camera.get_controls">
350  <span class="sig-name descname"><span class="pre">get_controls</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.camera.Camera.get_controls" title="Permalink to this definition">¶</a></dt>
351  <dd><div class="line-block">
352  <div class="line"><span class="summaryline">gets current values of user controls</span></div>
353  <div class="line"><span class="signature">get_controls() -&gt; (hflip = bool, vflip = bool, brightness)</span></div>
354  </div>
355  <p>If the camera supports it, get_controls will return the current settings
356  for horizontal and vertical image flip as bools and brightness as an int.
357  If unsupported, it will return the default values of (0, 0, 0). Note that
358  the return values here may be different than those returned by
359  set_controls, though these are more likely to be correct.</p>
360  </dd></dl>

362  <dl class="py method definition">
363  <dt class="sig sig-object py title" id="pygame.camera.Camera.set_controls">
364  <span class="sig-name descname"><span class="pre">set_controls</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.camera.Camera.set_controls" title="Permalink to this definition">¶</a></dt>
365  <dd><div class="line-block">
366  <div class="line"><span class="summaryline">changes camera settings if supported by the camera</span></div>
367  <div class="line"><span class="signature">set_controls(hflip = bool, vflip = bool, brightness) -&gt; (hflip = bool, vflip = bool, brightness)</span></div>
368  </div>
369  <p>Allows you to change camera settings if the camera supports it. The
370  return values will be the input values if the camera claims it succeeded
371  or the values previously in use if not. Each argument is optional, and
372  the desired one can be chosen by supplying the keyword, like hflip. Note
373  that the actual settings being used by the camera may not be the same as
374  those returned by set_controls. On Windows, <code class="code docutils literal notranslate"><span class="pre">hflip</span></code> and <code class="code docutils literal notranslate"><span class="pre">vflip</span></code> are
375  implemented by pygame, not by the Camera, so they should always work, but
376  <code class="code docutils literal notranslate"><span class="pre">brightness</span></code> is unsupported.</p>
377  </dd></dl>

395  <dd><div class="line-block">
396  <div class="line"><span class="summaryline">checks if a frame is ready</span></div>
397  <div class="line"><span class="signature">query_image() -&gt; bool</span></div>
398  </div>
399  <p>If an image is ready to get, it returns true. Otherwise it returns false.
400  Note that some webcams will always return False and will only queue a
401  frame when called with a blocking function like <a class="reference internal" href="#pygame.camera.Camera.get_image" title="pygame.camera.Camera.get_image"><code class="xref py py-func docutils literal notranslate"><span class="pre">get_image()</span></code></a>.
402  On Windows (MSMF), and the  OpenCV backends, <a class="reference internal" href="#pygame.camera.Camera.query_image" title="pygame.camera.Camera.query_image"><code class="xref py py-func docutils literal notranslate"><span class="pre">query_image()</span></code></a>
403  should be reliable, though. This is useful to separate the framerate of
404  the game from that of the camera without having to use threading.</p>
405  </dd></dl>

411  <div class="line"><span class="summaryline">captures an image as a Surface</span></div>
412  <div class="line"><span class="signature">get_image(Surface = None) -&gt; Surface</span></div>
413  </div>
414  <p>Pulls an image off of the buffer as an <code class="docutils literal notranslate"><span class="pre">RGB</span></code> Surface. It can optionally
