6    .. module:: pygame.mixer.music
7       :synopsis: pygame module for controlling streamed audio
8    
9    | :sl:`pygame module for controlling streamed audio`
10   
11   The music module is closely tied to :mod:`pygame.mixer`. Use the music module
12   to control the playback of music in the sound mixer.
13   
14   The difference between the music playback and regular Sound playback is that
15   the music is streamed, and never actually loaded all at once. The mixer system
16   only supports a single music stream at once.
17   
18   On older pygame versions, ``MP3`` support was limited under Mac and Linux. This
19   changed in pygame ``v2.0.2`` which got improved MP3 support. Consider using
20   ``OGG`` file format for music as that can give slightly better compression than
21   MP3 in most cases.
22   
23   .. function:: load
24   
25      | :sl:`Load a music file for playback`
26      | :sg:`load(filename) -> None`
27      | :sg:`load(fileobj, namehint="") -> None`
28   
29      This will load a music filename/file object and prepare it for playback. If
30      a music stream is already playing it will be stopped. This does not start
31      the music playing.
32   
33      If you are loading from a file object, the namehint parameter can be used to specify
34      the type of music data in the object. For example: :code:`load(fileobj, "ogg")`.
35

52   .. function:: play
53   
54      | :sl:`Start the playback of the music stream`
55      | :sg:`play(loops=0, start=0.0, fade_ms=0) -> None`
56   
57      This will play the loaded music stream. If the music is already playing it
58      will be restarted.
59      
60      ``loops`` is an optional integer argument, which is ``0`` by default, which 
61      indicates how many times to repeat the music. The music repeats indefinitely if 
62      this argument is set to ``-1``. 
63      
64      ``start`` is an optional float argument, which is ``0.0`` by default, which 
65      denotes the position in time from which the music starts playing. The starting 
66      position depends on the format of the music played. ``MP3`` and ``OGG`` use 
67      the position as time in seconds. For ``MP3`` files the start time position
68      selected may not be accurate as things like variable bit rate encoding and ID3
69      tags can throw off the timing calculations. For ``MOD``  music it is the pattern 
70      order number. Passing a start position will raise a NotImplementedError if 
71      the start position cannot be set.
72   
73      ``fade_ms`` is an optional integer argument, which is ``0`` by default,
74      which denotes the period of time (in milliseconds) over which the music
75      will fade up from volume level ``0.0`` to full volume (or the volume level
76      previously set by :func:`set_volume`). The sample may end before the fade-in
77      is complete. If the music is already streaming ``fade_ms`` is ignored.
78      
79      .. versionchanged:: 2.0.0 Added optional ``fade_ms`` argument

83   .. function:: rewind
84   
85      | :sl:`restart music`
86      | :sg:`rewind() -> None`
87   
88      Resets playback of the current music to the beginning. If :func:`pause` has
89      previoulsy been used to pause the music, the music will remain paused.
90      
91      .. note:: :func:`rewind` supports a limited number of file types and notably
92                ``WAV`` files are NOT supported. For unsupported file types use :func:`play`
93                which will restart the music that's already playing (note that this
94                will start the music playing again even if previously paused).

128  .. function:: fadeout
129  
130     | :sl:`stop music playback after fading out`
131     | :sg:`fadeout(time) -> None`
132  
133     Fade out and stop the currently playing music.
134  
135     The ``time`` argument denotes the integer milliseconds for which the 
136     fading effect is generated.
137  
138     Note, that this function blocks until the music has faded out. Calls 
139     to :func:`fadeout` and :func:`set_volume` will have no effect during 
140     this time. If an event was set using :func:`set_endevent` it will be 
141     called after the music has faded.

145  .. function:: set_volume
146  
147     | :sl:`set the music volume`
148     | :sg:`set_volume(volume) -> None`
149  
150     Set the volume of the music playback.
151     
152     The ``volume`` argument is a float between ``0.0`` and ``1.0`` that sets 
153     the volume level. When new music is loaded the volume is reset to full
154     volume. If ``volume`` is a negative value it will be ignored and the
155     volume will remain set at the current level. If the ``volume`` argument
156     is greater than ``1.0``, the volume will be set to ``1.0``.

184  .. function:: set_pos
185  
186     | :sl:`set position to play from`
187     | :sg:`set_pos(pos) -> None`
188  
189     This sets the position in the music file where playback will start.
190     The meaning of "pos", a float (or a number that can be converted to a float),
191     depends on the music format.
192     
193     For ``MOD`` files, pos is the integer pattern number in the module.
194     For ``OGG`` it is the absolute position, in seconds, from
195     the beginning of the sound. For ``MP3`` files, it is the relative position,
196     in seconds, from the current position. For absolute positioning in an ``MP3``
197     file, first call :func:`rewind`.
198  
199     Other file formats are unsupported. Newer versions of SDL_mixer have
200     better positioning support than earlier ones. An SDLError is raised if a
201     particular format does not support positioning.
202  
203     Function :func:`set_pos` calls underlining SDL_mixer function
204     ``Mix_SetMusicPosition``.
205  
206     .. versionadded:: 1.9.2
207

221  .. function:: queue
222  
223     | :sl:`queue a sound file to follow the current`
224     | :sg:`queue(filename) -> None`
225     | :sg:`queue(fileobj, namehint="", loops=0) -> None`
226  
227     This will load a sound file and queue it. A queued sound file will begin as
228     soon as the current sound naturally ends. Only one sound can be queued at a
229     time. Queuing a new sound while another sound is queued will result in the
230     new sound becoming the queued sound. Also, if the current sound is ever
231     stopped or changed, the queued sound will be lost.
232  
233     If you are loading from a file object, the namehint parameter can be used to specify
234     the type of music data in the object. For example: :code:`queue(fileobj, "ogg")`.
235  
236     The following example will play music by Bach six times, then play music by
237     Mozart once:
238  
239     ::
240  
241         pygame.mixer.music.load('bach.ogg')
242         pygame.mixer.music.play(5)        # Plays six times, not five!
243         pygame.mixer.music.queue('mozart.ogg')
244  
245     .. versionchanged:: 2.0.2 Added optional ``namehint`` argument
246

249  .. function:: set_endevent
250  
251     | :sl:`have the music send an event when playback stops`
252     | :sg:`set_endevent() -> None`
253     | :sg:`set_endevent(type) -> None`
254  
255     This causes pygame to signal (by means of the event queue) when the music is
256     done playing. The argument determines the type of event that will be queued.
257  
258     The event will be queued every time the music finishes, not just the first
259     time. To stop the event from being queued, call this method with no
260     argument.
261

267     | :sg:`get_endevent() -> type`

269     Returns the event type to be sent every time the music finishes playback. If
270     there is no endevent the function returns ``pygame.NOEVENT``.
