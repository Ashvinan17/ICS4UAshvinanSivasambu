112  <section id="introduction">
113  <h3>Introduction<a class="headerlink" href="#introduction" title="Permalink to this headline">¶</a></h3>
114  <p>In the <em>pygame</em> examples there is a simple example named &quot;chimp&quot;.
115  This example simulates a punchable monkey moving around the screen with
116  promises of riches and reward. The example itself is very simple, and a
117  bit thin on error-checking code. This example program demonstrates many of
118  pygame's abilities, like creating a window, loading images and sounds,
119  rendering text, and basic event and mouse handling.</p>
120  <p>The program and images can be found inside the standard source distribution
121  of pygame. You can run it by running <cite>python -m pygame.examples.chimp</cite> in
122  your terminal.</p>
123  <p>This tutorial will go through the code block by block. Explaining how
124  the code works. There will also be mention of how the code could be improved
125  and what error checking could help out.</p>
126  <p>This is an excellent tutorial for people getting their first look at
127  the <em>pygame</em> code. Once <em>pygame</em> is fully installed, you can find
128  and run the chimp demo for yourself in the examples directory.</p>

135  <section id="import-modules">
136  <h3>Import Modules<a class="headerlink" href="#import-modules" title="Permalink to this headline">¶</a></h3>
137  <p>This is the code that imports all the needed modules into your program.
138  It also checks for the availability of some of the optional pygame modules.</p>
139  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="c1"># Import Modules</span>
140  <span class="kn">import</span> <span class="nn">os</span>
141  <span class="kn">import</span> <span class="nn">pygame</span> <span class="k">as</span> <span class="nn">pg</span>
142  
143  <span class="k">if</span> <span class="ow">not</span> <span class="n">pg</span><span class="o">.</span><span class="n">font</span><span class="p">:</span>
144      <span class="nb">print</span><span class="p">(</span><span class="s2">&quot;Warning, fonts disabled&quot;</span><span class="p">)</span>
145  <span class="k">if</span> <span class="ow">not</span> <span class="n">pg</span><span class="o">.</span><span class="n">mixer</span><span class="p">:</span>
146      <span class="nb">print</span><span class="p">(</span><span class="s2">&quot;Warning, sound disabled&quot;</span><span class="p">)</span>
147  
148  <span class="n">main_dir</span> <span class="o">=</span> <span class="n">os</span><span class="o">.</span><span class="n">path</span><span class="o">.</span><span class="n">split</span><span class="p">(</span><span class="n">os</span><span class="o">.</span><span class="n">path</span><span class="o">.</span><span class="n">abspath</span><span class="p">(</span><span class="vm">__file__</span><span class="p">))[</span><span class="mi">0</span><span class="p">]</span>
149  <span class="n">data_dir</span> <span class="o">=</span> <span class="n">os</span><span class="o">.</span><span class="n">path</span><span class="o">.</span><span class="n">join</span><span class="p">(</span><span class="n">main_dir</span><span class="p">,</span> <span class="s2">&quot;data&quot;</span><span class="p">)</span>
150  </pre></div>
151  </div>
152  <p>First, we import the standard &quot;os&quot; python module. This allow
153  us to do things like create platform independent file paths.</p>
154  <p>In the next line, we import the pygame package. In our case, we import
155  pygame as <code class="docutils literal notranslate"><span class="pre">pg</span></code>, so that all of the functionality of pygame is able to
156  be referenced from the namespace <code class="docutils literal notranslate"><span class="pre">pg</span></code>.</p>
157  <p>Some pygame modules are optional, and if they aren't found,
158  they evaluate to <code class="docutils literal notranslate"><span class="pre">False</span></code>. Because of that, we decide to print
159  a nice warning message if the <a class="reference internal" href="../ref/font.html#module-pygame.font" title="pygame.font: pygame module for loading and rendering fonts"><code class="xref py py-mod docutils literal notranslate"><span class="pre">font</span></code></a> or
160  <a class="reference internal" href="../ref/mixer.html#module-pygame.mixer" title="pygame.mixer: pygame module for loading and playing sounds"><code class="xref py py-mod docutils literal notranslate"><span class="pre">mixer</span></code></a> modules in pygame are not available.
161  (Although they will only be unavailable in very uncommon situations).</p>
162  <p>Lastly, we prepare two paths for the rest of the code to use.
163  <code class="docutils literal notranslate"><span class="pre">main_dir</span></code> uses the <cite>os.path</cite> module and the <cite>__file__</cite> variable provided
164  by Python to locate the game's python file, and extract the folder from
165  that path. It then prepares the variable <code class="docutils literal notranslate"><span class="pre">data_dir</span></code> to tell the
166  loading functions exactly where to look.</p>

169  <h3>Loading Resources<a class="headerlink" href="#loading-resources" title="Permalink to this headline">¶</a></h3>
170  <p>Here we have two functions we can use to load images and sounds. We will
171  look at each function individually in this section.</p>
172  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">def</span> <span class="nf">load_image</span><span class="p">(</span><span class="n">name</span><span class="p">,</span> <span class="n">colorkey</span><span class="o">=</span><span class="kc">None</span><span class="p">,</span> <span class="n">scale</span><span class="o">=</span><span class="mi">1</span><span class="p">):</span>
173      <span class="n">fullname</span> <span class="o">=</span> <span class="n">os</span><span class="o">.</span><span class="n">path</span><span class="o">.</span><span class="n">join</span><span class="p">(</span><span class="n">data_dir</span><span class="p">,</span> <span class="n">name</span><span class="p">)</span>
174      <span class="n">image</span> <span class="o">=</span> <span class="n">pg</span><span class="o">.</span><span class="n">image</span><span class="o">.</span><span class="n">load</span><span class="p">(</span><span class="n">fullname</span><span class="p">)</span>
175  
176      <span class="n">size</span> <span class="o">=</span> <span class="n">image</span><span class="o">.</span><span class="n">get_size</span><span class="p">()</span>
177      <span class="n">size</span> <span class="o">=</span> <span class="p">(</span><span class="n">size</span><span class="p">[</span><span class="mi">0</span><span class="p">]</span> <span class="o">*</span> <span class="n">scale</span><span class="p">,</span> <span class="n">size</span><span class="p">[</span><span class="mi">1</span><span class="p">]</span> <span class="o">*</span> <span class="n">scale</span><span class="p">)</span>
178      <span class="n">image</span> <span class="o">=</span> <span class="n">pg</span><span class="o">.</span><span class="n">transform</span><span class="o">.</span><span class="n">scale</span><span class="p">(</span><span class="n">image</span><span class="p">,</span> <span class="n">size</span><span class="p">)</span>
179  
180      <span class="n">image</span> <span class="o">=</span> <span class="n">image</span><span class="o">.</span><span class="n">convert</span><span class="p">()</span>
181      <span class="k">if</span> <span class="n">colorkey</span> <span class="ow">is</span> <span class="ow">not</span> <span class="kc">None</span><span class="p">:</span>
182          <span class="k">if</span> <span class="n">colorkey</span> <span class="o">==</span> <span class="o">-</span><span class="mi">1</span><span class="p">:</span>
183              <span class="n">colorkey</span> <span class="o">=</span> <span class="n">image</span><span class="o">.</span><span class="n">get_at</span><span class="p">((</span><span class="mi">0</span><span class="p">,</span> <span class="mi">0</span><span class="p">))</span>
184          <span class="n">image</span><span class="o">.</span><span class="n">set_colorkey</span><span class="p">(</span><span class="n">colorkey</span><span class="p">,</span> <span class="n">pg</span><span class="o">.</span><span class="n">RLEACCEL</span><span class="p">)</span>
185      <span class="k">return</span> <span class="n">image</span><span class="p">,</span> <span class="n">image</span><span class="o">.</span><span class="n">get_rect</span><span class="p">()</span>
186  </pre></div>
187  </div>
188  <p>This function takes the name of an image to load. It also optionally
189  takes an argument it can use to set a colorkey for the image, and an argument
190  to scale the image. A colorkey is used in graphics to represent a color of the
191  image that is transparent.</p>
192  <p>The first thing this function does is create a full pathname to the file.
193  In this example all the resources are in a &quot;data&quot; subdirectory. By using
194  the <cite>os.path.join</cite> function, a pathname will be created that works for whatever
195  platform the game is running on.</p>
196  <p>Next we load the image using the <a class="tooltip reference internal" href="../ref/image.html#pygame.image.load" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.image.load()</span></code><span class="tooltip-content">load new image from a file (or file-like object)</span></a> function.
197  After the image is loaded, we make an important
198  call to the <cite>convert()</cite> function. This makes a new copy of a Surface and converts
199  its color format and depth to match the display. This means blitting the
200  image to the screen will happen as quickly as possible.</p>
201  <p>We then scale the image, using the <a class="tooltip reference internal" href="../ref/transform.html#pygame.transform.scale" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.transform.scale()</span></code><span class="tooltip-content">resize to new resolution</span></a> function.
202  This function takes a Surface and the size it should be scaled to. To scale
203  by a scalar, we can get the size and scale the x and y by the scalar.</p>
204  <p>Last, we set the colorkey for the image. If the user supplied an argument
205  for the colorkey argument we use that value as the colorkey for the image.
206  This would usually just be a color RGB value, like (255, 255, 255) for
207  white.  You can also pass a value of -1 as the colorkey. In this case the
208  function will lookup the color at the topleft pixel of the image, and use
209  that color for the colorkey.</p>
210  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">def</span> <span class="nf">load_sound</span><span class="p">(</span><span class="n">name</span><span class="p">):</span>
211      <span class="k">class</span> <span class="nc">NoneSound</span><span class="p">:</span>
212          <span class="k">def</span> <span class="nf">play</span><span class="p">(</span><span class="bp">self</span><span class="p">):</span>
213              <span class="k">pass</span>
214  
215      <span class="k">if</span> <span class="ow">not</span> <span class="n">pg</span><span class="o">.</span><span class="n">mixer</span> <span class="ow">or</span> <span class="ow">not</span> <span class="n">pg</span><span class="o">.</span><span class="n">mixer</span><span class="o">.</span><span class="n">get_init</span><span class="p">():</span>
216          <span class="k">return</span> <span class="n">NoneSound</span><span class="p">()</span>
217  
218      <span class="n">fullname</span> <span class="o">=</span> <span class="n">os</span><span class="o">.</span><span class="n">path</span><span class="o">.</span><span class="n">join</span><span class="p">(</span><span class="n">data_dir</span><span class="p">,</span> <span class="n">name</span><span class="p">)</span>
219      <span class="n">sound</span> <span class="o">=</span> <span class="n">pg</span><span class="o">.</span><span class="n">mixer</span><span class="o">.</span><span class="n">Sound</span><span class="p">(</span><span class="n">fullname</span><span class="p">)</span>
220  
221      <span class="k">return</span> <span class="n">sound</span>
222  </pre></div>
223  </div>
224  <p>Next is the function to load a sound file. The first thing this function
225  does is check to see if the <a class="tooltip reference internal" href="../ref/mixer.html#module-pygame.mixer" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.mixer</span></code><span class="tooltip-content">pygame module for loading and playing sounds</span></a> module was imported correctly.
226  If not, it returns a small class instance that has a dummy play method.
227  This will act enough like a normal Sound object for this game to run without
228  any extra error checking.</p>
229  <p>This function is similar to the image loading function, but handles some
230  different problems. First we create a full path to the sound image, and
231  load the sound file. Then we simply return the loaded Sound object.</p>
232  </section>

336  This class is also derived from the base <a class="reference internal" href="../ref/sprite.html#pygame.sprite.Sprite" title="pygame.sprite.Sprite"><code class="xref py py-class docutils literal notranslate"><span class="pre">Sprite</span></code></a>

338  also sets the attribute &quot;area&quot; to be the size of the display screen.</p>
339  <p>The <cite>update</cite> function for the chimp simply looks at the current &quot;dizzy&quot;
340  state, which is true when the monkey is spinning from a punch. It calls either
341  the <cite>_spin</cite> or <cite>_walk</cite> method. These functions are prefixed with an underscore.
342  This is just a standard python idiom which suggests these methods should
343  only be used by the <cite>Chimp</cite> class. We could go so far as to give them a double
344  underscore, which would tell python to really try to make them private
345  methods, but we don't need such protection. :)</p>

347  rect by a given offset. If this new position crosses outside the display
348  area of the screen, it reverses the movement offset. It also mirrors the
349  image using the <a class="tooltip reference internal" href="../ref/transform.html#pygame.transform.flip" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.transform.flip()</span></code><span class="tooltip-content">flip vertically and horizontally</span></a> function. 352  attribute is used to store the current amount of rotation. When the monkey
353  has rotated all the way around (360 degrees) it resets the monkey image
354  back to the original, non-rotated version. Before calling the
355  <a class="tooltip reference internal" href="../ref/transform.html#pygame.transform.rotate" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.transform.rotate()</span></code><span class="tooltip-content">rotate an image</span></a> function, you'll see the code makes a local

357  for this example, it is just done here to keep the following line's length a
358  little shorter. Note that when calling the <cite>rotate</cite> function, we are always
359  rotating from the original monkey image. When rotating, there is a slight loss
360  of quality. Repeatedly rotating the same image and the quality would get worse
361  each time. Also, when rotating an image, the size of the image will actually
362  change. This is because the corners of the image will be rotated out, making
363  the image bigger. We make sure the center of the new image matches the center
364  of the old image, so it rotates without moving.</p>

366  state.  This will cause the image to start spinning. It also makes a copy
367  of the current image named &quot;original&quot;.</p>

460  <p>Here we create all the objects that the game is going to need.</p>
461  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">whiff_sound</span> <span class="o">=</span> <span class="n">load_sound</span><span class="p">(</span><span class="s2">&quot;whiff.wav&quot;</span><span class="p">)</span>
462  <span class="n">punch_sound</span> <span class="o">=</span> <span class="n">load_sound</span><span class="p">(</span><span class="s2">&quot;punch.wav&quot;</span><span class="p">)</span>
463  <span class="n">chimp</span> <span class="o">=</span> <span class="n">Chimp</span><span class="p">()</span>
464  <span class="n">fist</span> <span class="o">=</span> <span class="n">Fist</span><span class="p">()</span>
465  <span class="n">allsprites</span> <span class="o">=</span> <span class="n">pg</span><span class="o">.</span><span class="n">sprite</span><span class="o">.</span><span class="n">RenderPlain</span><span class="p">((</span><span class="n">chimp</span><span class="p">,</span> <span class="n">fist</span><span class="p">))</span>
466  <span class="n">clock</span> <span class="o">=</span> <span class="n">pg</span><span class="o">.</span><span class="n">time</span><span class="o">.</span><span class="n">Clock</span><span class="p">()</span>
467  </pre></div>
468  </div>
469  <p>First we load two sound effects using the <cite>load_sound</cite> function we defined
470  above. Then we create an instance of each of our sprite classes. And lastly
471  we create a sprite <a class="reference internal" href="../ref/sprite.html#pygame.sprite.Group" title="pygame.sprite.Group"><code class="xref py py-class docutils literal notranslate"><span class="pre">Group</span></code></a> which will contain all
472  our sprites.</p>
473  <p>We actually use a special sprite group named <a class="reference internal" href="../ref/sprite.html#pygame.sprite.RenderPlain" title="pygame.sprite.RenderPlain"><code class="xref py py-class docutils literal notranslate"><span class="pre">RenderPlain</span></code></a>. This sprite group can draw all the sprites it
474  contains to the screen. It is called <cite>RenderPlain</cite> because there are actually
475  more advanced Render groups. But for our game, we just need simple drawing. We
476  create the group named &quot;allsprites&quot; by passing a list with all the sprites that
477  should belong in the group. We could later on add or remove sprites from this
478  group, but in this game we won't need to.</p>
479  <p>The <cite>clock</cite> object we create will be used to help control our game's framerate.
480  we will use it in the main loop of our game to make sure it doesn't run too fast.</p>
481  </section>
482  <section id="main-loop">
483  <h3>Main Loop<a class="headerlink" href="#main-loop" title="Permalink to this headline">¶</a></h3>
484  <p>Nothing much here, just an infinite loop.</p>
485  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">going</span> <span class="o">=</span> <span class="kc">True</span>
486  <span class="k">while</span> <span class="n">going</span><span class="p">:</span>
487      <span class="n">clock</span><span class="o">.</span><span class="n">tick</span><span class="p">(</span><span class="mi">60</span><span class="p">)</span>
488  </pre></div>
489  </div>
490  <p>All games run in some sort of loop. The usual order of things is to
491  check on the state of the computer and user input, move and update the
492  state of all the objects, and then draw them to the screen. You'll see
493  that this example is no different.</p>
494  <p>We also make a call to our <cite>clock</cite> object, which will make sure our game
495  doesn't run faster than 60 frames per second.</p>
496  </section>
497  <section id="handle-all-input-events">
498  <h3>Handle All Input Events<a class="headerlink" href="#handle-all-input-events" title="Permalink to this headline">¶</a></h3>
499  <p>This is an extremely simple case of working the event queue.</p>
500  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">for</span> <span class="n">event</span> <span class="ow">in</span> <span class="n">pg</span><span class="o">.</span><span class="n">event</span><span class="o">.</span><span class="n">get</span><span class="p">():</span>
501      <span class="k">if</span> <span class="n">event</span><span class="o">.</span><span class="n">type</span> <span class="o">==</span> <span class="n">pg</span><span class="o">.</span><span class="n">QUIT</span><span class="p">:</span>
502          <span class="n">going</span> <span class="o">=</span> <span class="kc">False</span>
503      <span class="k">elif</span> <span class="n">event</span><span class="o">.</span><span class="n">type</span> <span class="o">==</span> <span class="n">pg</span><span class="o">.</span><span class="n">KEYDOWN</span> <span class="ow">and</span> <span class="n">event</span><span class="o">.</span><span class="n">key</span> <span class="o">==</span> <span class="n">pg</span><span class="o">.</span><span class="n">K_ESCAPE</span><span class="p">:</span>
504          <span class="n">going</span> <span class="o">=</span> <span class="kc">False</span>
505      <span class="k">elif</span> <span class="n">event</span><span class="o">.</span><span class="n">type</span> <span class="o">==</span> <span class="n">pg</span><span class="o">.</span><span class="n">MOUSEBUTTONDOWN</span><span class="p">:</span>
506          <span class="k">if</span> <span class="n">fist</span><span class="o">.</span><span class="n">punch</span><span class="p">(</span><span class="n">chimp</span><span class="p">):</span>
507              <span class="n">punch_sound</span><span class="o">.</span><span class="n">play</span><span class="p">()</span>  <span class="c1"># punch</span>
508              <span class="n">chimp</span><span class="o">.</span><span class="n">punched</span><span class="p">()</span>
509          <span class="k">else</span><span class="p">:</span>
510              <span class="n">whiff_sound</span><span class="o">.</span><span class="n">play</span><span class="p">()</span>  <span class="c1"># miss</span>
511      <span class="k">elif</span> <span class="n">event</span><span class="o">.</span><span class="n">type</span> <span class="o">==</span> <span class="n">pg</span><span class="o">.</span><span class="n">MOUSEBUTTONUP</span><span class="p">:</span>
512          <span class="n">fist</span><span class="o">.</span><span class="n">unpunch</span><span class="p">()</span>
513  </pre></div>
514  </div>
515  <p>First we get all the available Events from pygame and loop through each
516  of them. The first two tests see if the user has quit our game, or pressed
517  the escape key. In these cases we just set <code class="docutils literal notranslate"><span class="pre">going</span></code> to <code class="docutils literal notranslate"><span class="pre">False</span></code>, allowing
518  us out of the infinite loop.</p>
519  <p>Next we just check to see if the mouse button was pressed or released.
520  If the button was pressed, we ask the fist object if it has collided with
521  the chimp. We play the appropriate sound effect, and if the monkey was hit,
522  we tell him to start spinning (by calling his <cite>punched()</cite> method).</p>
523  </section>
524  <section id="update-the-sprites">
525  <h3>Update the Sprites<a class="headerlink" href="#update-the-sprites" title="Permalink to this headline">¶</a></h3>
526  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">allsprites</span><span class="o">.</span><span class="n">update</span><span class="p">()</span>
527  </pre></div>
528  </div>
529  <p>Sprite groups have an <cite>update()</cite> method, which simply calls the update method
530  for all the sprites it contains. Each of the objects will move around, depending
531  on which state they are in. This is where the chimp will move one step side
532  to side, or spin a little farther if he was recently punched.</p>
533  </section>
534  <section id="draw-the-entire-scene">
535  <h3>Draw The Entire Scene<a class="headerlink" href="#draw-the-entire-scene" title="Permalink to this headline">¶</a></h3>
536  <p>Now that all the objects are in the right place, time to draw them.</p>
537  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">screen</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="n">background</span><span class="p">,</span> <span class="p">(</span><span class="mi">0</span><span class="p">,</span> <span class="mi">0</span><span class="p">))</span>
538  <span class="n">allsprites</span><span class="o">.</span><span class="n">draw</span><span class="p">(</span><span class="n">screen</span><span class="p">)</span>
539  <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">flip</span><span class="p">()</span>
540  </pre></div>
541  </div>
542  <p>The first blit call will draw the background onto the entire screen. This
543  erases everything we saw from the previous frame (slightly inefficient, but
544  good enough for this game). Next we call the <cite>draw()</cite> method of the sprite
545  container. Since this sprite container is really an instance of the &quot;DrawPlain&quot;
546  sprite group, it knows how to draw our sprites. Lastly, we <cite>flip()</cite> the contents
547  of pygame's software double buffer to the screen. This makes everything we've
548  drawn visible all at once.</p>
549  </section>
550  <section id="game-over">
551  <h3>Game Over<a class="headerlink" href="#game-over" title="Permalink to this headline">¶</a></h3>
552  <p>User has quit, time to clean up.</p>
553  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">pg</span><span class="o">.</span><span class="n">quit</span><span class="p">()</span>
554  </pre></div>
555  </div>
556  <p>Cleaning up the running game in <em>pygame</em> is extremely simple.
557  Since all variables are automatically destructed, we don't really have to do
558  anything, but calling <cite>pg.quit()</cite> explicitly cleans up pygame's internals.</p>
559  </section>
