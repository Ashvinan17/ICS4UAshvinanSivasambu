101  <h2>Setting Display Modes<a class="headerlink" href="#setting-display-modes" title="Permalink to this headline">¶</a></h2>
102  <dl class="docinfo field-list simple">
103  <dt class="field-odd">Author</dt>
104  <dd class="field-odd"><p>Pete Shinners</p>
105  </dd>
106  <dt class="field-even">Contact</dt>
107  <dd class="field-even"><p><a class="reference external" href="mailto:pete&#37;&#52;&#48;shinners&#46;org">pete<span>&#64;</span>shinners<span>&#46;</span>org</a></p>

110  <section id="introduction">
111  <h3>Introduction<a class="headerlink" href="#introduction" title="Permalink to this headline">¶</a></h3>
112  <p>Setting the display mode in <em>pygame</em> creates a visible image surface
113  on the monitor.
114  This surface can either cover the full screen, or be windowed
115  on platforms that support a window manager.
116  The display surface is nothing more than a standard <em>pygame</em> surface object.
117  There are special functions needed in the <a class="tooltip reference internal" href="../ref/display.html#module-pygame.display" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.display</span></code><span class="tooltip-content">pygame module to control the display window and screen</span></a>
118  module to keep the image surface contents updated on the monitor.</p>
119  <p>Setting the display mode in <em>pygame</em> is an easier task than with most
120  graphic libraries.
121  The advantage is if your display mode is not available,
122  <em>pygame</em> will emulate the display mode that you asked for.
123  <em>Pygame</em> will select a display resolution and color depth that best matches
124  the settings you have requested,
125  then allow you to access the display with the format you have requested.
126  In reality, since the <a class="tooltip reference internal" href="../ref/display.html#module-pygame.display" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.display</span></code><span class="tooltip-content">pygame module to control the display window and screen</span></a> module is
127  a binding around the SDL library, SDL is really doing all this work.</p>
128  <p>There are advantages and disadvantages to setting the display mode in this
129  manner.
130  The advantage is that if your game requires a specific display mode,
131  your game will run on platforms that do not support your requirements.
132  It also makes life easier when you're getting something started,
133  it is always easy to go back later and make the mode selection a little more
134  particular.
135  The disadvantage is that what you request is not always what you will get.
136  There is also a performance penalty when the display mode must be emulated.
137  This tutorial will help you understand the different methods for querying
138  the platforms display capabilities, and setting the display mode for your game.</p>
139  </section>
140  <section id="setting-basics">
141  <h3>Setting Basics<a class="headerlink" href="#setting-basics" title="Permalink to this headline">¶</a></h3>
142  <p>The first thing to learn about is how to actually set the current display mode.
143  The display mode may be set at any time after the <a class="tooltip reference internal" href="../ref/display.html#module-pygame.display" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.display</span></code><span class="tooltip-content">pygame module to control the display window and screen</span></a>
144  module has been initialized.
145  If you have previously set the display mode,
146  setting it again will change the current mode.
147  Setting the display mode is handled with the function
148  <a class="tooltip reference internal" href="../ref/display.html#pygame.display.set_mode" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.display.set_mode((width,</span> <span class="pre">height),</span> <span class="pre">flags,</span> <span class="pre">depth)</span></code><span class="tooltip-content">Initialize a window or screen for display</span></a>.
149  The only required argument in this function is a sequence containing
150  the width and height of the new display mode.
151  The depth flag is the requested bits per pixel for the surface.
152  If the given depth is 8, <em>pygame</em> will create a color-mapped surface.
153  When given a higher bit depth, <em>pygame</em> will use a packed color mode.
154  Much more information about depths and color modes can be found in the
155  documentation for the display and surface modules.
156  The default value for depth is 0.
157  When given an argument of 0, <em>pygame</em> will select the best bit depth to use,
158  usually the same as the system's current bit depth.
159  The flags argument lets you control extra features for the display mode.
160  Again, more information about this is found in the <em>pygame</em> reference documents.</p>
161  </section>
162  <section id="how-to-decide">
163  <h3>How to Decide<a class="headerlink" href="#how-to-decide" title="Permalink to this headline">¶</a></h3>
164  <p>So how do you select a display mode that is going to work best with your
165  graphic resources and the platform your game is running on?
166  There are several methods for gathering information about the display device.
167  All of these methods must be called after the display module has been
168  initialized, but you likely want to call them before setting the display mode.
169  First, <a class="tooltip reference internal" href="../ref/display.html#pygame.display.Info" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.display.Info()</span></code><span class="tooltip-content">Create a video display information object</span></a>
170  will return a special object type of VidInfo,
171  which can tell you a lot about the graphics driver capabilities.
172  The function
173  <a class="tooltip reference internal" href="../ref/display.html#pygame.display.list_modes" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.display.list_modes(depth,</span> <span class="pre">flags)</span></code><span class="tooltip-content">Get list of available fullscreen modes</span></a>
174  can be used to find the supported graphic modes by the system.
175  <a class="tooltip reference internal" href="../ref/display.html#pygame.display.mode_ok" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.display.mode_ok((width,</span> <span class="pre">height),</span> <span class="pre">flags,</span> <span class="pre">depth)</span></code><span class="tooltip-content">Pick the best color depth for a display mode</span></a> takes the same arguments as
176  <a class="reference internal" href="../ref/display.html#pygame.display.set_mode" title="pygame.display.set_mode"><code class="xref py py-func docutils literal notranslate"><span class="pre">set_mode()</span></code></a>,
177  but returns the closest matching bit depth to the one you request.
178  Lastly, <a class="tooltip reference internal" href="../ref/display.html#pygame.display.get_driver" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.display.get_driver()</span></code><span class="tooltip-content">Get the name of the pygame display backend</span></a>
179  will return the name of the graphics driver selected by <em>pygame</em>.</p>
180  <p>Just remember the golden rule.
181  <em>Pygame</em> will work with pretty much any display mode you request.
182  Some display modes will need to be emulated,
183  which will slow your game down,
184  since <em>pygame</em> will need to convert every update you make to the
185  &quot;real&quot; display mode. The best bet is to always let <em>pygame</em>
186  choose the best bit depth,
187  and convert all your graphic resources to that format when they are loaded.
188  You let <em>pygame</em> choose its bit depth by calling
189  <a class="reference internal" href="../ref/display.html#pygame.display.set_mode" title="pygame.display.set_mode"><code class="xref py py-func docutils literal notranslate"><span class="pre">set_mode()</span></code></a>
190  with no depth argument or a depth of 0,
191  or you can call
192  <a class="reference internal" href="../ref/display.html#pygame.display.mode_ok" title="pygame.display.mode_ok"><code class="xref py py-func docutils literal notranslate"><span class="pre">mode_ok()</span></code></a>
193  to find a closest matching bit depth to what you need.</p>
194  <p>When your display mode is windowed,
195  you usually must match the same bit depth as the desktop.
196  When you are fullscreen, some platforms can switch to any bit depth that
197  best suits your needs.
198  You can find the depth of the current desktop if you get a VidInfo object
199  before ever setting your display mode.</p>
200  <p>After setting the display mode,
201  you can find out information about its settings by getting a VidInfo object,
202  or by calling any of the Surface.get* methods on the display surface.</p>
203  </section>
204  <section id="functions">
205  <h3>Functions<a class="headerlink" href="#functions" title="Permalink to this headline">¶</a></h3>
206  <p>These are the routines you can use to determine the most appropriate
207  display mode.
208  You can find more information about these functions in the display module
209  documentation.</p>
210  <blockquote>
211  <div><p><a class="tooltip reference internal" href="../ref/display.html#pygame.display.mode_ok" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.display.mode_ok(size,</span> <span class="pre">flags,</span> <span class="pre">depth)</span></code><span class="tooltip-content">Pick the best color depth for a display mode</span></a></p>
212  <blockquote>
213  <div><p>This function takes the exact same arguments as pygame.display.set_mode().
214  It returns the best available bit depth for the mode you have described.
215  If this returns zero,
216  then the desired display mode is not available without emulation.</p>
217  </div></blockquote>
218  <p><a class="tooltip reference internal" href="../ref/display.html#pygame.display.list_modes" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.display.list_modes(depth,</span> <span class="pre">flags)</span></code><span class="tooltip-content">Get list of available fullscreen modes</span></a></p>
219  <blockquote>
220  <div><p>Returns a list of supported display modes with the requested
221  depth and flags.
222  An empty list is returned when there are no modes.
223  The flags argument defaults to <a class="reference internal" href="../ref/display.html#pygame.display.set_mode" title="pygame.display.set_mode"><code class="xref any py py-func docutils literal notranslate"><span class="pre">FULLSCREEN</span></code></a>.
224  If you specify your own flags without <a class="reference internal" href="../ref/display.html#pygame.display.set_mode" title="pygame.display.set_mode"><code class="xref any py py-func docutils literal notranslate"><span class="pre">FULLSCREEN</span></code></a>,
225  you will likely get a return value of -1.
226  This means that any display size is fine, since the display will be windowed.
227  Note that the listed modes are sorted largest to smallest.</p>
228  </div></blockquote>
229  <p><a class="tooltip reference internal" href="../ref/display.html#pygame.display.Info" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.display.Info()</span></code><span class="tooltip-content">Create a video display information object</span></a></p>
230  <blockquote>
231  <div><p>This function returns an object with many members describing
232  the display device.
233  Printing the VidInfo object will quickly show you all the
234  members and values for this object.</p>
235  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="kn">import</span> <span class="nn">pygame.display</span>
236  <span class="gp">&gt;&gt;&gt; </span><span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">init</span><span class="p">()</span>
237  <span class="gp">&gt;&gt;&gt; </span><span class="n">info</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">Info</span><span class="p">()</span>
238  <span class="gp">&gt;&gt;&gt; </span><span class="nb">print</span><span class="p">(</span><span class="n">info</span><span class="p">)</span>
239  <span class="go">&lt;VideoInfo(hw = 0, wm = 1,video_mem = 0</span>
240  <span class="go">        blit_hw = 0, blit_hw_CC = 0, blit_hw_A = 0,</span>
241  <span class="go">        blit_sw = 0, blit_sw_CC = 0, blit_sw_A = 0,</span>
242  <span class="go">        bitsize  = 32, bytesize = 4,</span>
243  <span class="go">        masks =  (16711680, 65280, 255, 0),</span>
244  <span class="go">        shifts = (16, 8, 0, 0),</span>
245  <span class="go">        losses =  (0, 0, 0, 8),</span>
246  <span class="go">        current_w = 1920, current_h = 1080</span>
247  <span class="go">&gt;</span>
248  </pre></div>
249  </div>
250  </div></blockquote>
251  </div></blockquote>
252  <p>You can test all these flags as simply members of the VidInfo object.</p>
253  </section>
254  <section id="examples">
255  <h3>Examples<a class="headerlink" href="#examples" title="Permalink to this headline">¶</a></h3>
256  <p>Here are some examples of different methods to init the graphics display.
257  They should help you get an idea of how to go about setting your display mode.</p>
258  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="c1">#give me the best depth with a 640 x 480 windowed display</span>
259  <span class="gp">&gt;&gt;&gt; </span><span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">set_mode</span><span class="p">((</span><span class="mi">640</span><span class="p">,</span> <span class="mi">480</span><span class="p">))</span>
260  
261  <span class="gp">&gt;&gt;&gt; </span><span class="c1">#give me the biggest 16-bit display available</span>
262  <span class="gp">&gt;&gt;&gt; </span><span class="n">modes</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">list_modes</span><span class="p">(</span><span class="mi">16</span><span class="p">)</span>
263  <span class="gp">&gt;&gt;&gt; </span><span class="k">if</span> <span class="ow">not</span> <span class="n">modes</span><span class="p">:</span>
264  <span class="gp">... </span>    <span class="nb">print</span><span class="p">(</span><span class="s1">&#39;16-bit not supported&#39;</span><span class="p">)</span>
265  <span class="gp">... </span><span class="k">else</span><span class="p">:</span>
266  <span class="gp">... </span>    <span class="nb">print</span><span class="p">(</span><span class="s1">&#39;Found Resolution:&#39;</span><span class="p">,</span> <span class="n">modes</span><span class="p">[</span><span class="mi">0</span><span class="p">])</span>
267  <span class="gp">... </span>    <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">set_mode</span><span class="p">(</span><span class="n">modes</span><span class="p">[</span><span class="mi">0</span><span class="p">],</span> <span class="n">FULLSCREEN</span><span class="p">,</span> <span class="mi">16</span><span class="p">)</span>
268  
269  <span class="gp">&gt;&gt;&gt; </span><span class="c1">#need an 8-bit surface, nothing else will do</span>
270  <span class="gp">&gt;&gt;&gt; </span><span class="k">if</span> <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">mode_ok</span><span class="p">((</span><span class="mi">800</span><span class="p">,</span> <span class="mi">600</span><span class="p">),</span> <span class="mi">0</span><span class="p">,</span> <span class="mi">8</span><span class="p">)</span> <span class="o">!=</span> <span class="mi">8</span><span class="p">:</span>
271  <span class="gp">... </span>    <span class="nb">print</span><span class="p">(</span><span class="s1">&#39;Can only work with an 8-bit display, sorry&#39;</span><span class="p">)</span>
272  <span class="gp">... </span><span class="k">else</span><span class="p">:</span>
273  <span class="gp">... </span>    <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">set_mode</span><span class="p">((</span><span class="mi">800</span><span class="p">,</span> <span class="mi">600</span><span class="p">),</span> <span class="mi">0</span><span class="p">,</span> <span class="mi">8</span><span class="p">)</span>
274  </pre></div>
