6    .. module:: pygame.font
7       :synopsis: pygame module for loading and rendering fonts
8    
9    | :sl:`pygame module for loading and rendering fonts`
10   
11   The font module allows for rendering TrueType fonts into a new Surface object.
12   It accepts any UCS-2 character ('\u0001' to '\uFFFF'). This module is optional
13   and requires SDL_ttf as a dependency. You should test that :mod:`pygame.font`
14   is available and initialized before attempting to use the module.
15   
16   Most of the work done with fonts are done by using the actual Font objects. The
17   module by itself only has routines to initialize the module and create Font
18   objects with ``pygame.font.Font()``.
19   
20   You can load fonts from the system by using the ``pygame.font.SysFont()``
21   function. There are a few other functions to help lookup the system fonts.
22   
23   Pygame comes with a builtin default font. This can always be accessed by
24   passing None as the font name.
25   
26   To use the :mod:`pygame.freetype` based ``pygame.ftfont`` as
27   :mod:`pygame.font` define the environment variable PYGAME_FREETYPE before the
28   first import of :mod:`pygame`. Module ``pygame.ftfont`` is a :mod:`pygame.font`
29   compatible module that passes all but one of the font module unit tests:
30   it does not have the UCS-2 limitation of the SDL_ttf based font module, so
31   fails to raise an exception for a code point greater than '\uFFFF'. If
32   :mod:`pygame.freetype` is unavailable then the SDL_ttf font module will be
33   loaded instead.

92   .. function:: match_font
93   
94      | :sl:`find a specific font on the system`
95      | :sg:`match_font(name, bold=False, italic=False) -> path`
96   
97      Returns the full path to a font file on the system. If bold or italic are
98      set to true, this will attempt to find the correct family of font.
99   
100     The font name can also be an iterable of font names, a string of
101     comma-separated font names, or a bytes of comma-separated font names, in
102     which case the set of names will be searched in order.
103     If none of the given names are found, None is returned.
104  
105     .. versionadded:: 2.0.1 Accept an iterable of font names.
106  
107     Example:
108  
109     ::
110  
111         print pygame.font.match_font('bitstreamverasans')
112         # output is: /usr/share/fonts/truetype/ttf-bitstream-vera/Vera.ttf
113         # (but only if you have Vera on your system)

117  .. function:: SysFont
118  
119     | :sl:`create a Font object from the system fonts`
120     | :sg:`SysFont(name, size, bold=False, italic=False) -> Font`
121  
122     Return a new Font object that is loaded from the system fonts. The font will
123     match the requested bold and italic flags. Pygame uses a small set of common
124     font aliases. If the specific font you ask for is not available, a reasonable
125     alternative may be used. If a suitable system font is not found this will
126     fall back on loading the default pygame font.
127  
128     The font name can also be an iterable of font names, a string of
129     comma-separated font names, or a bytes of comma-separated font names, in
130     which case the set of names will be searched in order.
131  
132     .. versionadded:: 2.0.1 Accept an iterable of font names.

136  .. class:: Font
137  
138     | :sl:`create a new Font object from a file`
139     | :sg:`Font(filename, size) -> Font`
140     | :sg:`Font(pathlib.Path, size) -> Font`
141     | :sg:`Font(object, size) -> Font`
142  
143     Load a new font from a given filename or a python file object. The size is
144     the height of the font in pixels. If the filename is None the pygame default
145     font will be loaded. If a font cannot be loaded from the arguments given an
146     exception will be raised. Once the font is created the size cannot be
147     changed.
148  
149     Font objects are mainly used to render text into new Surface objects. The
150     render can emulate bold or italic features, but it is better to load from a
151     font with actual italic or bold glyphs. The rendered text can be regular
152     strings or unicode.
153  
154     .. attribute:: bold
155  
156        | :sl:`Gets or sets whether the font should be rendered in (faked) bold.`
157        | :sg:`bold -> bool`
158  
159        Whether the font should be rendered in bold.
160  
161        When set to True, this enables the bold rendering of text. This
162        is a fake stretching of the font that doesn't look good on many
163        font types. If possible load the font from a real bold font
164        file. While bold, the font will have a different width than when
165        normal. This can be mixed with the italic and underline modes.

204     .. method:: render
205  
206        | :sl:`draw text on a new Surface`
207        | :sg:`render(text, antialias, color, background=None) -> Surface`
208  
209        This creates a new Surface with the specified text rendered on it. pygame
210        provides no way to directly draw text on an existing Surface: instead you
211        must use ``Font.render()`` to create an image (Surface) of the text, then
212        blit this image onto another Surface.
213  
214        The text can only be a single line: newline characters are not rendered.
215        Null characters ('\x00') raise a TypeError. Both Unicode and char (byte)
216        strings are accepted. For Unicode strings only UCS-2 characters
217        ('\u0001' to '\uFFFF') were previously supported and any greater unicode
218        codepoint would raise a UnicodeError. Now, characters in the UCS-4 range
219        are supported. For char strings a ``LATIN1`` encoding is assumed. The
220        antialias argument is a boolean: if true the characters will have smooth
221        edges. The color argument is the color of the text
222        [e.g.: (0,0,255) for blue]. The optional background argument is a color
223        to use for the text background. If no background is passed the area
224        outside the text will be transparent.
225  
226        The Surface returned will be of the dimensions required to hold the text.
227        (the same as those returned by Font.size()). If an empty string is passed
228        for the text, a blank surface will be returned that is zero pixel wide and
229        the height of the font.
230  
231        Depending on the type of background and antialiasing used, this returns
232        different types of Surfaces. For performance reasons, it is good to know
233        what type of image will be used. If antialiasing is not used, the return
234        image will always be an 8-bit image with a two-color palette. If the
235        background is transparent a colorkey will be set. Antialiased images are
236        rendered to 24-bit ``RGB`` images. If the background is transparent a
237        pixel alpha will be included.
238  
239        Optimization: if you know that the final destination for the text (on the
240        screen) will always have a solid background, and the text is antialiased,
241        you can improve performance by specifying the background color. This will
242        cause the resulting image to maintain transparency information by
243        colorkey rather than (much less efficient) alpha values.
244  
245        If you render '\\n' an unknown char will be rendered. Usually a
246        rectangle. Instead you need to handle new lines yourself.
247  
248        Font rendering is not thread safe: only a single thread can render text
249        at any time.
250  
251        .. versionchanged:: 2.0.3 Rendering UCS_4 unicode works and does not
252          raise an exception. Use `if hasattr(pygame.font, 'UCS_4'):` to see if
253          pygame supports rendering UCS_4 unicode including more languages and
254          emoji.

258     .. method:: size
259  
260        | :sl:`determine the amount of space needed to render text`
261        | :sg:`size(text) -> (width, height)`
262  
263        Returns the dimensions needed to render the text. This can be used to
264        help determine the positioning needed for text before it is rendered. It
265        can also be used for wordwrapping and other layout effects.
266  
267        Be aware that most fonts use kerning which adjusts the widths for
268        specific letter pairs. For example, the width for "ae" will not always
269        match the width for "a" + "e".

340        | :sg:`metrics(text) -> list`
341  
342        The list contains tuples for each character, which contain the minimum
343        ``X`` offset, the maximum ``X`` offset, the minimum ``Y`` offset, the
344        maximum ``Y`` offset and the advance offset (bearing plus width) of the
345        character. [(minx, maxx, miny, maxy, advance), (minx, maxx, miny, maxy,
346        advance), ...]. None is entered in the list for each unrecognized
347        character.
