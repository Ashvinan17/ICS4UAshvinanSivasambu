100  <section id="camera-module-introduction">
101  <h2>Camera Module Introduction<a class="headerlink" href="#camera-module-introduction" title="Permalink to this headline">¶</a></h2>
102  <dl class="docinfo field-list simple">
103  <dt class="field-odd">Author</dt>
104  <dd class="field-odd"><p>by Nirav Patel</p>
105  </dd>
106  <dt class="field-even">Contact</dt>
107  <dd class="field-even"><p><a class="reference external" href="mailto:nrp&#37;&#52;&#48;eclecti&#46;cc">nrp<span>&#64;</span>eclecti<span>&#46;</span>cc</a></p>
108  </dd>
109  </dl>
110  <p>Pygame 1.9 comes with support for interfacing cameras, allowing you to capture
111  still images, watch live streams, and do some simple computer vision.  This
112  tutorial will cover all of those use cases, providing code samples you can base
113  your app or game on.  You can refer to the <a class="reference internal" href="../ref/camera.html#module-pygame.camera" title="pygame.camera: pygame module for camera use"><code class="xref py py-mod docutils literal notranslate"><span class="pre">reference</span> <span class="pre">documentation</span></code></a>
114  for the full API.</p>
115  <div class="admonition note">
116  <p class="admonition-title">Note</p>
117  <p>As of Pygame 1.9, the camera module offers native support for cameras
118  that use v4l2 on Linux.  There is support for other platforms via Videocapture
119  or OpenCV, but this guide will focus on the native module.  Most of the code
120  will be valid for other platforms, but certain things like controls will not
121  work.  The module is also marked as <strong>EXPERIMENTAL</strong>, meaning the API could
122  change in subsequent versions.</p>
123  </div>

130  <span class="n">pygame</span><span class="o">.</span><span class="n">init</span><span class="p">()</span>

133  </div>

137  <section id="capturing-a-single-image">
138  <h3>Capturing a Single Image<a class="headerlink" href="#capturing-a-single-image" title="Permalink to this headline">¶</a></h3>
139  <p>Now we will go over the simplest case of opening a camera and capturing a frame
140  as a surface.  In the below example, we assume that there is a camera at
141  /dev/video0 on the computer, and initialize it with a size of 640 by 480.
142  The surface called image is whatever the camera was seeing when get_image() was
143  called.</p>
144  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">cam</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">camera</span><span class="o">.</span><span class="n">Camera</span><span class="p">(</span><span class="s2">&quot;/dev/video0&quot;</span><span class="p">,(</span><span class="mi">640</span><span class="p">,</span><span class="mi">480</span><span class="p">))</span>
145  <span class="n">cam</span><span class="o">.</span><span class="n">start</span><span class="p">()</span>
146  <span class="n">image</span> <span class="o">=</span> <span class="n">cam</span><span class="o">.</span><span class="n">get_image</span><span class="p">()</span>
147  </pre></div>
148  </div>
149  <section id="listing-connected-cameras">
150  <h4>Listing Connected Cameras<a class="headerlink" href="#listing-connected-cameras" title="Permalink to this headline">¶</a></h4>
151  <p>You may be wondering, what if we don't know the exact path of the camera?
152  We can ask the module to provide a list of cameras attached to the
153  computer and initialize the first camera in the list.</p>
154  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">camlist</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">camera</span><span class="o">.</span><span class="n">list_cameras</span><span class="p">()</span>

156      <span class="n">cam</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">camera</span><span class="o">.</span><span class="n">Camera</span><span class="p">(</span><span class="n">camlist</span><span class="p">[</span><span class="mi">0</span><span class="p">],(</span><span class="mi">640</span><span class="p">,</span><span class="mi">480</span><span class="p">))</span>
157  </pre></div>
158  </div>

170  <section id="capturing-a-live-stream">
171  <h3>Capturing a Live Stream<a class="headerlink" href="#capturing-a-live-stream" title="Permalink to this headline">¶</a></h3>
172  <p>The rest of this tutorial will be based around capturing a live stream of
173  images.  For this, we will be using the class below.  As described, it will
174  simply blit a constant stream of camera frames to the screen, effectively
175  showing live video.  It is basically what you would expect, looping get_image(),
176  blitting to the display surface, and flipping it.  For performance reasons,
177  we will be supplying the camera with the same surface to use each time.</p>
178  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">class</span> <span class="nc">Capture</span><span class="p">(</span><span class="nb">object</span><span class="p">):</span>
179      <span class="k">def</span> <span class="fm">__init__</span><span class="p">(</span><span class="bp">self</span><span class="p">):</span>
180          <span class="bp">self</span><span class="o">.</span><span class="n">size</span> <span class="o">=</span> <span class="p">(</span><span class="mi">640</span><span class="p">,</span><span class="mi">480</span><span class="p">)</span>
181          <span class="c1"># create a display surface. standard pygame stuff</span>
182          <span class="bp">self</span><span class="o">.</span><span class="n">display</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">set_mode</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">size</span><span class="p">,</span> <span class="mi">0</span><span class="p">)</span>
183  
184          <span class="c1"># this is the same as what we saw before</span>
185          <span class="bp">self</span><span class="o">.</span><span class="n">clist</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">camera</span><span class="o">.</span><span class="n">list_cameras</span><span class="p">()</span>
186          <span class="k">if</span> <span class="ow">not</span> <span class="bp">self</span><span class="o">.</span><span class="n">clist</span><span class="p">:</span>
187              <span class="k">raise</span> <span class="ne">ValueError</span><span class="p">(</span><span class="s2">&quot;Sorry, no cameras detected.&quot;</span><span class="p">)</span>
188          <span class="bp">self</span><span class="o">.</span><span class="n">cam</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">camera</span><span class="o">.</span><span class="n">Camera</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">clist</span><span class="p">[</span><span class="mi">0</span><span class="p">],</span> <span class="bp">self</span><span class="o">.</span><span class="n">size</span><span class="p">)</span>
189          <span class="bp">self</span><span class="o">.</span><span class="n">cam</span><span class="o">.</span><span class="n">start</span><span class="p">()</span>
190  
191          <span class="c1"># create a surface to capture to.  for performance purposes</span>
192          <span class="c1"># bit depth is the same as that of the display surface.</span>
193          <span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">surface</span><span class="o">.</span><span class="n">Surface</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">size</span><span class="p">,</span> <span class="mi">0</span><span class="p">,</span> <span class="bp">self</span><span class="o">.</span><span class="n">display</span><span class="p">)</span>
194  
195      <span class="k">def</span> <span class="nf">get_and_flip</span><span class="p">(</span><span class="bp">self</span><span class="p">):</span>
196          <span class="c1"># if you don&#39;t want to tie the framerate to the camera, you can check</span>
197          <span class="c1"># if the camera has an image ready.  note that while this works</span>
198          <span class="c1"># on most cameras, some will never return true.</span>
199          <span class="k">if</span> <span class="bp">self</span><span class="o">.</span><span class="n">cam</span><span class="o">.</span><span class="n">query_image</span><span class="p">():</span>
200              <span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span> <span class="o">=</span> <span class="bp">self</span><span class="o">.</span><span class="n">cam</span><span class="o">.</span><span class="n">get_image</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">)</span>
201  
202          <span class="c1"># blit it to the display surface.  simple!</span>
203          <span class="bp">self</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">,</span> <span class="p">(</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">))</span>
204          <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">flip</span><span class="p">()</span>
205  
206      <span class="k">def</span> <span class="nf">main</span><span class="p">(</span><span class="bp">self</span><span class="p">):</span>
207          <span class="n">going</span> <span class="o">=</span> <span class="kc">True</span>
208          <span class="k">while</span> <span class="n">going</span><span class="p">:</span>
209              <span class="n">events</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">event</span><span class="o">.</span><span class="n">get</span><span class="p">()</span>
210              <span class="k">for</span> <span class="n">e</span> <span class="ow">in</span> <span class="n">events</span><span class="p">:</span>
211                  <span class="k">if</span> <span class="n">e</span><span class="o">.</span><span class="n">type</span> <span class="o">==</span> <span class="n">QUIT</span> <span class="ow">or</span> <span class="p">(</span><span class="n">e</span><span class="o">.</span><span class="n">type</span> <span class="o">==</span> <span class="n">KEYDOWN</span> <span class="ow">and</span> <span class="n">e</span><span class="o">.</span><span class="n">key</span> <span class="o">==</span> <span class="n">K_ESCAPE</span><span class="p">):</span>
212                      <span class="c1"># close the camera safely</span>
213                      <span class="bp">self</span><span class="o">.</span><span class="n">cam</span><span class="o">.</span><span class="n">stop</span><span class="p">()</span>
214                      <span class="n">going</span> <span class="o">=</span> <span class="kc">False</span>
215  
216              <span class="bp">self</span><span class="o">.</span><span class="n">get_and_flip</span><span class="p">()</span>
217  </pre></div>
218  </div>
219  <p>Since get_image() is a blocking call that could take quite a bit of time on a
220  slow camera, this example uses query_image() to see if the camera is ready.
221  This allows you to separate the framerate of your game from that of your camera.
222  It is also possible to have the camera capturing images in a separate thread,
223  for approximately the same performance gain, if you find that your camera does
224  not support the query_image() function correctly.</p>
225  </section>

230  <section id="colorspaces">
231  <h4>Colorspaces<a class="headerlink" href="#colorspaces" title="Permalink to this headline">¶</a></h4>
232  <p>When initializing a camera, colorspace is an optional parameter, with 'RGB',
233  'YUV', and 'HSV' as the possible choices.  YUV and HSV are both generally more
234  useful for computer vision than RGB, and allow you to more easily threshold by
235  color, something we will look at later in the tutorial.</p>

249  <section id="thresholding">
250  <h4>Thresholding<a class="headerlink" href="#thresholding" title="Permalink to this headline">¶</a></h4>
251  <p>Using the threshold() function from the transform module, one can do simple
252  green screen like effects, or isolate specifically colored objects in a scene.
253  In the below example, we threshold out just the green tree and make the rest
254  of the image black.  Check the reference documentation for details on the
255  <a class="reference internal" href="../ref/transform.html#pygame.transform.threshold" title="pygame.transform.threshold"><code class="xref py py-func docutils literal notranslate"><span class="pre">threshold</span> <span class="pre">function</span></code></a>.</p>
256  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="bp">self</span><span class="o">.</span><span class="n">thresholded</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">surface</span><span class="o">.</span><span class="n">Surface</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">size</span><span class="p">,</span> <span class="mi">0</span><span class="p">,</span> <span class="bp">self</span><span class="o">.</span><span class="n">display</span><span class="p">)</span>
257  <span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span> <span class="o">=</span> <span class="bp">self</span><span class="o">.</span><span class="n">cam</span><span class="o">.</span><span class="n">get_image</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">)</span>
258  <span class="n">pygame</span><span class="o">.</span><span class="n">transform</span><span class="o">.</span><span class="n">threshold</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">thresholded</span><span class="p">,</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">,(</span><span class="mi">0</span><span class="p">,</span><span class="mi">255</span><span class="p">,</span><span class="mi">0</span><span class="p">),(</span><span class="mi">90</span><span class="p">,</span><span class="mi">170</span><span class="p">,</span><span class="mi">170</span><span class="p">),(</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">),</span><span class="mi">2</span><span class="p">)</span>
259  </pre></div>
260  </div>
261  <img alt="../_images/camera_thresholded.jpg" class="trailing" src="../_images/camera_thresholded.jpg" />
262  <p>Of course, this is only useful if you already know the exact color of the object
263  you are looking for.  To get around this and make thresholding usable in the
264  real world, we need to add a calibration stage where we identify the color of an
265  object and use it to threshold against.  We will be using the average_color()
266  function of the transform module to do this.  Below is an example calibration
267  function that you could loop until an event like a key press, and an image of
268  what it would look like.  The color inside the box will be the one that is
269  used for the threshold.  Note that we are using the HSV colorspace in the below
270  images.</p>
271  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">def</span> <span class="nf">calibrate</span><span class="p">(</span><span class="bp">self</span><span class="p">):</span>
272      <span class="c1"># capture the image</span>
273      <span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span> <span class="o">=</span> <span class="bp">self</span><span class="o">.</span><span class="n">cam</span><span class="o">.</span><span class="n">get_image</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">)</span>
274      <span class="c1"># blit it to the display surface</span>
275      <span class="bp">self</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">,</span> <span class="p">(</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">))</span>
276      <span class="c1"># make a rect in the middle of the screen</span>
277      <span class="n">crect</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">draw</span><span class="o">.</span><span class="n">rect</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">display</span><span class="p">,</span> <span class="p">(</span><span class="mi">255</span><span class="p">,</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">),</span> <span class="p">(</span><span class="mi">145</span><span class="p">,</span><span class="mi">105</span><span class="p">,</span><span class="mi">30</span><span class="p">,</span><span class="mi">30</span><span class="p">),</span> <span class="mi">4</span><span class="p">)</span>
278      <span class="c1"># get the average color of the area inside the rect</span>
279      <span class="bp">self</span><span class="o">.</span><span class="n">ccolor</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">transform</span><span class="o">.</span><span class="n">average_color</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">,</span> <span class="n">crect</span><span class="p">)</span>
280      <span class="c1"># fill the upper left corner with that color</span>
281      <span class="bp">self</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">fill</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">ccolor</span><span class="p">,</span> <span class="p">(</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">,</span><span class="mi">50</span><span class="p">,</span><span class="mi">50</span><span class="p">))</span>
282      <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">flip</span><span class="p">()</span>
283  </pre></div>
284  </div>
285  <img alt="../_images/camera_average.jpg" class="trailing" src="../_images/camera_average.jpg" />
286  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">pygame</span><span class="o">.</span><span class="n">transform</span><span class="o">.</span><span class="n">threshold</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">thresholded</span><span class="p">,</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">,</span><span class="bp">self</span><span class="o">.</span><span class="n">ccolor</span><span class="p">,(</span><span class="mi">30</span><span class="p">,</span><span class="mi">30</span><span class="p">,</span><span class="mi">30</span><span class="p">),(</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">),</span><span class="mi">2</span><span class="p">)</span>
287  </pre></div>
288  </div>
289  <img alt="../_images/camera_thresh.jpg" class="trailing" src="../_images/camera_thresh.jpg" />
290  <p>You can use the same idea to do a simple green screen/blue screen, by first
291  getting a background image and then thresholding against it.  The below example
292  just has the camera pointed at a blank white wall in HSV colorspace.</p>
293  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">def</span> <span class="nf">calibrate</span><span class="p">(</span><span class="bp">self</span><span class="p">):</span>
294      <span class="c1"># capture a bunch of background images</span>
295      <span class="n">bg</span> <span class="o">=</span> <span class="p">[]</span>
296      <span class="k">for</span> <span class="n">i</span> <span class="ow">in</span> <span class="nb">range</span><span class="p">(</span><span class="mi">0</span><span class="p">,</span><span class="mi">5</span><span class="p">):</span>
297        <span class="n">bg</span><span class="o">.</span><span class="n">append</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">cam</span><span class="o">.</span><span class="n">get_image</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">background</span><span class="p">))</span>
298      <span class="c1"># average them down to one to get rid of some noise</span>
299      <span class="n">pygame</span><span class="o">.</span><span class="n">transform</span><span class="o">.</span><span class="n">average_surfaces</span><span class="p">(</span><span class="n">bg</span><span class="p">,</span><span class="bp">self</span><span class="o">.</span><span class="n">background</span><span class="p">)</span>
300      <span class="c1"># blit it to the display surface</span>
301      <span class="bp">self</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">background</span><span class="p">,</span> <span class="p">(</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">))</span>
302      <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">flip</span><span class="p">()</span>
303  </pre></div>
304  </div>
305  <img alt="../_images/camera_background.jpg" class="trailing" src="../_images/camera_background.jpg" />
306  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">pygame</span><span class="o">.</span><span class="n">transform</span><span class="o">.</span><span class="n">threshold</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">thresholded</span><span class="p">,</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">,(</span><span class="mi">0</span><span class="p">,</span><span class="mi">255</span><span class="p">,</span><span class="mi">0</span><span class="p">),(</span><span class="mi">30</span><span class="p">,</span><span class="mi">30</span><span class="p">,</span><span class="mi">30</span><span class="p">),(</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">),</span><span class="mi">1</span><span class="p">,</span><span class="bp">self</span><span class="o">.</span><span class="n">background</span><span class="p">)</span>
307  </pre></div>
308  </div>

311  <section id="using-the-mask-module">
312  <h4>Using the Mask Module<a class="headerlink" href="#using-the-mask-module" title="Permalink to this headline">¶</a></h4>
313  <p>The stuff above is great if you just want to display images, but with the
314  <a class="reference internal" href="../ref/mask.html#module-pygame.mask" title="pygame.mask: pygame module for image masks."><code class="xref py py-mod docutils literal notranslate"><span class="pre">mask</span> <span class="pre">module</span></code></a>, you can also use a camera as an
315  input device for a game.  For example, going back to the example of
316  thresholding out a specific object, we can find the position of that object and
317  use it to control an on screen object.</p>
318  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">def</span> <span class="nf">get_and_flip</span><span class="p">(</span><span class="bp">self</span><span class="p">):</span>
319      <span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span> <span class="o">=</span> <span class="bp">self</span><span class="o">.</span><span class="n">cam</span><span class="o">.</span><span class="n">get_image</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">)</span>
320      <span class="c1"># threshold against the color we got before</span>
321      <span class="n">mask</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">mask</span><span class="o">.</span><span class="n">from_threshold</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">,</span> <span class="bp">self</span><span class="o">.</span><span class="n">ccolor</span><span class="p">,</span> <span class="p">(</span><span class="mi">30</span><span class="p">,</span> <span class="mi">30</span><span class="p">,</span> <span class="mi">30</span><span class="p">))</span>
322      <span class="bp">self</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">snapshot</span><span class="p">,(</span><span class="mi">0</span><span class="p">,</span><span class="mi">0</span><span class="p">))</span>
323      <span class="c1"># keep only the largest blob of that color</span>
324      <span class="n">connected</span> <span class="o">=</span> <span class="n">mask</span><span class="o">.</span><span class="n">connected_component</span><span class="p">()</span>
325      <span class="c1"># make sure the blob is big enough that it isn&#39;t just noise</span>
326      <span class="k">if</span> <span class="n">mask</span><span class="o">.</span><span class="n">count</span><span class="p">()</span> <span class="o">&gt;</span> <span class="mi">100</span><span class="p">:</span>
327          <span class="c1"># find the center of the blob</span>
328          <span class="n">coord</span> <span class="o">=</span> <span class="n">mask</span><span class="o">.</span><span class="n">centroid</span><span class="p">()</span>
329          <span class="c1"># draw a circle with size variable on the size of the blob</span>
330          <span class="n">pygame</span><span class="o">.</span><span class="n">draw</span><span class="o">.</span><span class="n">circle</span><span class="p">(</span><span class="bp">self</span><span class="o">.</span><span class="n">display</span><span class="p">,</span> <span class="p">(</span><span class="mi">0</span><span class="p">,</span><span class="mi">255</span><span class="p">,</span><span class="mi">0</span><span class="p">),</span> <span class="n">coord</span><span class="p">,</span> <span class="nb">max</span><span class="p">(</span><span class="nb">min</span><span class="p">(</span><span class="mi">50</span><span class="p">,</span><span class="n">mask</span><span class="o">.</span><span class="n">count</span><span class="p">()</span><span class="o">/</span><span class="mi">400</span><span class="p">),</span><span class="mi">5</span><span class="p">))</span>
331      <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">flip</span><span class="p">()</span>
332  </pre></div>
333  </div>
334  <img alt="../_images/camera_mask.jpg" class="trailing" src="../_images/camera_mask.jpg" />
335  <p>This is just the most basic example.  You can track multiple different colored
336  blobs, find the outlines of objects, have collision detection between real life
337  and in game objects, get the angle of an object to allow for even finer control,
338  and more.  Have fun!</p>
