4    ======================

6    .. module:: pygame.joystick
7       :synopsis: Pygame module for interacting with joysticks, gamepads, and trackballs.
8    
9    | :sl:`Pygame module for interacting with joysticks, gamepads, and trackballs.`
10   
11   The joystick module manages the joystick devices on a computer.
12   Joystick devices include trackballs and video-game-style
13   gamepads, and the module allows the use of multiple buttons and "hats".
14   Computers may manage multiple joysticks at a time.
15   
16   Each instance of the Joystick class represents one gaming device plugged
17   into the computer. If a gaming pad has multiple joysticks on it, then the
18   joystick object can actually represent multiple joysticks on that single
19   game device.
20   
21   For a quick way to initialise the joystick module and get a list of Joystick instances
22   use the following code::
23   
24       pygame.joystick.init()
25       joysticks = [pygame.joystick.Joystick(x) for x in range(pygame.joystick.get_count())]
26   
27   The following event types will be generated by the joysticks ::
28   
29       JOYAXISMOTION JOYBALLMOTION JOYBUTTONDOWN JOYBUTTONUP JOYHATMOTION
30   
31   And in pygame 2, which supports hotplugging::
32   
33      JOYDEVICEADDED JOYDEVICEREMOVED
34   
35   Note that in pygame 2, joysticks events use a unique "instance ID". The device index
36   passed in the constructor to a Joystick object is not unique after devices have
37   been added and removed. You must call :meth:`Joystick.get_instance_id()` to find
38   the instance ID that was assigned to a Joystick on opening.
39   
40   The event queue needs to be pumped frequently for some of the methods to work.
41   So call one of pygame.event.get, pygame.event.wait, or pygame.event.pump regularly.

92   .. class:: Joystick
93   
94      | :sl:`Create a new Joystick object.`
95      | :sg:`Joystick(id) -> Joystick`
96   
97      Create a new joystick to access a physical device. The id argument must be a
98      value from ``0`` to ``pygame.joystick.get_count() - 1``.
99   
100     Joysticks are initialised on creation and are shut down when deallocated.
101     Once the device is initialized the pygame event queue will start receiving
102     events about its input.

104     .. versionchanged:: 2.0.0 Joystick objects are now opened immediately on creation.

202     .. method:: get_numaxes
203  
204        | :sl:`get the number of axes on a Joystick`
205        | :sg:`get_numaxes() -> int`
206  
207        Returns the number of input axes are on a Joystick. There will usually be
208        two for the position. Controls like rudders and throttles are treated as
209        additional axes.
210  
211        The ``pygame.JOYAXISMOTION`` events will be in the range from ``-1.0`` 
212        to ``1.0``. A value of ``0.0`` means the axis is centered. Gamepad devices 
213        will usually be ``-1``, ``0``, or ``1`` with no values in between. Older 
214        analog joystick axes will not always use the full ``-1`` to ``1`` range, 
215        and the centered value will be some area around ``0``. 
216        
217        Analog joysticks usually have a bit of noise in their axis, which will 
218        generate a lot of rapid small motion events.
219  
220        .. ## Joystick.get_numaxes ##
221  
222     .. method:: get_axis
223  
224        | :sl:`get the current position of an axis`
225        | :sg:`get_axis(axis_number) -> float`
226  
227        Returns the current position of a joystick axis. The value will range
228        from ``-1`` to ``1`` with a value of ``0`` being centered. You may want 
229        to take into account some tolerance to handle jitter, and joystick drift 
230        may keep the joystick from centering at ``0`` or using the full range of 
231        position values.
232  
233        The axis number must be an integer from ``0`` to ``get_numaxes() - 1``.
234  
235        When using gamepads both the control sticks and the analog triggers are
236        usually reported as axes.
237

288     .. method:: get_numhats
289  
290        | :sl:`get the number of hat controls on a Joystick`
291        | :sg:`get_numhats() -> int`
292  
293        Returns the number of joystick hats on a Joystick. Hat devices are like
294        miniature digital joysticks on a joystick. Each hat has two axes of
295        input.
296  
297        The ``pygame.JOYHATMOTION`` event is generated when the hat changes
298        position. The ``position`` attribute for the event contains a pair of 
299        values that are either ``-1``, ``0``, or ``1``. A position of ``(0, 0)`` 
300        means the hat is centered.
301  
302        .. ## Joystick.get_numhats ##
303  
304     .. method:: get_hat
305  
306        | :sl:`get the position of a joystick hat`
307        | :sg:`get_hat(hat_number) -> x, y`
308  
309        Returns the current position of a position hat. The position is given as
310        two values representing the ``x`` and ``y`` position for the hat. ``(0, 0)``
311        means centered. A value of ``-1`` means left/down and a value of ``1`` means
312        right/up: so ``(-1, 0)`` means left; ``(1, 0)`` means right; ``(0, 1)`` means 
313        up; ``(1, 1)`` means upper-right; etc.
314  
315        This value is digital, ``i.e.``, each coordinate can be ``-1``, ``0`` or ``1`` 
316        but never in-between.
317  
318        The hat number must be between ``0`` and ``get_numhats() - 1``.
319

322     .. method:: rumble
323  
324        | :sl:`Start a rumbling effect`
325        | :sg:`rumble(low_frequency, high_frequency, duration) -> bool`
326  
327        Start a rumble effect on the joystick, with the specified strength ranging
328        from 0 to 1. Duration is length of the effect, in ms. Setting the duration
329        to 0 will play the effect until another one overwrites it or
330        :meth:`Joystick.stop_rumble` is called. If an effect is already
331        playing, then it will be overwritten.
332  
333        Returns True if the rumble was played successfully or False if the
334        joystick does not support it or :meth:`pygame.version.SDL` is below 2.0.9.
335

369  Controller mappings are drawn from the underlying SDL library which pygame uses and they differ
