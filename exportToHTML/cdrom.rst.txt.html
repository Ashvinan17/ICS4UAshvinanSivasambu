6    .. module:: pygame.cdrom
7       :synopsis: pygame module for audio cdrom control
8    
9    | :sl:`pygame module for audio cdrom control`
10   
11   .. warning::
12   	This module is non functional in pygame 2.0 and above, unless you have manually compiled pygame with SDL1.
13   	This module will not be supported in the future.
14   	One alternative for python cdrom functionality is `pycdio <https://pypi.org/project/pycdio/>`_.
15   	
16   The cdrom module manages the ``CD`` and ``DVD`` drives on a computer. It can
17   also control the playback of audio CDs. This module needs to be initialized
18   before it can do anything. Each ``CD`` object you create represents a cdrom
19   drive and must also be initialized individually before it can do most things.
20   
21   .. function:: init
22   
23      | :sl:`initialize the cdrom module`
24      | :sg:`init() -> None`
25   
26      Initialize the cdrom module. This will scan the system for all ``CD``
27      devices. The module must be initialized before any other functions will
28      work. This automatically happens when you call ``pygame.init()``.
29   
30      It is safe to call this function more than once.

67   .. class:: CD
68   
69      | :sl:`class to manage a cdrom drive`
70      | :sg:`CD(id) -> CD`
71   
72      You can create a ``CD`` object for each cdrom on the system. Use
73      ``pygame.cdrom.get_count()`` to determine how many drives actually exist.
74      The id argument is an integer of the drive, starting at zero.
75   
76      The ``CD`` object is not initialized, you can only call ``CD.get_id()`` and
77      ``CD.get_name()`` on an uninitialized drive.
78   
79      It is safe to create multiple ``CD`` objects for the same drive, they will
80      all cooperate normally.
81   
82      .. method:: init
83   
84         | :sl:`initialize a cdrom drive for use`
85         | :sg:`init() -> None`
86   
87         Initialize the cdrom drive for use. The drive must be initialized for
88         most ``CD`` methods to work. Even if the rest of pygame has been
89         initialized.
90   
91         There may be a brief pause while the drive is initialized. Avoid
92         ``CD.init()`` if the program should not stop for a second or two.

117     .. method:: play
118  
119        | :sl:`start playing audio`
120        | :sg:`play(track, start=None, end=None) -> None`
121  
122        Playback audio from an audio cdrom in the drive. Besides the track number
123        argument, you can also pass a starting and ending time for playback. The
124        start and end time are in seconds, and can limit the section of an audio
125        track played.
126  
127        If you pass a start time but no end, the audio will play to the end of
128        the track. If you pass a start time and 'None' for the end time, the
129        audio will play to the end of the entire disc.
130  
131        See the ``CD.get_numtracks()`` and ``CD.get_track_audio()`` to find
132        tracks to playback.
133  
134        Note, track 0 is the first track on the ``CD``. Track numbers start at
135        zero.

152        | :sl:`temporarily stop audio playback`
153        | :sg:`pause() -> None`
154  
155        Temporarily stop audio playback on the ``CD``. The playback can be
156        resumed at the same point with the ``CD.resume()`` method. If the ``CD``
157        is not playing this method does nothing.

159        Note, track 0 is the first track on the ``CD``. Track numbers start at

273        | :sg:`get_all() -> [(audio, start, end, length), ...]`

276        information consists of a tuple with four values. The audio value is True
