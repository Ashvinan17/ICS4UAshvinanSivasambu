110  <section id="introduction">
111  <h3>Introduction<a class="headerlink" href="#introduction" title="Permalink to this headline">¶</a></h3>
112  <p>This tutorial will attempt to introduce users to both NumPy and the pygame
113  surfarray module. To beginners, the code that uses surfarray can be quite
114  intimidating. But actually there are only a few concepts to understand and
115  you will be up and running. Using the surfarray module, it becomes possible
116  to perform pixel level operations from straight python code. The performance
117  can become quite close to the level of doing the code in C.</p>
118  <p>You may just want to jump down to the <em>&quot;Examples&quot;</em> section to get an
119  idea of what is possible with this module, then start at the beginning here
120  to work your way up.</p>
121  <p>Now I won't try to fool you into thinking everything is very easy. To get
122  more advanced effects by modifying pixel values is very tricky. Just mastering
123  Numeric Python (SciPy's original array package was Numeric, the predecessor of NumPy)
124  takes a lot of learning. In this tutorial I'll be sticking with
125  the basics and using a lot of examples in an attempt to plant seeds of wisdom.
126  After finishing the tutorial you should have a basic handle on how the surfarray
127  works.</p>
128  </section>

147  <p>As you can see, the NumPy module gives us a new data type, the <em>array</em>.
148  This object holds an array of fixed size, and all values inside are of the same
149  type. The arrays can also be multidimensional, which is how we will use them
150  with images. There's a bit more to it than this, but it is enough to get us
151  started.</p>
152  <p>If you look at the last command above, you'll see that mathematical operations
153  on NumPy arrays apply to all values in the array. This is called &quot;element-wise
154  operations&quot;. These arrays can also be sliced like normal lists. The slicing
155  syntax is the same as used on standard python objects.
156  <em>(so study up if you need to :] )</em>.
157  Here are some more examples of working with arrays.</p>

199  two dimensional arrays.</p>
200  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">row1</span> <span class="o">=</span> <span class="p">(</span><span class="mi">1</span><span class="p">,</span><span class="mi">2</span><span class="p">,</span><span class="mi">3</span><span class="p">)</span>                         <span class="c1">#create a tuple of vals</span>

220  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">b</span>                                      <span class="c1">#show our array from above</span>

257  <h3>Import Surfarray<a class="headerlink" href="#import-surfarray" title="Permalink to this headline">¶</a></h3>
258  <p>In order to use the surfarray module we need to import it. Since both surfarray
259  and NumPy are optional components for pygame, it is nice to make sure they
260  import correctly before using them. In these examples I'm going to import
261  NumPy into a variable named <em>N</em>. This will let you know which functions
262  I'm using are from the NumPy package.
263  <em>(and is a lot shorter than typing NumPy before each function)</em></p>
264  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">try</span><span class="p">:</span>
265      <span class="kn">import</span> <span class="nn">numpy</span> <span class="k">as</span> <span class="nn">N</span>
266      <span class="kn">import</span> <span class="nn">pygame.surfarray</span> <span class="k">as</span> <span class="nn">surfarray</span>
267  <span class="k">except</span> <span class="ne">ImportError</span><span class="p">:</span>
268      <span class="k">raise</span> <span class="ne">ImportError</span><span class="p">,</span> <span class="s2">&quot;NumPy and Surfarray are required.&quot;</span>
269  </pre></div>
270  </div>

272  <section id="id1">
273  <h3>Surfarray Introduction<a class="headerlink" href="#id1" title="Permalink to this headline">¶</a></h3>
274  <p>There are two main types of functions in surfarray. One set of functions for
275  creating an array that is a copy of a surface pixel data. The other functions
276  create a referenced copy of the array pixel data, so that changes to the array
277  directly affect the original surface. There are other functions that allow you
278  to access any per-pixel alpha values as arrays along with a few other helpful
279  functions. We will look at these other functions later on.</p>
280  <p>When working with these surface arrays, there are two ways of representing the
281  pixel values. First, they can be represented as mapped integers. This type of
282  array is a simple 2D array with a single integer representing the surface's
283  mapped color value. This type of array is good for moving parts of an image
284  around. The other type of array uses three RGB values to represent each pixel
285  color. This type of array makes it extremely simple to do types of effects that
286  change the color of each pixel. This type of array is also a little trickier to
287  deal with, since it is essentially a 3D numeric array. Still, once you get your
288  mind into the right mode, it is not much harder than using the normal 2D arrays.</p>
289  <p>The NumPy module uses a machine's natural number types to represent the data
290  values, so a NumPy array can consist of integers that are 8-bits, 16-bits, and 32-bits.
291  <em>(the arrays can also use other types like floats and doubles, but for our image
292  manipulation we mainly need to worry about the integer types)</em>.
293  Because of this limitation of integer sizes, you must take a little extra care
294  that the type of arrays that reference pixel data can be properly mapped to a
295  proper type of data. The functions create these arrays from surfaces are:</p>
296  <dl class="py function definition">
297  <dt class="sig sig-object py title">

301  </dd></dl>

526  <section id="surface-locking">
527  <h3>Surface Locking<a class="headerlink" href="#surface-locking" title="Permalink to this headline">¶</a></h3>
528  <p>Like the rest of pygame, surfarray will lock any Surfaces it needs to
529  automatically when accessing pixel data. There is one extra thing to be aware
530  of though. When creating the <em>pixel</em> arrays, the original surface will
531  be locked during the lifetime of that pixel array. This is important to remember.
532  Be sure to <em>&quot;del&quot;</em> the pixel array or let it go out of scope
533  <em>(ie, when the function returns, etc)</em>.</p>
534  <p>Also be aware that you really don't want to be doing much <em>(if any)</em>
535  direct pixel access on hardware surfaces <em>(HWSURFACE)</em>. This is because
536  the actual surface data lives on the graphics card, and transferring pixel
537  changes over the PCI/AGP bus is not fast.</p>
538  </section>

550  </dd></dl>

563  <span class="sig-prename descclassname"><span class="pre">surfarray.</span></span><span class="sig-name descname"><span class="pre">array_colorkey</span></span><span class="sig-paren">(</span><em class="sig-param"><span class="n"><span class="pre">surface</span></span></em><span class="sig-paren">)</span></dt>
564  <dd><p>Creates a 2D array <em>(integer pixel values)</em> that is set to transparent
565  <em>(0)</em> wherever that pixel color matches the Surface colorkey.</p>
566  </dd></dl>
567  
568  </section>
569  <section id="other-surfarray-functions">
570  <h3>Other Surfarray Functions<a class="headerlink" href="#other-surfarray-functions" title="Permalink to this headline">¶</a></h3>
571  <p>There are only a few other functions available in surfarray. You can get a better
572  list with more documentation on the
573  <a class="reference internal" href="../ref/surfarray.html#module-pygame.surfarray" title="pygame.surfarray: pygame module for accessing surface pixel data using array interfaces"><code class="xref py py-mod docutils literal notranslate"><span class="pre">surfarray</span> <span class="pre">reference</span> <span class="pre">page</span></code></a>.
574  There is one very useful function though.</p>
575  <dl class="py function definition">
576  <dt class="sig sig-object py title">
577  <span class="sig-prename descclassname"><span class="pre">surfarray.</span></span><span class="sig-name descname"><span class="pre">blit_array</span></span><span class="sig-paren">(</span><em class="sig-param"><span class="n"><span class="pre">surface</span></span></em>, <em class="sig-param"><span class="n"><span class="pre">array</span></span></em><span class="sig-paren">)</span></dt>
578  <dd><p>This will transfer any type of 2D or 3D surface array onto a Surface
579  of the same dimensions.
580  This surfarray blit will generally be faster than assigning an array to a
581  referenced pixel array.
582  Still, it should not be as fast as normal Surface blitting,
583  since those are very optimized.</p>
584  </dd></dl>
585  
586  </section>
587  <section id="more-advanced-numpy">
588  <h3>More Advanced NumPy<a class="headerlink" href="#more-advanced-numpy" title="Permalink to this headline">¶</a></h3>
589  <p>There's a couple last things you should know about NumPy arrays. When dealing
590  with very large arrays, like the kind that are 640x480 big, there are some extra
591  things you should be careful about. Mainly, while using the operators like + and
592  * on the arrays makes them easy to use, it is also very expensive on big arrays.
593  These operators must make new temporary copies of the array, that are then
594  usually copied into another array. This can get very time consuming. Fortunately,
595  all the NumPy operators come with special functions that can perform the
596  operation <em>&quot;in place&quot;</em>. For example, you would want to replace
597  <code class="docutils literal notranslate"><span class="pre">screen[:]</span> <span class="pre">=</span> <span class="pre">screen</span> <span class="pre">+</span> <span class="pre">brightmap</span></code> with the much faster
598  <code class="docutils literal notranslate"><span class="pre">add(screen,</span> <span class="pre">brightmap,</span> <span class="pre">screen)</span></code>.
599  Anyway, you'll want to read up on the NumPy UFunc
600  documentation for more about this.
601  It is important when dealing with the arrays.</p>
602  <p>Another thing to be aware of when working with NumPy arrays is the datatype
603  of the array. Some of the arrays (especially the mapped pixel type) often return
604  arrays with an unsigned 8-bit value. These arrays will easily overflow if you are
605  not careful. NumPy will use the same coercion that you find in C programs, so
606  mixing an operation with 8-bit numbers and 32-bit numbers will give a result as
607  32-bit numbers. You can convert the datatype of an array, but definitely be
608  aware of what types of arrays you have, if NumPy gets in a situation where
609  precision would be ruined, it will raise an exception.</p>
610  <p>Lastly, be aware that when assigning values into the 3D arrays, they must be
611  between 0 and 255, or you will get some undefined truncating.</p>
612  </section>
613  <section id="graduation">
614  <h3>Graduation<a class="headerlink" href="#graduation" title="Permalink to this headline">¶</a></h3>
615  <p>Well there you have it. My quick primer on Numeric Python and surfarray.
616  Hopefully now you see what is possible, and even if you never use them for
617  yourself, you do not have to be afraid when you see code that does. Look into
618  the vgrade example for more numeric array action. There are also some <em>&quot;flame&quot;</em>
619  demos floating around that use surfarray to create a realtime fire effect.</p>
620  <p>Best of all, try some things on your own. Take it slow at first and build up,
621  I've seen some great things with surfarray already like radial gradients and
622  more. Good Luck.</p>
623  </section>
