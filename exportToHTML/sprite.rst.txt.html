6    .. module:: pygame.sprite
7       :synopsis: pygame module with basic game object classes
8    
9    | :sl:`pygame module with basic game object classes`
10   
11   This module contains several simple classes to be used within games. There is
12   the main Sprite class and several Group classes that contain Sprites. The use
13   of these classes is entirely optional when using pygame. The classes are fairly
14   lightweight and only provide a starting place for the code that is common to
15   most games.
16   
17   The Sprite class is intended to be used as a base class for the different types
18   of objects in the game. There is also a base Group class that simply stores
19   sprites. A game could create new types of Group classes that operate on
20   specially customized Sprite instances they contain.
21   
22   The basic Sprite class can draw the Sprites it contains to a Surface. The
23   ``Group.draw()`` method requires that each Sprite have a ``Surface.image``
24   attribute and a ``Surface.rect``. The ``Group.clear()`` method requires these
25   same attributes, and can be used to erase all the Sprites with background.
26   There are also more advanced Groups: ``pygame.sprite.RenderUpdates()`` and
27   ``pygame.sprite.OrderedUpdates()``.
28   
29   Lastly, this module contains several collision functions. These help find
30   sprites inside multiple groups that have intersecting bounding rectangles. To
31   find the collisions, the Sprites are required to have a ``Surface.rect``
32   attribute assigned.
33   
34   The groups are designed for high efficiency in removing and adding Sprites to
35   them. They also allow cheap testing to see if a Sprite already exists in a
36   Group. A given Sprite can exist in any number of groups. A game could use some
37   groups to control object rendering, and a completely separate set of groups to
38   control interaction or player movement. Instead of adding type attributes or
39   bools to a derived Sprite class, consider keeping the Sprites inside organized
40   Groups. This will allow for easier lookup later in the game.
41   
42   Sprites and Groups manage their relationships with the ``add()`` and
43   ``remove()`` methods. These methods can accept a single or multiple targets for
44   membership. The default initializers for these classes also takes a single or
45   list of targets for initial membership. It is safe to repeatedly add and remove
46   the same Sprite from a Group.
47   
48   While it is possible to design sprite and group classes that don't derive from
49   the Sprite and AbstractGroup classes below, it is strongly recommended that you
50   extend those when you add a Sprite or Group class.
51   
52   Sprites are not thread safe. So lock them yourself if using threads.
53   
54   .. class:: Sprite
55   
56      | :sl:`Simple base class for visible game objects.`
57      | :sg:`Sprite(*groups) -> Sprite`
58   
59      The base class for visible game objects. Derived classes will want to
60      override the ``Sprite.update()`` and assign a ``Sprite.image`` and
61      ``Sprite.rect`` attributes. The initializer can accept any number of Group
62      instances to be added to.
63   
64      When subclassing the Sprite, be sure to call the base initializer before
65      adding the Sprite to Groups. For example:
66   
67      .. code-block:: python
68      
69          class Block(pygame.sprite.Sprite):
70               
71              # Constructor. Pass in the color of the block, 
72              # and its x and y position
73              def __init__(self, color, width, height):
74                 # Call the parent class (Sprite) constructor
75                 pygame.sprite.Sprite.__init__(self) 
76           
77                 # Create an image of the block, and fill it with a color.
78                 # This could also be an image loaded from the disk.
79                 self.image = pygame.Surface([width, height])
80                 self.image.fill(color)
81           
82                 # Fetch the rectangle object that has the dimensions of the image
83                 # Update the position of this object by setting the values of rect.x and rect.y
84                 self.rect = self.image.get_rect()   
85         
86      .. method:: update
87   
88         | :sl:`method to control sprite behavior`
89         | :sg:`update(*args, **kwargs) -> None`
90   
91         The default implementation of this method does nothing; it's just a
92         convenient "hook" that you can override. This method is called by
93         ``Group.update()`` with whatever arguments you give it.
94   
95         There is no need to use this method if not using the convenience method
96         by the same name in the Group class.

199  .. class:: Group
200  
201     | :sl:`A container class to hold and manage multiple Sprite objects.`
202     | :sg:`Group(*sprites) -> Group`
203  
204     A simple container for Sprite objects. This class can be inherited to create
205     containers with more specific behaviors. The constructor takes any number of
206     Sprite arguments to add to the Group. The group supports the following
207     standard Python operations:
208  
209     ::
210  
211         in      test if a Sprite is contained
212         len     the number of Sprites contained
213         bool    test if any Sprites are contained
214         iter    iterate through all the Sprites
215  
216     The Sprites in the Group are ordered only on python 3.6 and higher.
217     Below python 3.6 drawing and iterating over the Sprites is in no particular order.

308     .. method:: clear
309  
310        | :sl:`draw a background over the Sprites`
311        | :sg:`clear(Surface_dest, background) -> None`
312  
313        Erases the Sprites used in the last ``Group.draw()`` call. The
314        destination Surface is cleared by filling the drawn Sprite positions with
315        the background.
316  
317        The background is usually a Surface image the same dimensions as the
318        destination Surface. However, it can also be a callback function that
319        takes two arguments; the destination Surface and an area to clear. The
320        background callback function will be called several times each clear.
321  
322        Here is an example callback that will clear the Sprites with solid red:
323  
324        ::
325  
326            def clear_callback(surf, rect):
327                color = 255, 0, 0
328                surf.fill(color, rect)

676  .. function:: spritecollide
677  
678     | :sl:`Find sprites in a group that intersect another sprite.`
679     | :sg:`spritecollide(sprite, group, dokill, collided = None) -> Sprite_list`
680  
681     Return a list containing all Sprites in a Group that intersect with another
682     Sprite. Intersection is determined by comparing the ``Sprite.rect``
683     attribute of each Sprite.
684  
685     The dokill argument is a bool. If set to True, all Sprites that collide will
686     be removed from the Group.
687  
688     The collided argument is a callback function used to calculate if two
689     sprites are colliding. it should take two sprites as values, and return a
690     bool value indicating if they are colliding. If collided is not passed, all
691     sprites must have a "rect" value, which is a rectangle of the sprite area,
692     which will be used to calculate the collision.
693  
694     collided callables:
695  
696     ::
697  
698         collide_rect, collide_rect_ratio, collide_circle,
699         collide_circle_ratio, collide_mask
700  
701     Example:
702     
703     .. code-block:: python
704  
705      # See if the Sprite block has collided with anything in the Group block_list
706      # The True flag will remove the sprite in block_list
707      blocks_hit_list = pygame.sprite.spritecollide(player, block_list, True)  
708       
709      # Check the list of colliding sprites, and add one to the score for each one
710      for block in blocks_hit_list:
711          score +=1

791  .. function:: collide_mask
792  
793     | :sl:`Collision detection between two sprites, using masks.`
794     | :sg:`collide_mask(sprite1, sprite2) -> (int, int)`
795     | :sg:`collide_mask(sprite1, sprite2) -> None`
796  
797     Tests for collision between two sprites, by testing if their bitmasks
798     overlap (uses :func:`pygame.mask.Mask.overlap`). If the sprites have a
799     ``mask`` attribute, it is used as the mask, otherwise a mask is created from
800     the sprite's ``image`` (uses :func:`pygame.mask.from_surface`). Sprites must
801     have a ``rect`` attribute; the ``mask`` attribute is optional.
802  
803     The first point of collision between the masks is returned. The collision
804     point is offset from ``sprite1``'s mask's topleft corner (which is always
805     (0, 0)). The collision point is a position within the mask and is not
806     related to the actual screen position of ``sprite1``.
807  
808     This function is intended to be passed as a ``collided`` callback function
809     to the group collide functions (see :meth:`spritecollide`,
810     :meth:`groupcollide`, :meth:`spritecollideany`).
811  
812     .. note::
813        To increase performance, create and set a ``mask`` attibute for all
814        sprites that will use this function to check for collisions. Otherwise,
815        each time this function is called it will create new masks.
816  
817     .. note::
818        A new mask needs to be recreated each time a sprite's image is changed
819        (e.g. if a new image is used or the existing image is rotated).
820  
821     ::
822  
823        # Example of mask creation for a sprite.
824        sprite.mask = pygame.mask.from_surface(sprite.image)
825  
826     :returns: first point of collision between the masks or ``None`` if no
827        collision
828     :rtype: tuple(int, int) or NoneType
829  
830     .. versionadded:: 1.8.0
831

834  .. function:: groupcollide
835  
836     | :sl:`Find all sprites that collide between two groups.`
837     | :sg:`groupcollide(group1, group2, dokill1, dokill2, collided = None) -> Sprite_dict`
838  
839     This will find collisions between all the Sprites in two groups.
840     Collision is determined by comparing the ``Sprite.rect`` attribute of
841     each Sprite or by using the collided function if it is not None.
842  
843     Every Sprite inside group1 is added to the return dictionary. The value for
844     each item is the list of Sprites in group2 that intersect.
845  
846     If either dokill argument is True, the colliding Sprites will be removed
847     from their respective Group.
848  
849     The collided argument is a callback function used to calculate if two sprites are
850     colliding. It should take two sprites as values and return a bool value
851     indicating if they are colliding. If collided is not passed, then all
852     sprites must have a "rect" value, which is a rectangle of the sprite area,
853     which will be used to calculate the collision.
854

857  .. function:: spritecollideany
858  
859     | :sl:`Simple test if a sprite intersects anything in a group.`

861     | :sg:`spritecollideany(sprite, group, collided = None) -> None` No collision

863     If the sprite collides with any single sprite in the group, a single

872     sprites must have a "rect" value, which is a rectangle of the sprite area,
