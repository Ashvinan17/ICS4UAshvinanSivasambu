6    .. module:: pygame.sndarray
7       :synopsis: pygame module for accessing sound sample data
8    
9    | :sl:`pygame module for accessing sound sample data`
10   
11   Functions to convert between NumPy arrays and Sound objects. This
12   module will only be functional when pygame can use the external NumPy
13   package. If NumPy can't be imported, ``surfarray`` becomes a ``MissingModule``
14   object.
15   
16   Sound data is made of thousands of samples per second, and each sample is the
17   amplitude of the wave at a particular moment in time. For example, in 22-kHz
18   format, element number 5 of the array is the amplitude of the wave after
19   5/22000 seconds.
20   
21   The arrays are indexed by the ``X`` axis first, followed by the ``Y`` axis.
22   Each sample is an 8-bit or 16-bit integer, depending on the data format. A
23   stereo sound file has two values per sample, while a mono sound file only has
24   one.

26   .. function:: array
27   
28      | :sl:`copy Sound samples into an array`
29      | :sg:`array(Sound) -> array`
30   
31      Creates a new array for the sound data and copies the samples. The array
32      will always be in the format returned from ``pygame.mixer.get_init()``.

36   .. function:: samples
37   
38      | :sl:`reference Sound samples into an array`
39      | :sg:`samples(Sound) -> array`
40   
41      Creates a new array that directly references the samples in a Sound object.
42      Modifying the array will change the Sound. The array will always be in the
43      format returned from ``pygame.mixer.get_init()``.

47   .. function:: make_sound
48   
49      | :sl:`convert an array into a Sound object`
50      | :sg:`make_sound(array) -> Sound`
51   
52      Create a new playable Sound object from an array. The mixer module must be
53      initialized and the array format must be similar to the mixer audio format.

57   .. function:: use_arraytype
58   
59      | :sl:`Sets the array system to be used for sound arrays`
60      | :sg:`use_arraytype (arraytype) -> None`
61   
62      DEPRECATED: Uses the requested array type for the module functions. The
63      only supported arraytype is ``'numpy'``. Other values will raise ValueError.
64      Using this function will raise a ``DeprecationWarning``.

69      | :sl:`Gets the currently active array type.`
70      | :sg:`get_arraytype () -> str`
71   
72      DEPRECATED: Returns the currently active array type. This will be a value of the
73      ``get_arraytypes()`` tuple and indicates which type of array module is used
74      for the array creation. Using this function will raise a ``DeprecationWarning``.
