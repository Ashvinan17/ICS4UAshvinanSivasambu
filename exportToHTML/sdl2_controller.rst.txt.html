6    .. module:: pygame._sdl2.controller
7       :synopsis: pygame module to work with controllers
8    
9    | :sl:`Pygame module to work with controllers.`
10   
11   This module offers control over common controller types like the dualshock 4 or
12   the xbox 360 controllers: They have two analog sticks, two triggers, two shoulder buttons,
13   a dpad, 4 buttons on the side, 2 (or 3) buttons in the middle.
14   
15   Pygame uses xbox controllers naming conventions (like a, b, x, y for buttons) but
16   they always refer to the same buttons. For example ``CONTROLLER_BUTTON_X`` is
17   always the leftmost button of the 4 buttons on the right.
18   
19   Controllers can generate the following events::
20   
21      CONTROLLERAXISMOTION, CONTROLLERBUTTONDOWN, CONTROLLERBUTTONUP,
22      CONTROLLERDEVICEREMAPPED, CONTROLLERDEVICEADDED, CONTROLLERDEVICEREMOVED
23   
24   Additionally if pygame is built with SDL 2.0.14 or higher the following events can also be generated
25   (to get the version of sdl pygame is built with use :meth:`pygame.version.SDL`)::
26   
27      CONTROLLERTOUCHPADDOWN, CONTROLLERTOUCHPADMOTION, CONTROLLERTOUCHPADUP
28   
29   These events can be enabled/disabled by :meth:`pygame._sdl2.controller.set_eventstate`
30   Note that controllers can generate joystick events as well. This function only toggles
31   events related to controllers.
32   
33   .. note::
34      See the :mod:`pygame.joystick` for a more versatile but more advanced api.
35

227     .. method:: set_mapping
228  
229        | :sl:`Assign a mapping to the controller`
230        | :sg:`set_mapping(mapping) -> int`
231  
232        Rebind buttons, axes, triggers and dpads. The mapping should be a 
233        dict containing all buttons, hats and axes. The easiest way to get this
234        is to use the dict returned by :meth:`Controller.get_mapping`. To edit
235        this mapping assign a value to the original button. The value of the
236        dictionary must be a button, hat or axis represented in the following way:
237  
238        * For a button use: bX where X is the index of the button.
239        * For a hat use: hX.Y where X is the index and the Y is the direction (up: 1, right: 2, down: 3, left: 4).
240        * For an axis use: aX where x is the index of the axis.
241  
242        An example of mapping::
243  
244           mapping = controller.get_mapping() # Get current mapping
245           mapping["a"] = "b3" # Remap button a to y
246           mapping["y"] = "b0" # Remap button y to a
247           controller.set_mapping(mapping) # Set the mapping
248  
249  
250        The function will return 1 if a new mapping is added or 0 if an existing one is updated.
251  
252        .. versionchanged:: 2.0.2: Renamed from ``add_mapping`` to ``set_mapping``
253        .. versionchanged:: 2.0.2: Argument type changed from ``str`` to ``dict``
254

257     .. method:: rumble
258  
259        | :sl:`Start a rumbling effect`
260        | :sg:`rumble(low_frequency, high_frequency, duration) -> bool`
261  
262        Start a rumble effect on the controller, with the specified strength ranging
263        from 0 to 1. Duration is length of the effect, in ms. Setting the duration
264        to 0 will play the effect until another one overwrites it or
265        :meth:`Controller.stop_rumble` is called. If an effect is already
266        playing, then it will be overwritten.
267  
268        Returns True if the rumble was played successfully or False if the
269        controller does not support it or :meth:`pygame.version.SDL` is below 2.0.9.
270  
271        .. versionadded:: 2.0.2
