170  <p>The pygame package represents the top-level package for others to use. Pygame
171  itself is broken into many submodules, but this does not affect programs that
172  use pygame.</p>
173  <p>As a convenience, most of the top-level variables in pygame have been placed
174  inside a module named <a class="tooltip reference internal" href="locals.html#module-pygame.locals" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.locals</span></code><span class="tooltip-content">pygame constants</span></a>. This is meant to be used with
175  <code class="docutils literal notranslate"><span class="pre">from</span> <span class="pre">pygame.locals</span> <span class="pre">import</span> <span class="pre">*</span></code>, in addition to <code class="docutils literal notranslate"><span class="pre">import</span> <span class="pre">pygame</span></code>.</p>
176  <p>When you <code class="docutils literal notranslate"><span class="pre">import</span> <span class="pre">pygame</span></code> all available pygame submodules are automatically
177  imported. Be aware that some of the pygame modules are considered <em>optional</em>,
178  and may not be available. In that case, pygame will provide a placeholder
179  object instead of the module, which can be used to test for availability.</p>
180  <dl class="py function definition">
181  <dt class="sig sig-object py title" id="pygame.init">
182  <span class="sig-prename descclassname"><span class="pre">pygame.</span></span><span class="sig-name descname"><span class="pre">init</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.init" title="Permalink to this definition">¶</a></dt>
183  <dd><div class="line-block">
184  <div class="line"><span class="summaryline">initialize all imported pygame modules</span></div>
185  <div class="line"><span class="signature">init() -&gt; (numpass, numfail)</span></div>
186  </div>
187  <p>Initialize all imported pygame modules. No exceptions will be raised if a
188  module fails, but the total number if successful and failed inits will be
189  returned as a tuple. You can always initialize individual modules manually,
190  but <a class="tooltip reference internal" href="#pygame.init" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.init()</span></code><span class="tooltip-content">initialize all imported pygame modules</span></a> is a convenient way to get everything started. The
191  <code class="docutils literal notranslate"><span class="pre">init()</span></code> functions for individual modules will raise exceptions when they
192  fail.</p>
193  <p>You may want to initialize the different modules separately to speed up your
194  program or to not use modules your game does not require.</p>
195  <p>It is safe to call this <code class="docutils literal notranslate"><span class="pre">init()</span></code> more than once as repeated calls will have
196  no effect. This is true even if you have <code class="docutils literal notranslate"><span class="pre">pygame.quit()</span></code> all the modules.</p>
197  </dd></dl>

200  <dt class="sig sig-object py title" id="pygame.quit">
201  <span class="sig-prename descclassname"><span class="pre">pygame.</span></span><span class="sig-name descname"><span class="pre">quit</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.quit" title="Permalink to this definition">¶</a></dt>
202  <dd><div class="line-block">
203  <div class="line"><span class="summaryline">uninitialize all pygame modules</span></div>
204  <div class="line"><span class="signature">quit() -&gt; None</span></div>
205  </div>
206  <p>Uninitialize all pygame modules that have previously been initialized. When
207  the Python interpreter shuts down, this method is called regardless, so your
208  program should not need it, except when it wants to terminate its pygame
209  resources and continue. It is safe to call this function more than once as
210  repeated calls have no effect.</p>
211  <div class="admonition note">
212  <p class="admonition-title">Note</p>
213  <p>Calling <a class="tooltip reference internal" href="#pygame.quit" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.quit()</span></code><span class="tooltip-content">uninitialize all pygame modules</span></a> will not exit your program. Consider letting
214  your program end in the same way a normal Python program will end.</p>
215  </div>

312  <span class="sig-prename descclassname"><span class="pre">pygame.</span></span><span class="sig-name descname"><span class="pre">encode_string</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.encode_string" title="Permalink to this definition">¶</a></dt>
313  <dd><div class="line-block">
314  <div class="line"><span class="summaryline">Encode a Unicode or bytes object</span></div>
315  <div class="line"><span class="signature">encode_string([obj [, encoding [, errors [, etype]]]]) -&gt; bytes or None</span></div>
316  </div>
317  <p>obj: If Unicode, encode; if bytes, return unaltered; if anything else,
318  return <code class="docutils literal notranslate"><span class="pre">None</span></code>; if not given, raise <code class="docutils literal notranslate"><span class="pre">SyntaxError</span></code>.</p>
319  <p>encoding (string): If present, encoding to use. The default is
320  <code class="docutils literal notranslate"><span class="pre">'unicode_escape'</span></code>.</p>
321  <p>errors (string): If given, how to handle unencodable characters. The default
322  is <code class="docutils literal notranslate"><span class="pre">'backslashreplace'</span></code>.</p>
323  <p>etype (exception type): If given, the exception type to raise for an
324  encoding error. The default is <code class="docutils literal notranslate"><span class="pre">UnicodeEncodeError</span></code>, as returned by
325  <code class="docutils literal notranslate"><span class="pre">PyUnicode_AsEncodedString()</span></code>. For the default encoding and errors values
326  there should be no encoding errors.</p>
327  <p>This function is used in encoding file paths. Keyword arguments are
328  supported.</p>
329  <div class="versionadded">
330  <p><span class="versionmodified added">New in pygame 1.9.2: </span>(primarily for use in unit tests)</p>
331  </div>
332  </dd></dl>

482  <p id="environment-variables"><strong>Setting Environment Variables</strong></p>
483  <p>Some aspects of pygame's behaviour can be controlled by setting environment variables, they cover a wide
484  range of the library's functionality. Some of the variables are from pygame itself, while others come from
485  the underlying C SDL library that pygame uses.</p>
486  <p>In python, environment variables are usually set in code like this:</p>
487  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="kn">import</span> <span class="nn">os</span>
488  <span class="n">os</span><span class="o">.</span><span class="n">environ</span><span class="p">[</span><span class="s1">&#39;NAME_OF_ENVIRONMENT_VARIABLE&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="s1">&#39;value_to_set&#39;</span>
489  </pre></div>
490  </div>
491  <p>Or to preserve users ability to override the variable:</p>
492  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="kn">import</span> <span class="nn">os</span>
493  <span class="n">os</span><span class="o">.</span><span class="n">environ</span><span class="p">[</span><span class="s1">&#39;ENV_VAR&#39;</span><span class="p">]</span> <span class="o">=</span> <span class="n">os</span><span class="o">.</span><span class="n">environ</span><span class="o">.</span><span class="n">get</span><span class="p">(</span><span class="s1">&#39;ENV_VAR&#39;</span><span class="p">,</span> <span class="s1">&#39;value&#39;</span><span class="p">)</span>
494  </pre></div>
495  </div>
496  <p>If the variable is more useful for users of an app to set than the developer then they can set it like this:</p>
497  <p><strong>Windows</strong>:</p>
498  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="nb">set</span> <span class="n">NAME_OF_ENVIRONMENT_VARIABLE</span><span class="o">=</span><span class="n">value_to_set</span>
499  <span class="n">python</span> <span class="n">my_application</span><span class="o">.</span><span class="n">py</span>
500  </pre></div>
501  </div>
502  <p><strong>Linux/Mac</strong>:</p>
503  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">ENV_VAR</span><span class="o">=</span><span class="n">value</span> <span class="n">python</span> <span class="n">my_application</span><span class="o">.</span><span class="n">py</span>
504  </pre></div>
505  </div>
506  <p>For some variables they need to be set before initialising pygame, some must be set before even importing pygame,
507  and others can simply be set right before the area of code they control is run.</p>
508  <p>Below is a list of environment variables, their settable values, and a brief description of what they do.</p>

647  <span class="n">Set</span> <span class="n">to</span> <span class="s2">&quot;0&quot;</span> <span class="n">to</span> <span class="n">re</span><span class="o">-</span><span class="n">enable</span> <span class="n">the</span> <span class="n">compositor</span><span class="o">.</span>
648  </pre></div>
649  </div>
650  <p>By default SDL tries to disable the X11 compositor for all pygame
651  apps. This is usually a good thing as it's faster, however if you
652  have an app which <em>doesn't</em> update every frame and are using linux
653  you may want to disable this bypass. The bypass has reported problems
654  on KDE linux. This variable is only used on x11/linux platforms.</p>
