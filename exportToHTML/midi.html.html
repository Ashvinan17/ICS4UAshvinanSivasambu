190  <div class="versionadded">
191  <p><span class="versionmodified added">New in pygame 1.9.0.</span></p>
192  </div>
193  <p>The midi module can send output to midi devices and get input from midi
194  devices. It can also list midi devices on the system.</p>
195  <p>The midi module supports real and virtual midi devices.</p>
196  <p>It uses the portmidi library. Is portable to which ever platforms portmidi
197  supports (currently Windows, Mac OS X, and Linux).</p>
198  <p>This uses pyportmidi for now, but may use its own bindings at some point in the
199  future. The pyportmidi bindings are included with pygame.</p>
200  <div class="line-block">
201  <div class="line"><br /></div>
202  </div>
203  <div class="versionadded">
204  <p><span class="versionmodified added">New in pygame 2.0.0.</span></p>
205  </div>
206  <p>These are pygame events (<a class="tooltip reference internal" href="event.html#module-pygame.event" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.event</span></code><span class="tooltip-content">pygame module for interacting with events and queues</span></a>) reserved for midi use. The
207  <code class="docutils literal notranslate"><span class="pre">MIDIIN</span></code> event is used by <a class="tooltip reference internal" href="#pygame.midi.midis2events" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.midi.midis2events()</span></code><span class="tooltip-content">converts midi events to pygame events</span></a> when converting
208  midi events to pygame events.</p>
209  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">MIDIIN</span>
210  <span class="n">MIDIOUT</span>
211  </pre></div>
212  </div>

425  <p>The <code class="docutils literal notranslate"><span class="pre">buffer_size</span></code> specifies the number of output events to be buffered
426  waiting for output. In some cases (see below) PortMidi does not buffer
427  output at all and merely passes data to a lower-level API, in which case
428  buffersize is ignored.</p>
429  <p><code class="docutils literal notranslate"><span class="pre">latency</span></code> is the delay in milliseconds applied to timestamps to determine
430  when the output should actually occur. If <code class="docutils literal notranslate"><span class="pre">latency</span></code> is &lt;&lt;0, 0 is assumed.</p>
431  <p>If <code class="docutils literal notranslate"><span class="pre">latency</span></code> is zero, timestamps are ignored and all output is delivered
432  immediately. If <code class="docutils literal notranslate"><span class="pre">latency</span></code> is greater than zero, output is delayed until the
433  message timestamp plus the <code class="docutils literal notranslate"><span class="pre">latency</span></code>. In some cases, PortMidi can obtain
434  better timing than your application by passing timestamps along to the
435  device driver or hardware. Latency may also help you to synchronize midi
436  data to audio data by matching midi latency to the audio buffer latency.</p>
437  <div class="admonition note">
438  <p class="admonition-title">Note</p>
439  <p>Time is measured relative to the time source indicated by time_proc.
440  Timestamps are absolute, not relative delays or offsets.</p>

618  <dt class="sig sig-object py title" id="pygame.midi.get_default_input_id">
619  <span class="sig-prename descclassname"><span class="pre">pygame.midi.</span></span><span class="sig-name descname"><span class="pre">get_default_input_id</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.midi.get_default_input_id" title="Permalink to this definition">¶</a></dt>
620  <dd><div class="line-block">
621  <div class="line"><span class="summaryline">gets default input device number</span></div>
622  <div class="line"><span class="signature">get_default_input_id() -&gt; default_id</span></div>
623  </div>
624  <p>The following describes the usage details for this function and the
625  <a class="reference internal" href="#pygame.midi.get_default_output_id" title="pygame.midi.get_default_output_id"><code class="xref py py-func docutils literal notranslate"><span class="pre">get_default_output_id()</span></code></a> function.</p>
626  <p>Return the default device ID or <code class="docutils literal notranslate"><span class="pre">-1</span></code> if there are no devices. The result
627  can be passed to the <a class="reference internal" href="#pygame.midi.Input" title="pygame.midi.Input"><code class="xref py py-class docutils literal notranslate"><span class="pre">Input</span></code></a>/<a class="reference internal" href="#pygame.midi.Output" title="pygame.midi.Output"><code class="xref py py-class docutils literal notranslate"><span class="pre">Output</span></code></a> class.</p>
628  <p>On a PC the user can specify a default device by setting an environment
629  variable. To use device #1, for example:</p>
630  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="nb">set</span> <span class="n">PM_RECOMMENDED_INPUT_DEVICE</span><span class="o">=</span><span class="mi">1</span>
631  <span class="ow">or</span>
632  <span class="nb">set</span> <span class="n">PM_RECOMMENDED_OUTPUT_DEVICE</span><span class="o">=</span><span class="mi">1</span>
633  </pre></div>
634  </div>
635  <p>The user should first determine the available device ID by using the
636  supplied application &quot;testin&quot; or &quot;testout&quot;.</p>
637  <p>In general, the registry is a better place for this kind of info. With
638  USB devices that can come and go, using integers is not very reliable
639  for device identification. Under Windows, if <code class="docutils literal notranslate"><span class="pre">PM_RECOMMENDED_INPUT_DEVICE</span></code>
640  (or <code class="docutils literal notranslate"><span class="pre">PM_RECOMMENDED_OUTPUT_DEVICE</span></code>) is NOT found in the environment,
641  then the default device is obtained by looking for a string in the registry
642  under:</p>
643  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">HKEY_LOCAL_MACHINE</span><span class="o">/</span><span class="n">SOFTWARE</span><span class="o">/</span><span class="n">PortMidi</span><span class="o">/</span><span class="n">Recommended_Input_Device</span>
644  <span class="ow">or</span>
645  <span class="n">HKEY_LOCAL_MACHINE</span><span class="o">/</span><span class="n">SOFTWARE</span><span class="o">/</span><span class="n">PortMidi</span><span class="o">/</span><span class="n">Recommended_Output_Device</span>
646  </pre></div>
647  </div>
648  <p>The number of the first device with a substring that matches the
649  string exactly is returned. For example, if the string in the registry is
650  &quot;USB&quot; and device 1 is named &quot;In USB MidiSport 1x1&quot;, then that will be
651  the default input because it contains the string &quot;USB&quot;.</p>
652  <p>In addition to the name, <a class="reference internal" href="#pygame.midi.get_device_info" title="pygame.midi.get_device_info"><code class="xref py py-func docutils literal notranslate"><span class="pre">get_device_info()</span></code></a> returns &quot;interf&quot;, which is
653  the interface name. The &quot;interface&quot; is the underlying software system or
654  API used by PortMidi to access devices. Supported interfaces:</p>
655  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">MMSystem</span>   <span class="c1"># the only Win32 interface currently supported</span>
656  <span class="n">ALSA</span>       <span class="c1"># the only Linux interface currently supported</span>
657  <span class="n">CoreMIDI</span>   <span class="c1"># the only Mac OS X interface currently supported</span>
658  <span class="c1"># DirectX - not implemented</span>
659  <span class="c1"># OSS     - not implemented</span>
660  </pre></div>
661  </div>
662  <p>To specify both the interface and the device name in the registry, separate
663  the two with a comma and a space. The string before the comma must be a
664  substring of the &quot;interf&quot; string and the string after the space must be a
665  substring of the &quot;name&quot; name string in order to match the device. e.g.:</p>
666  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">MMSystem</span><span class="p">,</span> <span class="n">In</span> <span class="n">USB</span> <span class="n">MidiSport</span> <span class="mi">1</span><span class="n">x1</span>
667  </pre></div>
668  </div>
669  <div class="admonition note">
670  <p class="admonition-title">Note</p>
671  <p>In the current release, the default is simply the first device (the
672  input or output device with the lowest PmDeviceID).</p>
673  </div>
