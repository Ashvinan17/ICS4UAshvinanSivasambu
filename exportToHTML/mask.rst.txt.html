6    .. module:: pygame.mask
7       :synopsis: pygame module for image masks.
8    
9    | :sl:`pygame module for image masks.`
10   
11   Useful for fast pixel perfect collision detection. A mask uses 1 bit per-pixel
12   to store which parts collide.

19                       support :class:`pygame.math.Vector2` arguments.

22   .. function:: from_surface
23   
24      | :sl:`Creates a Mask from the given surface`
25      | :sg:`from_surface(surface) -> Mask`
26      | :sg:`from_surface(surface, threshold=127) -> Mask`
27   
28      Creates a :class:`Mask` object from the given surface by setting all the
29      opaque pixels and not setting the transparent pixels.
30   
31      If the surface uses a color-key, then it is used to decide which bits in
32      the resulting mask are set. All the pixels that are **not** equal to the
33      color-key are **set** and the pixels equal to the color-key are not set.
34   
35      If a color-key is not used, then the alpha value of each pixel is used to
36      decide which bits in the resulting mask are set. All the pixels that have an
37      alpha value **greater than** the ``threshold`` parameter are **set** and the
38      pixels with an alpha value less than or equal to the ``threshold`` are
39      not set.
40   
41      :param Surface surface: the surface to create the mask from
42      :param int threshold: (optional) the alpha threshold (default is 127) to
43         compare with each surface pixel's alpha value, if the ``surface`` is
44         color-keyed this parameter is ignored
45   
46      :returns: a newly created :class:`Mask` object from the given surface
47      :rtype: Mask
48   
49      .. note::
50         This function is used to create the masks for
51         :func:`pygame.sprite.collide_mask`.

55   .. function:: from_threshold
56   
57      | :sl:`Creates a mask by thresholding Surfaces`
58      | :sg:`from_threshold(surface, color) -> Mask`
59      | :sg:`from_threshold(surface, color, threshold=(0, 0, 0, 255), othersurface=None, palette_colors=1) -> Mask`
60   
61      This is a more featureful method of getting a :class:`Mask` from a surface.
62   
63      If the optional ``othersurface`` is not used, all the pixels **within** the
64      ``threshold`` of the ``color`` parameter are **set** in the resulting mask.
65   
66      If the optional ``othersurface`` is used, every pixel in the first surface
67      that is **within** the ``threshold`` of the corresponding pixel in
68      ``othersurface`` is **set** in the resulting mask.
69   
70      :param Surface surface: the surface to create the mask from
71      :param color: color used to check if the surface's pixels are within the
72         given ``threshold`` range, this parameter is ignored if the optional
73         ``othersurface`` parameter is supplied
74      :type color: Color or int or tuple(int, int, int, [int]) or list[int, int, int, [int]]
75      :param threshold: (optional) the threshold range used to check the difference
76         between two colors (default is ``(0, 0, 0, 255)``)
77      :type threshold: Color or int or tuple(int, int, int, [int]) or list[int, int, int, [int]]
78      :param Surface othersurface: (optional) used to check whether the pixels of
79         the first surface are within the given ``threshold`` range of the pixels
80         from this surface (default is ``None``)
81      :param int palette_colors: (optional) indicates whether to use the palette
82         colors or not, a nonzero value causes the palette colors to be used and a
83         0 causes them not to be used (default is 1)
84   
85      :returns: a newly created :class:`Mask` object from the given surface
86      :rtype: Mask

90   .. class:: Mask
91   
92      | :sl:`pygame object for representing 2D bitmasks`
93      | :sg:`Mask(size=(width, height)) -> Mask`
94      | :sg:`Mask(size=(width, height), fill=False) -> Mask`
95   
96      A ``Mask`` object is used to represent a 2D bitmask. Each bit in
97      the mask represents a pixel. 1 is used to indicate a set bit and 0 is used
98      to indicate an unset bit. Set bits in a mask can be used to detect collisions
99      with other masks and their set bits.
100  
101     A filled mask has all of its bits set to 1, conversely an
102     unfilled/cleared/empty mask has all of its bits set to 0. Masks can be
103     created unfilled (default) or filled by using the ``fill`` parameter. Masks
104     can also be cleared or filled using the :func:`pygame.mask.Mask.clear()` and
105     :func:`pygame.mask.Mask.fill()` methods respectively.
106  
107     A mask's coordinates start in the top left corner at ``(0, 0)`` just like
108     :mod:`pygame.Surface`. Individual bits can be accessed using the
109     :func:`pygame.mask.Mask.get_at()` and :func:`pygame.mask.Mask.set_at()`
110     methods.
111  
112     .. _mask-offset-label:
113  
114     The methods :meth:`overlap`, :meth:`overlap_area`, :meth:`overlap_mask`,
115     :meth:`draw`, :meth:`erase`, and :meth:`convolve` use an offset parameter
116     to indicate the offset of another mask's top left corner from the calling
117     mask's top left corner. The calling mask's top left corner is considered to
118     be the origin ``(0, 0)``. Offsets are a sequence of two values
119     ``(x_offset, y_offset)``. Positive and negative offset values are supported.
120  
121     ::
122  
123                   0 to x (x_offset)
124                   :    :
125           0 ..... +----:---------+
126           to      |    :         |
127           y .......... +-----------+
128        (y_offset) |    | othermask |
129                   |    +-----------+
130                   | calling_mask |
131                   +--------------+
132  
133     :param size: the dimensions of the mask (width and height)
134     :param bool fill: (optional) create an unfilled mask (default: ``False``) or
135        filled mask (``True``)
136  
137     :returns: a newly created :class:`Mask` object
138     :rtype: Mask
139

243     .. method:: overlap
244  
245        | :sl:`Returns the point of intersection`
246        | :sg:`overlap(other, offset) -> (x, y)`
247        | :sg:`overlap(other, offset) -> None`
248  
249        Returns the first point of intersection encountered between this mask and
250        ``other``. A point of intersection is 2 overlapping set bits.
251  
252        The current algorithm searches the overlapping area in
253        ``sizeof(unsigned long int) * CHAR_BIT`` bit wide column blocks (the value
254        of ``sizeof(unsigned long int) * CHAR_BIT`` is platform dependent, for
255        clarity it will be referred to as ``W``). Starting at the top left corner
256        it checks bits 0 to ``W - 1`` of the first row (``(0, 0)`` to
257        ``(W - 1, 0)``) then continues to the next row (``(0, 1)`` to
258        ``(W - 1, 1)``). Once this entire column block is checked, it continues to
259        the next one (``W`` to ``2 * W - 1``). This is repeated until it finds a
260        point of intersection or the entire overlapping area is checked.
261  
262        :param Mask other: the other mask to overlap with this mask
263        :param offset: the offset of ``other`` from this mask, for more
264           details refer to the :ref:`Mask offset notes <mask-offset-label>`
265  
266        :returns: point of intersection or ``None`` if no intersection
267        :rtype: tuple(int, int) or NoneType

271     .. method:: overlap_area
272  
273        | :sl:`Returns the number of overlapping set bits`
274        | :sg:`overlap_area(other, offset) -> numbits`
275  
276        Returns the number of overlapping set bits between between this mask and
277        ``other``.
278  
279        This can be useful for collision detection. An approximate collision
280        normal can be found by calculating the gradient of the overlapping area
281        through the finite difference.
282  
283        ::
284  
285           dx = mask.overlap_area(other, (x + 1, y)) - mask.overlap_area(other, (x - 1, y))
286           dy = mask.overlap_area(other, (x, y + 1)) - mask.overlap_area(other, (x, y - 1))
287  
288        :param Mask other: the other mask to overlap with this mask
289        :param offset: the offset of ``other`` from this mask, for more
290           details refer to the :ref:`Mask offset notes <mask-offset-label>`
291  
292        :returns: the number of overlapping set bits
293        :rtype: int

495     .. method:: connected_component
496  
497        | :sl:`Returns a mask containing a connected component`
498        | :sg:`connected_component() -> Mask`
499        | :sg:`connected_component(pos) -> Mask`
500  
501        A connected component is a group (1 or more) of connected set bits
502        (orthogonally and diagonally). The SAUF algorithm, which checks 8 point
503        connectivity, is used to find a connected component in the mask.
504  
505        By default this method will return a :class:`Mask` containing the largest
506        connected component in the mask. Optionally, a bit coordinate can be
507        specified and the connected component containing it will be returned. If
508        the bit at the given location is not set, the returned :class:`Mask` will
509        be empty (no bits set).
510  
511        :param pos: (optional) selects the connected component that contains the
512           bit at this position
513  
514        :returns: a :class:`Mask` object (same size as this mask) with the largest
515           connected component from this mask, if this mask has no bits set then
516           an empty mask will be returned
517  
518           If the ``pos`` parameter is provided then the mask returned will have
519           the connected component that contains this position. An empty mask will
520           be returned if the ``pos`` parameter selects an unset bit.
521        :rtype: Mask
522  
523        :raises IndexError: if the optional ``pos`` parameter is outside of the
524           mask's bounds

569     .. method:: to_surface
570  
571        | :sl:`Returns a surface with the mask drawn on it`
572        | :sg:`to_surface() -> Surface`
573        | :sg:`to_surface(surface=None, setsurface=None, unsetsurface=None, setcolor=(255, 255, 255, 255), unsetcolor=(0, 0, 0, 255), dest=(0, 0)) -> Surface`
574  
575        Draws this mask on the given surface. Set bits (bits set to 1) and unset
576        bits (bits set to 0) can be drawn onto a surface.
577  
578        :param surface: (optional) Surface to draw mask onto, if no surface is
579           provided one will be created (default is ``None``, which will cause a
580           surface with the parameters
581           ``Surface(size=mask.get_size(), flags=SRCALPHA, depth=32)`` to be
582           created, drawn on, and returned)
583        :type surface: Surface or None
584        :param setsurface: (optional) use this surface's color values to draw
585           set bits (default is ``None``), if this surface is smaller than the
586           mask any bits outside its bounds will use the ``setcolor`` value
587        :type setsurface: Surface or None
588        :param unsetsurface: (optional) use this surface's color values to draw
589           unset bits (default is ``None``), if this surface is smaller than the
590           mask any bits outside its bounds will use the ``unsetcolor`` value
591        :type unsetsurface: Surface or None
592        :param setcolor: (optional) color to draw set bits (default is
593           ``(255, 255, 255, 255)``, white), use ``None`` to skip drawing the set
594           bits, the ``setsurface`` parameter (if set) will takes precedence over
595           this parameter
596        :type setcolor: Color or str or int or tuple(int, int, int, [int]) or
597           list(int, int, int, [int]) or None
598        :param unsetcolor: (optional) color to draw unset bits (default is
599           ``(0, 0, 0, 255)``, black), use ``None`` to skip drawing the unset
600           bits, the ``unsetsurface`` parameter (if set) will takes precedence
601           over this parameter
602        :type unsetcolor: Color or str or int or tuple(int, int, int, [int]) or
603           list(int, int, int, [int]) or None
604        :param dest: (optional) surface destination of where to position the
605           topleft corner of the mask being drawn (default is ``(0, 0)``), if a
606           Rect is used as the ``dest`` parameter, its ``x`` and ``y`` attributes
607           will be used as the destination, **NOTE1:** rects with a negative width
608           or height value will not be normalized before using their ``x`` and
609           ``y`` values, **NOTE2:** this destination value is only used to
610           position the mask on the surface, it does not offset the ``setsurface``
611           and ``unsetsurface`` from the mask, they are always aligned with the
612           mask (i.e. position ``(0, 0)`` on the mask always corresponds to
613           position ``(0, 0)`` on the ``setsurface`` and ``unsetsurface``)
614        :type dest: Rect or tuple(int, int) or list(int, int) or Vector2(int, int)
615  
616        :returns: the ``surface`` parameter (or a newly created surface if no
617           ``surface`` parameter was provided) with this mask drawn on it
618        :rtype: Surface
619  
620        :raises ValueError: if the ``setsurface`` parameter or ``unsetsurface``
621           parameter does not have the same format (bytesize/bitsize/alpha) as
622           the ``surface`` parameter
623  
624        .. note ::
625           To skip drawing the set bits, both ``setsurface`` and ``setcolor`` must
626           be ``None``. The ``setsurface`` parameter defaults to ``None``, but
627           ``setcolor`` defaults to a color value and therefore must be set to
628           ``None``.
629  
630        .. note ::
631           To skip drawing the unset bits, both ``unsetsurface`` and
632           ``unsetcolor`` must be ``None``. The ``unsetsurface`` parameter
633           defaults to ``None``, but ``unsetcolor`` defaults to a color value and
