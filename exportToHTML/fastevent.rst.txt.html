6    .. module:: pygame.fastevent
7       :synopsis: pygame module for interacting with events and queues from multiple
8                  threads.
9    
10   | :sl:`pygame module for interacting with events and queues`
11   
12   IMPORTANT NOTE: THIS MODULE IS DEPRECATED IN PYGAME 2.2
13   
14   In older pygame versions before pygame 2, :mod:`pygame.event` was not well
15   suited for posting events from different threads. This module served as a
16   replacement (with less features) for multithreaded use. Now, the usage of this
17   module is highly discouraged in favour of use of the main :mod:`pygame.event`
18   module. This module will be removed in a future pygame version.
19   
20   Below, the legacy docs of the module is provided

40   .. function:: pump
41   
42      | :sl:`internally process pygame event handlers`
43      | :sg:`pump() -> None`
44   
45      For each frame of your game, you will need to make some sort of call to the
46      event queue. This ensures your program can internally interact with the rest
47      of the operating system.
48   
49      This function is not necessary if your program is consistently processing
50      events on the queue through the other :mod:`pygame.fastevent` functions.
51   
52      There are important things that must be dealt with internally in the event
53      queue. The main window may need to be repainted or respond to the system. If
54      you fail to make a call to the event queue for too long, the system may
55      decide your program has locked up.
56   
57      .. ## pygame.fastevent.pump ##

59   .. function:: wait
60   
61      | :sl:`wait for an event`
62      | :sg:`wait() -> Event`
63   
64      Returns the current event on the queue. If there are no messages
65      waiting on the queue, this will not return until one is available.
66      Sometimes it is important to use this wait to get events from the queue,
67      it will allow your application to idle when the user isn't doing anything
68      with it.
69   
70      .. ## pygame.fastevent.wait ##
71   
72   .. function:: poll
73   
74      | :sl:`get an available event`
75      | :sg:`poll() -> Event`
76   
77      Returns next event on queue. If there is no event waiting on the queue,
78      this will return an event with type NOEVENT.
79   
80      .. ## pygame.fastevent.poll ##

83   
84      | :sl:`get all events from the queue`
85      | :sg:`get() -> list of Events`

88   
89      .. ## pygame.fastevent.get ##
90   
91   .. function:: post
92   
93      | :sl:`place an event on the queue`
94      | :sg:`post(Event) -> None`
95   
96      This will post your own event objects onto the event queue. You can post
97      any event type you want, but some care must be taken. For example, if you
98      post a MOUSEBUTTONDOWN event to the queue, it is likely any code receiving
99      the event will expect the standard MOUSEBUTTONDOWN attributes to be
100     available, like 'pos' and 'button'.
101  
102     Because pygame.fastevent.post() may have to wait for the queue to empty,
103     you can get into a dead lock if you try to append an event on to a full
104     queue from the thread that processes events. For that reason I do not
105     recommend using this function in the main thread of an SDL program.
