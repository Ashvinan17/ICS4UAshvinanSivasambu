12   .. class:: Overlay
13   
14      | :sl:`pygame object for video overlay graphics`
15      | :sg:`Overlay(format, (width, height)) -> Overlay`
16   
17      The Overlay objects provide support for accessing hardware video overlays.
18      Video overlays do not use standard ``RGB`` pixel formats, and can use
19      multiple resolutions of data to create a single image.
20   
21      The Overlay objects represent lower level access to the display hardware. To
22      use the object you must understand the technical details of video overlays.
23   
24      The Overlay format determines the type of pixel data used. Not all hardware
25      will support all types of overlay formats. Here is a list of available
26      format types:
27   
28      ::
29   
30        YV12_OVERLAY, IYUV_OVERLAY, YUY2_OVERLAY, UYVY_OVERLAY, YVYU_OVERLAY
31   
32      The width and height arguments control the size for the overlay image data.
33      The overlay image can be displayed at any size, not just the resolution of
34      the overlay.
35   
36      The overlay objects are always visible, and always show above the regular
37      display contents.
38   
39      .. method:: display
40   
41         | :sl:`set the overlay pixel data`
42         | :sg:`display((y, u, v)) -> None`
43         | :sg:`display() -> None`
44   
45         Display the YUV data in SDL's overlay planes. The y, u, and v arguments
46         are strings of binary data. The data must be in the correct format used
47         to create the Overlay.
48   
49         If no argument is passed in, the Overlay will simply be redrawn with the
50         current data. This can be useful when the Overlay is not really hardware
51         accelerated.
52   
53         The strings are not validated, and improperly sized strings could crash
54         the program.
55   
56         .. ## Overlay.display ##
57   
58      .. method:: set_location
59   
60         | :sl:`control where the overlay is displayed`
61         | :sg:`set_location(rect) -> None`
62   
63         Set the location for the overlay. The overlay will always be shown
64         relative to the main display Surface. This does not actually redraw the
65         overlay, it will be updated on the next call to ``Overlay.display()``.
66   
67         .. ## Overlay.set_location ##
68   
69      .. method:: get_hardware
70   
71         | :sl:`test if the Overlay is hardware accelerated`
72         | :sg:`get_hardware(rect) -> int`
73   
74         Returns a True value when the Overlay is hardware accelerated. If the
75         platform does not support acceleration, software rendering is used.
