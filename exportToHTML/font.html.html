155  <p>The font module allows for rendering TrueType fonts into a new Surface object.
156  It accepts any UCS-2 character ('u0001' to 'uFFFF'). This module is optional
157  and requires SDL_ttf as a dependency. You should test that <a class="tooltip reference internal" href="#module-pygame.font" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.font</span></code><span class="tooltip-content">pygame module for loading and rendering fonts</span></a>
158  is available and initialized before attempting to use the module.</p>
159  <p>Most of the work done with fonts are done by using the actual Font objects. The
160  module by itself only has routines to initialize the module and create Font
161  objects with <code class="docutils literal notranslate"><span class="pre">pygame.font.Font()</span></code>.</p>
162  <p>You can load fonts from the system by using the <code class="docutils literal notranslate"><span class="pre">pygame.font.SysFont()</span></code>
163  function. There are a few other functions to help lookup the system fonts.</p>
164  <p>Pygame comes with a builtin default font. This can always be accessed by
165  passing None as the font name.</p>
166  <p>To use the <a class="tooltip reference internal" href="freetype.html#module-pygame.freetype" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.freetype</span></code><span class="tooltip-content">Enhanced pygame module for loading and rendering computer fonts</span></a> based <code class="docutils literal notranslate"><span class="pre">pygame.ftfont</span></code> as
167  <a class="tooltip reference internal" href="#module-pygame.font" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.font</span></code><span class="tooltip-content">pygame module for loading and rendering fonts</span></a> define the environment variable PYGAME_FREETYPE before the
168  first import of <a class="tooltip reference internal" href="pygame.html#module-pygame" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame</span></code><span class="tooltip-content">the top level pygame package</span></a>. Module <code class="docutils literal notranslate"><span class="pre">pygame.ftfont</span></code> is a <a class="tooltip reference internal" href="#module-pygame.font" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.font</span></code><span class="tooltip-content">pygame module for loading and rendering fonts</span></a>
169  compatible module that passes all but one of the font module unit tests:
170  it does not have the UCS-2 limitation of the SDL_ttf based font module, so
171  fails to raise an exception for a code point greater than 'uFFFF'. If
172  <a class="tooltip reference internal" href="freetype.html#module-pygame.freetype" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.freetype</span></code><span class="tooltip-content">Enhanced pygame module for loading and rendering computer fonts</span></a> is unavailable then the SDL_ttf font module will be
173  loaded instead.</p>

235  <dt class="sig sig-object py title" id="pygame.font.match_font">
236  <span class="sig-prename descclassname"><span class="pre">pygame.font.</span></span><span class="sig-name descname"><span class="pre">match_font</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.font.match_font" title="Permalink to this definition">¶</a></dt>
237  <dd><div class="line-block">
238  <div class="line"><span class="summaryline">find a specific font on the system</span></div>
239  <div class="line"><span class="signature">match_font(name, bold=False, italic=False) -&gt; path</span></div>
240  </div>
241  <p>Returns the full path to a font file on the system. If bold or italic are
242  set to true, this will attempt to find the correct family of font.</p>
243  <p>The font name can also be an iterable of font names, a string of
244  comma-separated font names, or a bytes of comma-separated font names, in
245  which case the set of names will be searched in order.
246  If none of the given names are found, None is returned.</p>
247  <div class="versionadded">
248  <p><span class="versionmodified added">New in pygame 2.0.1: </span>Accept an iterable of font names.</p>
249  </div>
250  <p>Example:</p>
251  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="nb">print</span> <span class="n">pygame</span><span class="o">.</span><span class="n">font</span><span class="o">.</span><span class="n">match_font</span><span class="p">(</span><span class="s1">&#39;bitstreamverasans&#39;</span><span class="p">)</span>
252  <span class="c1"># output is: /usr/share/fonts/truetype/ttf-bitstream-vera/Vera.ttf</span>
253  <span class="c1"># (but only if you have Vera on your system)</span>
254  </pre></div>
255  </div>
256  </dd></dl>

259  <dt class="sig sig-object py title" id="pygame.font.SysFont">
260  <span class="sig-prename descclassname"><span class="pre">pygame.font.</span></span><span class="sig-name descname"><span class="pre">SysFont</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.font.SysFont" title="Permalink to this definition">¶</a></dt>
261  <dd><div class="line-block">
262  <div class="line"><span class="summaryline">create a Font object from the system fonts</span></div>
263  <div class="line"><span class="signature">SysFont(name, size, bold=False, italic=False) -&gt; Font</span></div>
264  </div>
265  <p>Return a new Font object that is loaded from the system fonts. The font will
266  match the requested bold and italic flags. Pygame uses a small set of common
267  font aliases. If the specific font you ask for is not available, a reasonable
268  alternative may be used. If a suitable system font is not found this will
269  fall back on loading the default pygame font.</p>
270  <p>The font name can also be an iterable of font names, a string of
271  comma-separated font names, or a bytes of comma-separated font names, in
272  which case the set of names will be searched in order.</p>
273  <div class="versionadded">
274  <p><span class="versionmodified added">New in pygame 2.0.1: </span>Accept an iterable of font names.</p>
275  </div>

376  <p>Load a new font from a given filename or a python file object. The size is
377  the height of the font in pixels. If the filename is None the pygame default
378  font will be loaded. If a font cannot be loaded from the arguments given an
379  exception will be raised. Once the font is created the size cannot be
380  changed.</p>
381  <p>Font objects are mainly used to render text into new Surface objects. The
382  render can emulate bold or italic features, but it is better to load from a
383  font with actual italic or bold glyphs. The rendered text can be regular
384  strings or unicode.</p>

438  <dl class="py method definition">
439  <dt class="sig sig-object py title" id="pygame.font.Font.render">
440  <span class="sig-name descname"><span class="pre">render</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.font.Font.render" title="Permalink to this definition">¶</a></dt>
441  <dd><div class="line-block">
442  <div class="line"><span class="summaryline">draw text on a new Surface</span></div>
443  <div class="line"><span class="signature">render(text, antialias, color, background=None) -&gt; Surface</span></div>
444  </div>
445  <p>This creates a new Surface with the specified text rendered on it. pygame
446  provides no way to directly draw text on an existing Surface: instead you
447  must use <code class="docutils literal notranslate"><span class="pre">Font.render()</span></code> to create an image (Surface) of the text, then
448  blit this image onto another Surface.</p>
449  <p>The text can only be a single line: newline characters are not rendered.
450  Null characters ('x00') raise a TypeError. Both Unicode and char (byte)
451  strings are accepted. For Unicode strings only UCS-2 characters
452  ('u0001' to 'uFFFF') were previously supported and any greater unicode
453  codepoint would raise a UnicodeError. Now, characters in the UCS-4 range
454  are supported. For char strings a <code class="docutils literal notranslate"><span class="pre">LATIN1</span></code> encoding is assumed. The
455  antialias argument is a boolean: if true the characters will have smooth
456  edges. The color argument is the color of the text
457  [e.g.: (0,0,255) for blue]. The optional background argument is a color
458  to use for the text background. If no background is passed the area
459  outside the text will be transparent.</p>
460  <p>The Surface returned will be of the dimensions required to hold the text.
461  (the same as those returned by Font.size()). If an empty string is passed
462  for the text, a blank surface will be returned that is zero pixel wide and
463  the height of the font.</p>
464  <p>Depending on the type of background and antialiasing used, this returns
465  different types of Surfaces. For performance reasons, it is good to know
466  what type of image will be used. If antialiasing is not used, the return
467  image will always be an 8-bit image with a two-color palette. If the
468  background is transparent a colorkey will be set. Antialiased images are
469  rendered to 24-bit <code class="docutils literal notranslate"><span class="pre">RGB</span></code> images. If the background is transparent a
470  pixel alpha will be included.</p>
471  <p>Optimization: if you know that the final destination for the text (on the
472  screen) will always have a solid background, and the text is antialiased,
473  you can improve performance by specifying the background color. This will
474  cause the resulting image to maintain transparency information by
475  colorkey rather than (much less efficient) alpha values.</p>
476  <p>If you render '\n' an unknown char will be rendered. Usually a
477  rectangle. Instead you need to handle new lines yourself.</p>
478  <p>Font rendering is not thread safe: only a single thread can render text
479  at any time.</p>
480  <div class="versionchanged">
481  <p><span class="versionmodified changed">Changed in pygame 2.0.3: </span>Rendering UCS_4 unicode works and does not
482  raise an exception. Use <cite>if hasattr(pygame.font, 'UCS_4'):</cite> to see if
483  pygame supports rendering UCS_4 unicode including more languages and
484  emoji.</p>
485  </div>
486  </dd></dl>
