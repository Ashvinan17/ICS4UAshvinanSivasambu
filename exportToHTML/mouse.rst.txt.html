3    :mod:`pygame.mouse`
4    ===================
5    
6    .. module:: pygame.mouse
7       :synopsis: pygame module to work with the mouse
8    
9    | :sl:`pygame module to work with the mouse`
10   
11   The mouse functions can be used to get the current state of the mouse device.
12   These functions can also alter the system cursor for the mouse.
13   
14   When the display mode is set, the event queue will start receiving mouse
15   events. The mouse buttons generate ``pygame.MOUSEBUTTONDOWN`` and
16   ``pygame.MOUSEBUTTONUP`` events when they are pressed and released. These
17   events contain a button attribute representing which button was pressed. The
18   mouse wheel will generate ``pygame.MOUSEBUTTONDOWN`` and 
19   ``pygame.MOUSEBUTTONUP`` events when rolled. The button will be set to 4 
20   when the wheel is rolled up, and to button 5 when the wheel is rolled down. 
21   Whenever the mouse is moved it generates a ``pygame.MOUSEMOTION`` event. The 
22   mouse movement is broken into small and accurate motion events. As the mouse 
23   is moving many motion events will be placed on the queue. Mouse motion events 
24   that are not properly cleaned from the event queue are the primary reason the 
25   event queue fills up.
26   
27   If the mouse cursor is hidden, and input is grabbed to the current display the
28   mouse will enter a virtual input mode, where the relative movements of the
29   mouse will never be stopped by the borders of the screen. See the functions
30   ``pygame.mouse.set_visible()`` and ``pygame.event.set_grab()`` to get this
31   configured.
32   
33   
34   **Mouse Wheel Behavior in pygame 2**
35   
36   There is proper functionality for mouse wheel behaviour with pygame 2 supporting
37   ``pygame.MOUSEWHEEL`` events.  The new events support horizontal and vertical 
38   scroll movements, with signed integer values representing the amount scrolled 
39   (``x`` and ``y``), as well as ``flipped`` direction (the set positive and 
40   negative values for each axis is flipped). Read more about SDL2 
41   input-related changes here `<https://wiki.libsdl.org/MigrationGuide#Input>`_
42   
43   In pygame 2, the mouse wheel functionality can be used by listening for the 
44   ``pygame.MOUSEWHEEL`` type of an event (Bear in mind they still emit 
45   ``pygame.MOUSEBUTTONDOWN`` events like in pygame 1.x, as well).
46   When this event is triggered, a developer can access the appropriate ``Event`` object 
47   with ``pygame.event.get()``. The object can be used to access data about the mouse 
48   scroll, such as ``which`` (it will tell you what exact mouse device trigger the event).
49   
50   .. code-block:: python
51      :caption: Code example of mouse scroll (tested on 2.0.0.dev7)
52      :name: test.py
53   
54      # Taken from husano896's PR thread (slightly modified)
55      import pygame
56      from pygame.locals import *
57      pygame.init()
58      screen = pygame.display.set_mode((640, 480))
59      clock = pygame.time.Clock()
60      
61      def main():
62         while True:
63            for event in pygame.event.get():
64                  if event.type == QUIT:
65                     pygame.quit()
66                     return
67                  elif event.type == MOUSEWHEEL:
68                     print(event) 
69                     print(event.x, event.y)
70                     print(event.flipped)
71                     print(event.which)
72                     # can access properties with 
73                     # proper notation(ex: event.y)
74            clock.tick(60)
75   
76      # Execute game:
77      main()

79   .. function:: get_pressed
80   
81      | :sl:`get the state of the mouse buttons`
82      | :sg:`get_pressed(num_buttons=3) -> (button1, button2, button3)`
83      | :sg:`get_pressed(num_buttons=5) -> (button1, button2, button3, button4, button5)`
84   
85      Returns a sequence of booleans representing the state of all the mouse
86      buttons. A true value means the mouse is currently being pressed at the time
87      of the call.
88   
89      Note, to get all of the mouse events it is better to use either 
90      ``pygame.event.wait()`` or ``pygame.event.get()`` and check all of those 
91      events to see if they are ``MOUSEBUTTONDOWN``, ``MOUSEBUTTONUP``, or
92      ``MOUSEMOTION``.
93   
94      Note, that on ``X11`` some X servers use middle button emulation. When you
95      click both buttons ``1`` and ``3`` at the same time a ``2`` button event 
96      can be emitted.
97   
98      Note, remember to call ``pygame.event.get()`` before this function.
99      Otherwise it will not work as expected.
100  
101     To support five button mice, an optional parameter ``num_buttons`` has been 
102     added in pygame 2. When this is set to ``5``, ``button4`` and ``button5`` 
103     are added to the returned tuple. Only ``3`` and ``5`` are valid values 
104     for this parameter.
105     
106     .. versionchanged:: 2.0.0 ``num_buttons`` argument added

110  .. function:: get_pos
111  
112     | :sl:`get the mouse cursor position`
113     | :sg:`get_pos() -> (x, y)`
114  
115     Returns the ``x`` and ``y`` position of the mouse cursor. The position is
116     relative to the top-left corner of the display. The cursor position can be
117     located outside of the display window, but is always constrained to the
118     screen.

122  .. function:: get_rel
123  
124     | :sl:`get the amount of mouse movement`
125     | :sg:`get_rel() -> (x, y)`
126  
127     Returns the amount of movement in ``x`` and ``y`` since the previous call to
128     this function. The relative movement of the mouse cursor is constrained to
129     the edges of the screen, but see the virtual input mouse mode for a way
130     around this. Virtual input mode is described at the top of the page.

167  .. function:: get_focused
168  
169     | :sl:`check if the display is receiving mouse input`
170     | :sg:`get_focused() -> bool`
171  
172     Returns true when pygame is receiving mouse input events (or, in windowing
173     terminology, is "active" or has the "focus").
174  
175     This method is most useful when working in a window. By contrast, in
176     full-screen mode, this method always returns true.
177  
178     Note: under ``MS`` Windows, the window that has the mouse focus also has the
179     keyboard focus. But under X-Windows, one window can receive mouse events and
180     another receive keyboard events. ``pygame.mouse.get_focused()`` indicates
181     whether the pygame window receives mouse events.
