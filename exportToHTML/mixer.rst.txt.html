3    :mod:`pygame.mixer`
4    ===================
5    
6    .. module:: pygame.mixer
7       :synopsis: pygame module for loading and playing sounds
8    
9    | :sl:`pygame module for loading and playing sounds`
10   
11   This module contains classes for loading Sound objects and controlling
12   playback. The mixer module is optional and depends on SDL_mixer. Your program
13   should test that :mod:`pygame.mixer` is available and initialized before using
14   it.
15   
16   The mixer module has a limited number of channels for playback of sounds.
17   Usually programs tell pygame to start playing audio and it selects an available
18   channel automatically. The default is 8 simultaneous channels, but complex
19   programs can get more precise control over the number of channels and their
20   use.
21   
22   All sound playback is mixed in background threads. When you begin to play a
23   Sound object, it will return immediately while the sound continues to play. A
24   single Sound object can also be actively played back multiple times.
25   
26   The mixer also has a special streaming channel. This is for music playback and
27   is accessed through the :mod:`pygame.mixer.music` module. Consider using this
28   module for playing long running music. Unlike mixer module, the music module
29   streams the music from the files without loading music at once into memory.
30   
31   The mixer module must be initialized like other pygame modules, but it has some
32   extra conditions. The ``pygame.mixer.init()`` function takes several optional
33   arguments to control the playback rate and sample size. Pygame will default to
34   reasonable values, but pygame cannot perform Sound resampling, so the mixer
35   should be initialized to match the values of your audio resources.
36   
37   ``NOTE``: For less laggy sound use a smaller buffer size. The default
38   is set to reduce the chance of scratchy sounds on some computers. You can
39   change the default buffer by calling :func:`pygame.mixer.pre_init` before
40   :func:`pygame.mixer.init` or :func:`pygame.init` is called. For example:
41   ``pygame.mixer.pre_init(44100,-16,2, 1024)``
42   
43   
44   .. function:: init
45   
46      | :sl:`initialize the mixer module`
47      | :sg:`init(frequency=44100, size=-16, channels=2, buffer=512, devicename=None, allowedchanges=AUDIO_ALLOW_FREQUENCY_CHANGE | AUDIO_ALLOW_CHANNELS_CHANGE) -> None`
48   
49      Initialize the mixer module for Sound loading and playback. The default
50      arguments can be overridden to provide specific audio mixing. Keyword
51      arguments are accepted. For backwards compatibility, argument values of 
52      0 are replaced with the startup defaults, except for ``allowedchanges``,
53      where -1 is used. (startup defaults may be changed by a :func:`pre_init` call).
54   
55      The size argument represents how many bits are used for each audio sample.
56      If the value is negative then signed sample values will be used. Positive
57      values mean unsigned audio samples will be used. An invalid value raises an
58      exception.
59   
60      The channels argument is used to specify whether to use mono or stereo. 1
61      for mono and 2 for stereo.
62   
63      The buffer argument controls the number of internal samples used in the
64      sound mixer. The default value should work for most cases. It can be lowered
65      to reduce latency, but sound dropout may occur. It can be raised to larger
66      values to ensure playback never skips, but it will impose latency on sound
67      playback. The buffer size must be a power of two (if not it is rounded up to
68      the next nearest power of 2).
69   
70      Some platforms require the :mod:`pygame.mixer` module to be initialized
71      after the display modules have initialized. The top level ``pygame.init()``
72      takes care of this automatically, but cannot pass any arguments to the mixer
73      init. To solve this, mixer has a function ``pygame.mixer.pre_init()`` to set
74      the proper defaults before the toplevel init is used.
75   
76      When using allowedchanges=0 it will convert the samples at runtime to match
77      what the hardware supports. For example a sound card may not
78      support 16bit sound samples, so instead it will use 8bit samples internally.
79      If AUDIO_ALLOW_FORMAT_CHANGE is supplied, then the requested format will
80      change to the closest that SDL2 supports.
81   
82      Apart from 0, allowedchanged accepts the following constants ORed together:
83   
84         - AUDIO_ALLOW_FREQUENCY_CHANGE
85         - AUDIO_ALLOW_FORMAT_CHANGE
86         - AUDIO_ALLOW_CHANNELS_CHANGE
87         - AUDIO_ALLOW_ANY_CHANGE
88   
89      It is safe to call this more than once, but after the mixer is initialized
90      you cannot change the playback arguments without first calling
91      ``pygame.mixer.quit()``.

263  .. class:: Sound
264  
265     | :sl:`Create a new Sound object from a file or buffer object`
266     | :sg:`Sound(filename) -> Sound`
267     | :sg:`Sound(file=filename) -> Sound`
268     | :sg:`Sound(file=pathlib_path) -> Sound`
269     | :sg:`Sound(buffer) -> Sound`
270     | :sg:`Sound(buffer=buffer) -> Sound`
271     | :sg:`Sound(object) -> Sound`
272     | :sg:`Sound(file=object) -> Sound`
273     | :sg:`Sound(array=object) -> Sound`
274  
275     Load a new sound buffer from a filename, a python file object or a readable
276     buffer object. Limited resampling will be performed to help the sample match
277     the initialize arguments for the mixer. A Unicode string can only be a file
278     pathname. A bytes object can be either a pathname or a buffer object.
279     Use the 'file' or 'buffer' keywords to avoid ambiguity; otherwise Sound may
280     guess wrong. If the array keyword is used, the object is expected to export 
281     a new buffer interface (The object is checked for a buffer interface first.)
282  
283     The Sound object represents actual sound sample data. Methods that change
284     the state of the Sound object will the all instances of the Sound playback.
285     A Sound object also exports a new buffer interface.
286  
287     The Sound can be loaded from an ``OGG`` audio file or from an uncompressed
288     ``WAV``.
289  
290     Note: The buffer will be copied internally, no data will be shared between
291     it and the Sound object.
292  
293     For now buffer and array support is consistent with ``sndarray.make_sound``
294     for Numeric arrays, in that sample sign and byte order are ignored. This
295     will change, either by correctly handling sign and byte order, or by raising
296     an exception when different. Also, source samples are truncated to fit the
297     audio sample size. This will not change.
298  
299     .. versionadded:: 1.8 ``pygame.mixer.Sound(buffer)``
300     .. versionadded:: 1.9.2
301        :class:`pygame.mixer.Sound` keyword arguments and array interface support
302     .. versionadded:: 2.0.1 pathlib.Path support on Python 3.
303  
304     .. method:: play
305  
306        | :sl:`begin sound playback`
307        | :sg:`play(loops=0, maxtime=0, fade_ms=0) -> Channel`
308  
309        Begin playback of the Sound (i.e., on the computer's speakers) on an
310        available Channel. This will forcibly select a Channel, so playback may
311        cut off a currently playing sound if necessary.
312  
313        The loops argument controls how many times the sample will be repeated
314        after being played the first time. A value of 5 means that the sound will
315        be played once, then repeated five times, and so is played a total of six
316        times. The default value (zero) means the Sound is not repeated, and so
317        is only played once. If loops is set to -1 the Sound will loop
318        indefinitely (though you can still call ``stop()`` to stop it).
319  
320        The maxtime argument can be used to stop playback after a given number of
321        milliseconds.
322  
323        The fade_ms argument will make the sound start playing at 0 volume and
324        fade up to full volume over the time given. The sample may end before the
325        fade-in is complete.
326  
327        This returns the Channel object for the channel that was selected.
328

433        of the Sound after a given number of milliseconds.

479     .. method:: set_volume
480  
481        | :sl:`set the volume of a playing channel`
482        | :sg:`set_volume(value) -> None`
483        | :sg:`set_volume(left, right) -> None`
484  
485        Set the volume (loudness) of a playing sound. When a channel starts to
486        play its volume value is reset. This only affects the current sound. The
487        value argument is between 0.0 and 1.0.
488  
489        If one argument is passed, it will be the volume of both speakers. If two
490        arguments are passed and the mixer is in stereo mode, the first argument
491        will be the volume of the left speaker and the second will be the volume
492        of the right speaker. (If the second argument is ``None``, the first
493        argument will be the volume of both speakers.)
494  
495        If the channel is playing a Sound on which ``set_volume()`` has also been
496        called, both calls are taken into account. For example:
497  
498        ::
499  
500            sound = pygame.mixer.Sound("s.wav")
501            channel = s.play()      # Sound plays at full volume by default
502            sound.set_volume(0.9)   # Now plays at 90% of full volume.
503            sound.set_volume(0.6)   # Now plays at 60% (previous value replaced).
504            channel.set_volume(0.5) # Now plays at 30% (0.6 * 0.5).
505

566     .. method:: set_endevent
567  
568        | :sl:`have the channel send an event when playback stops`
569        | :sg:`set_endevent() -> None`
570        | :sg:`set_endevent(type) -> None`
571  
572        When an endevent is set for a channel, it will send an event to the
573        pygame queue every time a sound finishes playing on that channel (not
574        just the first time). Use ``pygame.event.get()`` to retrieve the endevent
575        once it's sent.
576  
577        Note that if you called ``Sound.play(n)`` or ``Channel.play(sound,n)``,
578        the end event is sent only once: after the sound has been played "n+1"
579        times (see the documentation of Sound.play).
580  
581        If ``Channel.stop()`` or ``Channel.play()`` is called while the sound was
582        still playing, the event will be posted immediately.
583  
584        The type argument will be the event id sent to the queue. This can be any
585        valid event type, but a good choice would be a value between
586        ``pygame.locals.USEREVENT`` and ``pygame.locals.NUMEVENTS``. If no type
587        argument is given then the Channel will stop sending endevents.
588
