130  <p>Pygame offers control over the system hardware cursor. Pygame supports
131  black and white cursors (bitmap cursors), as well as system variant cursors and color cursors.
132  You control the cursor with functions inside <a class="tooltip reference internal" href="mouse.html#module-pygame.mouse" title=""><code class="xref py py-mod docutils literal notranslate"><span class="pre">pygame.mouse</span></code><span class="tooltip-content">pygame module to work with the mouse</span></a>.</p>
133  <p>This cursors module contains functions for loading and decoding various
134  cursor formats. These allow you to easily store your cursors in external files
135  or directly as encoded python strings.</p>
136  <p>The module includes several standard cursors. The <a class="tooltip reference internal" href="mouse.html#pygame.mouse.set_cursor" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.mouse.set_cursor()</span></code><span class="tooltip-content">set the mouse cursor to a new cursor</span></a>
137  function takes several arguments. All those arguments have been stored in a
138  single tuple you can call like this:</p>
139  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">pygame</span><span class="o">.</span><span class="n">mouse</span><span class="o">.</span><span class="n">set_cursor</span><span class="p">(</span><span class="o">*</span><span class="n">pygame</span><span class="o">.</span><span class="n">cursors</span><span class="o">.</span><span class="n">arrow</span><span class="p">)</span>
140  </pre></div>
141  </div>
142  <p>The following variables can be passed to <code class="docutils literal notranslate"><span class="pre">pygame.mouse.set_cursor</span></code> function:</p>
143  <blockquote>
144  <div><ul class="simple">
145  <li><p><code class="docutils literal notranslate"><span class="pre">pygame.cursors.arrow</span></code></p></li>
146  <li><p><code class="docutils literal notranslate"><span class="pre">pygame.cursors.diamond</span></code></p></li>
147  <li><p><code class="docutils literal notranslate"><span class="pre">pygame.cursors.broken_x</span></code></p></li>
148  <li><p><code class="docutils literal notranslate"><span class="pre">pygame.cursors.tri_left</span></code></p></li>
149  <li><p><code class="docutils literal notranslate"><span class="pre">pygame.cursors.tri_right</span></code></p></li>
150  </ul>
151  </div></blockquote>
152  <p>This module also contains a few cursors as formatted strings. You'll need to
153  pass these to <code class="docutils literal notranslate"><span class="pre">pygame.cursors.compile()</span></code> function before you can use them.
154  The example call would look like this:</p>
155  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">cursor</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">cursors</span><span class="o">.</span><span class="n">compile</span><span class="p">(</span><span class="n">pygame</span><span class="o">.</span><span class="n">cursors</span><span class="o">.</span><span class="n">textmarker_strings</span><span class="p">)</span>
156  <span class="gp">&gt;&gt;&gt; </span><span class="n">pygame</span><span class="o">.</span><span class="n">mouse</span><span class="o">.</span><span class="n">set_cursor</span><span class="p">((</span><span class="mi">8</span><span class="p">,</span> <span class="mi">16</span><span class="p">),</span> <span class="p">(</span><span class="mi">0</span><span class="p">,</span> <span class="mi">0</span><span class="p">),</span> <span class="o">*</span><span class="n">cursor</span><span class="p">)</span>
157  </pre></div>
158  </div>
159  <p>The following strings can be converted into cursor bitmaps with
160  <code class="docutils literal notranslate"><span class="pre">pygame.cursors.compile()</span></code> :</p>
161  <blockquote>
162  <div><ul class="simple">
163  <li><p><code class="docutils literal notranslate"><span class="pre">pygame.cursors.thickarrow_strings</span></code></p></li>

178  system cursor. This returns the binary data in the form of two tuples.

182  the black and white pixels. Some system allow you to set a special toggle
183  color for the system color, this is also called the xor color. If the system

219  <dt class="sig sig-object py title" id="pygame.cursors.load_xbm">
220  <span class="sig-prename descclassname"><span class="pre">pygame.cursors.</span></span><span class="sig-name descname"><span class="pre">load_xbm</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.cursors.load_xbm" title="Permalink to this definition">¶</a></dt>
221  <dd><div class="line-block">
222  <div class="line"><span class="summaryline">load cursor data from an XBM file</span></div>
223  <div class="line"><span class="signature">load_xbm(cursorfile) -&gt; cursor_args</span></div>
224  <div class="line"><span class="signature">load_xbm(cursorfile, maskfile) -&gt; cursor_args</span></div>
225  </div>
226  <p>This loads cursors for a simple subset of <code class="docutils literal notranslate"><span class="pre">XBM</span></code> files. <code class="docutils literal notranslate"><span class="pre">XBM</span></code> files are
227  traditionally used to store cursors on UNIX systems, they are an ASCII
228  format used to represent simple images.</p>
229  <p>Sometimes the black and white color values will be split into two separate
230  <code class="docutils literal notranslate"><span class="pre">XBM</span></code> files. You can pass a second maskfile argument to load the two
231  images into a single cursor.</p>
232  <p>The cursorfile and maskfile arguments can either be filenames or file-like
233  object with the readlines method.</p>
234  <p>The return value cursor_args can be passed directly to the
235  <code class="docutils literal notranslate"><span class="pre">pygame.mouse.set_cursor()</span></code> function.</p>
236  </dd></dl>

273  <p>In pygame 2, there are 3 types of cursors you can create to give your
274  game that little bit of extra polish. There's <strong>bitmap</strong> type cursors,
275  which existed in pygame 1.x, and are compiled from a string or load from an xbm file.
276  Then there are <strong>system</strong> type cursors, where you choose a preset that will
277  convey the same meaning but look native across different operating systems.
278  Finally you can create a <strong>color</strong> cursor, which displays a pygame surface as the cursor.</p>
279  <p><strong>Creating a system cursor</strong></p>
280  <p>Choose a constant from this list, pass it into <code class="docutils literal notranslate"><span class="pre">pygame.cursors.Cursor(constant)</span></code>,
281  and you're good to go. Be advised that not all systems support every system
282  cursor, and you may get a substitution instead. For example, on MacOS,
283  WAIT/WAITARROW should show up as an arrow, and SIZENWSE/SIZENESW/SIZEALL
284  should show up as a closed hand. And on Wayland, every SIZE cursor should
285  show up as a hand.</p>

308  <p><strong>Creating a cursor without passing arguments</strong></p>
309  <p>In addition to the cursor constants available and described above,
310  you can also call <code class="docutils literal notranslate"><span class="pre">pygame.cursors.Cursor()</span></code>, and your cursor is ready (doing that is the same as
311  calling <code class="docutils literal notranslate"><span class="pre">pygame.cursors.Cursor(pygame.SYSTEM_CURSOR_ARROW)</span></code>.
312  Doing one of those calls actually creates a system cursor using the default native image.</p>
313  <p><strong>Creating a color cursor</strong></p>
314  <p>To create a color cursor, create a <code class="docutils literal notranslate"><span class="pre">Cursor</span></code> from a <code class="docutils literal notranslate"><span class="pre">hotspot</span></code> and a <code class="docutils literal notranslate"><span class="pre">surface</span></code>.
315  <code class="docutils literal notranslate"><span class="pre">hotspot</span></code> is an (x,y) coordinate that determines where in the cursor the exact point is.
316  The hotspot position must be within the bounds of the <code class="docutils literal notranslate"><span class="pre">surface</span></code>.</p>
317  <p><strong>Creating a bitmap cursor</strong></p>
318  <p>When the mouse cursor is visible, it will be displayed as a black and white
319  bitmap using the given bitmask arrays. The <code class="docutils literal notranslate"><span class="pre">size</span></code> is a sequence containing
320  the cursor width and height. <code class="docutils literal notranslate"><span class="pre">hotspot</span></code> is a sequence containing the cursor
321  hotspot position.</p>
322  <p>A cursor has a width and height, but a mouse position is represented by a
323  set of point coordinates. So the value passed into the cursor <code class="docutils literal notranslate"><span class="pre">hotspot</span></code>
324  variable helps pygame to actually determine at what exact point the cursor
325  is at.</p>
326  <p><code class="docutils literal notranslate"><span class="pre">xormasks</span></code> is a sequence of bytes containing the cursor xor data masks.
327  Lastly <code class="docutils literal notranslate"><span class="pre">andmasks</span></code>, a sequence of bytes containing the cursor bitmask data.
328  To create these variables, we can make use of the
329  <a class="tooltip reference internal" href="#pygame.cursors.compile" title=""><code class="xref py py-func docutils literal notranslate"><span class="pre">pygame.cursors.compile()</span></code><span class="tooltip-content">create binary cursor data from simple strings</span></a> function.</p>
330  <p>Width and height must be a multiple of 8, and the mask arrays must be the
331  correct size for the given width and height. Otherwise an exception is raised.</p>
