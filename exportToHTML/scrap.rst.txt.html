6    .. module:: pygame.scrap
7       :synopsis: pygame module for clipboard support.
8    
9    | :sl:`pygame module for clipboard support.`
10   
11   **EXPERIMENTAL!**: This API may change or disappear in later pygame releases. If
12   you use this, your code may break with the next pygame release.
13   
14   The scrap module is for transferring data to/from the clipboard. This allows
15   for cutting and pasting data between pygame and other applications. Some basic
16   data (MIME) types are defined and registered:
17   
18   ::
19   
20      pygame         string
21     constant        value        description
22     --------------------------------------------------
23     SCRAP_TEXT   "text/plain"    plain text
24     SCRAP_BMP    "image/bmp"     BMP encoded image data
25     SCRAP_PBM    "image/pbm"     PBM encoded image data
26     SCRAP_PPM    "image/ppm"     PPM encoded image data
27   
28   ``pygame.SCRAP_PPM``, ``pygame.SCRAP_PBM`` and ``pygame.SCRAP_BMP`` are
29   suitable for surface buffers to be shared with other applications.
30   ``pygame.SCRAP_TEXT`` is an alias for the plain text clipboard type.
31   
32   Depending on the platform, additional types are automatically registered when
33   data is placed into the clipboard to guarantee a consistent sharing behaviour
34   with other applications. The following listed types can be used as strings to
35   be passed to the respective :mod:`pygame.scrap` module functions.
36   
37   For **Windows** platforms, these additional types are supported automatically
38   and resolve to their internal definitions:
39   
40   ::
41   
42     "text/plain;charset=utf-8"   UTF-8 encoded text
43     "audio/wav"                  WAV encoded audio
44     "image/tiff"                 TIFF encoded image data
45   
46   For **X11** platforms, these additional types are supported automatically and
47   resolve to their internal definitions:
48   
49   ::
50   
51     "text/plain;charset=utf-8"   UTF-8 encoded text
52     "UTF8_STRING"                UTF-8 encoded text
53     "COMPOUND_TEXT"              COMPOUND text
54   
55   User defined types can be used, but the data might not be accessible by other
56   applications unless they know what data type to look for.
57   Example: Data placed into the clipboard by
58   ``pygame.scrap.put("my_data_type", byte_data)`` can only be accessed by
59   applications which query the clipboard for the ``"my_data_type"`` data type.
60   
61   For an example of how the scrap module works refer to the examples page
62   (:func:`pygame.examples.scrap_clipboard.main`) or the code directly in GitHub
63   (`pygame/examples/scrap_clipboard.py <https://github.com/pygame/pygame/blob/master/examples/scrap_clipboard.py>`_).
64   
65   .. versionadded:: 1.8
66   
67   .. note::
68      The scrap module is currently only supported for Windows, X11 and Mac OS X.
69      On Mac OS X only text works at the moment - other types may be supported in
70      future releases.

101  .. function:: get
102  
103     | :sl:`Gets the data for the specified type from the clipboard.`
104     | :sg:`get(type) -> bytes | None`
105  
106     Retrieves the data for the specified type from the clipboard. The data is
107     returned as a byte string and might need further processing (such as
108     decoding to Unicode).
109  
110     :param string type: data type to retrieve from the clipboard
111  
112     :returns: data (bytes object) for the given type identifier or ``None`` if
113        no data for the given type is available
114     :rtype: bytes | None
115  
116     ::
117  
118       text = pygame.scrap.get(pygame.SCRAP_TEXT)
119       if text:
120           print("There is text in the clipboard.")
121       else:
122           print("There does not seem to be text in the clipboard.")
123

126  .. function:: get_types
127  
128     | :sl:`Gets a list of the available clipboard types.`
129     | :sg:`get_types() -> list`
130  
131     Gets a list of data type string identifiers for the data currently
132     available on the clipboard. Each identifier can be used in the
133     :func:`pygame.scrap.get()` method to get the clipboard content of the
134     specific type.
135  
136     :returns: list of strings of the available clipboard data types, if there
137        is no data in the clipboard an empty list is returned
138     :rtype: list
139  
140     ::
141  
142       for t in pygame.scrap.get_types():
143           if "text" in t:
144               # There is some content with the word "text" in its type string.
145               print(pygame.scrap.get(t))

149  .. function:: put
150  
151     | :sl:`Places data into the clipboard.`
152     | :sg:`put(type, data) -> None`
153  
154     Places data for a given clipboard type into the clipboard. The data must
155     be a string buffer. The type is a string identifying the type of data to be
156     placed into the clipboard. This can be one of the predefined
157     ``pygame.SCRAP_PBM``, ``pygame.SCRAP_PPM``, ``pygame.SCRAP_BMP`` or
158     ``pygame.SCRAP_TEXT`` values or a user defined string identifier.
159  
160     :param string type: type identifier of the data to be placed into the
161        clipboard
162     :param data: data to be place into the clipboard, a bytes object
163     :type data: bytes
164  
165     :raises pygame.error: if unable to put the data into the clipboard
166  
167     ::
168  
169       with open("example.bmp", "rb") as fp:
170           pygame.scrap.put(pygame.SCRAP_BMP, fp.read())
171       # The image data is now on the clipboard for other applications to access
172       # it.
173       pygame.scrap.put(pygame.SCRAP_TEXT, b"A text to copy")
174       pygame.scrap.put("Plain text", b"Data for user defined type 'Plain text'")
175

178  .. function:: contains
179  
180     | :sl:`Checks whether data for a given type is available in the clipboard.`
181     | :sg:`contains(type) -> bool`
182  
183     Checks whether data for the given type is currently available in the
184     clipboard.
185  
186     :param string type: data type to check availability of
187  
188     :returns: ``True`` if data for the passed type is available in the
189        clipboard, ``False`` otherwise
190     :rtype: bool
191  
192     ::
193  
194       if pygame.scrap.contains(pygame.SCRAP_TEXT):
195           print("There is text in the clipboard.")
196       if pygame.scrap.contains("own_data_type"):
197           print("There is stuff in the clipboard.")

220  .. function:: set_mode
221  
222     | :sl:`Sets the clipboard access mode.`
223     | :sg:`set_mode(mode) -> None`
224  
225     Sets the access mode for the clipboard. This is only of interest for X11
226     environments where clipboard modes ``pygame.SCRAP_SELECTION`` (for mouse
227     selections) and ``pygame.SCRAP_CLIPBOARD`` (for the clipboard) are
228     available. Setting the mode to ``pygame.SCRAP_SELECTION`` in other
229     environments will not change the mode from ``pygame.SCRAP_CLIPBOARD``.
230  
231     :param mode: access mode, supported values are ``pygame.SCRAP_CLIPBOARD``
232        and ``pygame.SCRAP_SELECTION`` (``pygame.SCRAP_SELECTION`` only has an
233        effect when used on X11 platforms)
234  
235     :raises ValueError: if the ``mode`` parameter is not
236        ``pygame.SCRAP_CLIPBOARD`` or ``pygame.SCRAP_SELECTION``
237
