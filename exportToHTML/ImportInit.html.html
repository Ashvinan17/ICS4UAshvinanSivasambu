100  <section id="import-and-initialize">
101  <h2>Import and Initialize<a class="headerlink" href="#import-and-initialize" title="Permalink to this headline">¶</a></h2>
102  <dl class="docinfo field-list simple">
103  <dt class="field-odd">Author</dt>
104  <dd class="field-odd"><p>Pete Shinners</p>
105  </dd>
106  <dt class="field-even">Contact</dt>
107  <dd class="field-even"><p><a class="reference external" href="mailto:pete&#37;&#52;&#48;shinners&#46;org">pete<span>&#64;</span>shinners<span>&#46;</span>org</a></p>
108  </dd>
109  </dl>
110  <p>Getting pygame imported and initialized is a very simple process. It is also
111  flexible enough to give you control over what is happening. Pygame is a
112  collection of different modules in a single python package. Some of the
113  modules are written in C, and some are written in python. Some modules
114  are also optional, and might not always be present.</p>
115  <p>This is just a quick introduction on what is going on when you import pygame.
116  For a clearer explanation definitely see the pygame examples.</p>
117  <section id="import">
118  <h3>Import<a class="headerlink" href="#import" title="Permalink to this headline">¶</a></h3>
119  <p>First we must import the pygame package. Since pygame version 1.4 this
120  has been updated to be much easier. Most games will import all of pygame like this.</p>
121  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="kn">import</span> <span class="nn">pygame</span>
122  <span class="kn">from</span> <span class="nn">pygame.locals</span> <span class="kn">import</span> <span class="o">*</span>
123  </pre></div>
124  </div>
125  <p>The first line here is the only necessary one. It imports all the available pygame
126  modules into the pygame package. The second line is optional, and puts a limited
127  set of constants and functions into the global namespace of your script.</p>
128  <p>An important thing to keep in mind is that several pygame modules are optional.
129  For example, one of these is the font module. When  you &quot;import pygame&quot;, pygame
130  will check to see if the font module is available. If the font module is available
131  it will be imported as &quot;pygame.font&quot;. If the module is not available, &quot;pygame.font&quot;
132  will be set to None. This makes it fairly easy to later on test if the font module is available.</p>
133  </section>
134  <section id="init">
135  <h3>Init<a class="headerlink" href="#init" title="Permalink to this headline">¶</a></h3>
136  <p>Before you can do much with pygame, you will need to initialize it. The most common
137  way to do this is just make one call.</p>
138  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">pygame</span><span class="o">.</span><span class="n">init</span><span class="p">()</span>
139  </pre></div>
140  </div>
141  <p>This will attempt to initialize all the pygame modules for you. Not all pygame modules
142  need to be initialized, but this will automatically initialize the ones that do. You can
143  also easily initialize each pygame module by hand. For example to only initialize the
144  font module you would just call.</p>
145  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">pygame</span><span class="o">.</span><span class="n">font</span><span class="o">.</span><span class="n">init</span><span class="p">()</span>
146  </pre></div>
147  </div>
148  <p>Note that if there is an error when you initialize with &quot;pygame.init()&quot;, it will silently fail.
149  When hand initializing modules like this, any errors will raise an exception. Any
150  modules that must be initialized also have a &quot;get_init()&quot; function, which will return true
151  if the module has been initialized.</p>
152  <p>It is safe to call the init() function for any module more than once.</p>
153  </section>
154  <section id="quit">
155  <h3>Quit<a class="headerlink" href="#quit" title="Permalink to this headline">¶</a></h3>
156  <p>Modules that are initialized also usually have a quit() function that will clean up.
157  There is no need to explicitly call these, as pygame will cleanly quit all the
158  initialized modules when python finishes.</p>
159  </section>
160  </section>
