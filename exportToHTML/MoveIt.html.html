100  <section id="help-how-do-i-move-an-image">
101  <h2>Help! How Do I Move An Image?<a class="headerlink" href="#help-how-do-i-move-an-image" title="Permalink to this headline">¶</a></h2>
102  <dl class="docinfo field-list simple">
103  <dt class="field-odd">Author</dt>
104  <dd class="field-odd"><p>Pete Shinners</p>
105  </dd>
106  <dt class="field-even">Contact</dt>
107  <dd class="field-even"><p><a class="reference external" href="mailto:pete&#37;&#52;&#48;shinners&#46;org">pete<span>&#64;</span>shinners<span>&#46;</span>org</a></p>
108  </dd>
109  </dl>
110  <p>Many people new to programming and graphics have a hard time figuring
111  out  how to make an image move around the screen. Without understanding
112  all  the  concepts, it can be very confusing. You're not the first person
113  to be  stuck  here, I'll do my best to take things step by step. We'll even
114  try to end with methods of keeping your animations efficient.</p>
115  <p>Note that we won't be teaching you to program with python in this article,
116  just introduce you to some of the basics with pygame.</p>
117  <section id="just-pixels-on-the-screen">
118  <h3>Just Pixels On The Screen<a class="headerlink" href="#just-pixels-on-the-screen" title="Permalink to this headline">¶</a></h3>
119  <p>Pygame has a display Surface. This is basically an image that is visible
120  on the screen, and the image is made up of pixels. The main way you change
121  these pixels is by calling the blit() function. This copies the pixels
122  from   one image onto another.</p>
123  <p>This is the first thing to understand. When you blit an image onto the
124  screen, you are simply changing the color of the pixels on the screen.
125  Pixels aren't added or moved, we just change the colors of the pixels already
126  on the screen. These images you blit to the screen are also Surfaces in
127  pygame, but they are in no way connected to the display Surface. When they
128  are blitted  to the screen they are copied into the display, but you still
129  have a unique  copy of the original.</p>
130  <p>With this brief description. Perhaps you can already understand what
131  is needed to &quot;move&quot; an image. We don't actually move anything at all. We
132  simply blit the image in a new position. But before we draw the image in
133  the new position, we'll need to &quot;erase&quot; the old one. Otherwise the image
134  will be visible in two places on the screen. By rapidly erasing the image
135  and redrawing it in a new place, we achieve the &quot;illusion&quot; of movement.</p>
136  <p>Through the rest of this tutorial we will break this process down into
137  simpler steps. Even explaining the best ways to have multiple images moving
138  around the screen. You probably already have questions. Like, how do we
139  &quot;erase&quot; the image before drawing it in a new position? Perhaps you're still
140  totally lost? Well hopefully the rest of this tutorial can straighten things
141  out for you.</p>
142  </section>
143  <section id="let-s-go-back-a-step">
144  <h3>Let's Go Back A Step<a class="headerlink" href="#let-s-go-back-a-step" title="Permalink to this headline">¶</a></h3>
145  <p>Perhaps the concept of pixels and images is still a little foreign to
146  you?  Well good news, for the next few sections we are going to use code that
147  does  everything we want, it just doesn't use pixels. We're going to create
148  a small  python list of 6 numbers, and imagine it represents some fantastic
149  graphics  we could see on the screen. It might actually be surprising how
150  closely this  represents exactly what we'll later be doing with real graphics.</p>
151  <p>So let's begin by creating our screen list and fill it with a beautiful
152  landscape of 1s and 2s.</p>
153  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span> <span class="o">=</span> <span class="p">[</span><span class="mi">1</span><span class="p">,</span> <span class="mi">1</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">1</span><span class="p">]</span>
154  <span class="gp">&gt;&gt;&gt; </span><span class="nb">print</span> <span class="n">screen</span>
155  <span class="go">[1, 1, 2, 2, 2, 1]</span>
156  </pre></div>
157  </div>
158  <p>Now we've created our background. It's not going to be very exciting
159  unless   we also draw a player on the screen. We'll create a mighty hero
160  that looks   like the number 8. Let's stick him near the middle of the map
161  and see what   it looks like.</p>
162  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="p">[</span><span class="mi">3</span><span class="p">]</span> <span class="o">=</span> <span class="mi">8</span>
163  <span class="gp">&gt;&gt;&gt; </span><span class="nb">print</span> <span class="n">screen</span>
164  <span class="go">[1, 1, 2, 8, 2, 1]</span>
165  </pre></div>
166  </div>
167  <p>This might have been as far as you've gotten if you jumped right in doing
168  some graphics programming with pygame. You've got some nice looking stuff
169  on the screen, but it cannot move anywhere. Perhaps now that our screen
170  is  just a list of numbers, it's easier to see how to move him?</p>
171  </section>
172  <section id="making-the-hero-move">
173  <h3>Making The Hero Move<a class="headerlink" href="#making-the-hero-move" title="Permalink to this headline">¶</a></h3>
174  <p>Before we can start moving the character. We need to keep track of some
175  sort of position for him. In the last section when we drew him, we just picked
176  an arbitrary position. Let's do it a little more officially this time.</p>
177  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">playerpos</span> <span class="o">=</span> <span class="mi">3</span>
178  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="p">[</span><span class="n">playerpos</span><span class="p">]</span> <span class="o">=</span> <span class="mi">8</span>
179  <span class="gp">&gt;&gt;&gt; </span><span class="nb">print</span> <span class="n">screen</span>
180  <span class="go">[1, 1, 2, 8, 2, 1]</span>
181  </pre></div>
182  </div>
183  <p>Now it is pretty easy to move him to a new position. We simply change
184  the  value of playerpos, and draw him on the screen again.</p>
185  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">playerpos</span> <span class="o">=</span> <span class="n">playerpos</span> <span class="o">-</span> <span class="mi">1</span>
186  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="p">[</span><span class="n">playerpos</span><span class="p">]</span> <span class="o">=</span> <span class="mi">8</span>
187  <span class="gp">&gt;&gt;&gt; </span><span class="nb">print</span> <span class="n">screen</span>
188  <span class="go">[1, 1, 8, 8, 2, 1]</span>
189  </pre></div>
190  </div>
191  <p>Whoops. Now we can see two heroes. One in the old position, and one
192  in his new position. This is exactly the reason we need to &quot;erase&quot; the hero
193  in his old position before we draw him in the new position. To erase him,
194  we need to change that value in the list back to what it was before the hero
195  was there. That means we need to keep track of the values on the screen before
196  the hero replaced them. There's several way you could do this, but the easiest
197  is usually to keep a separate copy of the screen background. This means
198  we  need to make some changes to our little game.</p>
199  </section>
200  <section id="creating-a-map">
201  <h3>Creating A Map<a class="headerlink" href="#creating-a-map" title="Permalink to this headline">¶</a></h3>
202  <p>What we want to do is create a separate list we will call our background.
203  We will create the background so it looks like our original screen did,
204  with  1s and 2s. Then we will copy each item from the background to the screen.
205  After that we can finally draw our hero back onto the screen.</p>
206  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">background</span> <span class="o">=</span> <span class="p">[</span><span class="mi">1</span><span class="p">,</span> <span class="mi">1</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">1</span><span class="p">]</span>

208  <span class="gp">&gt;&gt;&gt; </span><span class="k">for</span> <span class="n">i</span> <span class="ow">in</span> <span class="nb">range</span><span class="p">(</span><span class="mi">6</span><span class="p">):</span>
209  <span class="gp">... </span>    <span class="n">screen</span><span class="p">[</span><span class="n">i</span><span class="p">]</span> <span class="o">=</span> <span class="n">background</span><span class="p">[</span><span class="n">i</span><span class="p">]</span>
210  <span class="gp">&gt;&gt;&gt; </span><span class="nb">print</span> <span class="n">screen</span>
211  <span class="go">[1, 1, 2, 2, 2, 1]</span>

213  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="p">[</span><span class="n">playerpos</span><span class="p">]</span> <span class="o">=</span> <span class="mi">8</span>
214  <span class="gp">&gt;&gt;&gt; </span><span class="nb">print</span> <span class="n">screen</span>
215  <span class="go">[1, 1, 2, 8, 2, 1]</span>
216  </pre></div>
217  </div>
218  <p>It may seem like a lot of extra work. We're no farther off than we were
219  before the last time we tried to make him move. But this time we have the
220  extra information we need to move him properly.</p>
221  </section>

223  <h3>Making The Hero Move (Take 2)<a class="headerlink" href="#making-the-hero-move-take-2" title="Permalink to this headline">¶</a></h3>
224  <p>This time it will be easy to move the hero around. First we will erase
225  the  hero from his old position. We do this by copying the correct value
226  from the background onto the screen. Then we will draw the character in his
227  new position on the screen</p>
228  <div class="doctest highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="nb">print</span> <span class="n">screen</span>
229  <span class="go">[1, 1, 2, 8, 2, 1]</span>
230  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="p">[</span><span class="n">playerpos</span><span class="p">]</span> <span class="o">=</span> <span class="n">background</span><span class="p">[</span><span class="n">playerpos</span><span class="p">]</span>
231  <span class="gp">&gt;&gt;&gt; </span><span class="n">playerpos</span> <span class="o">=</span> <span class="n">playerpos</span> <span class="o">-</span> <span class="mi">1</span>
232  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="p">[</span><span class="n">playerpos</span><span class="p">]</span> <span class="o">=</span> <span class="mi">8</span>
233  <span class="gp">&gt;&gt;&gt; </span><span class="nb">print</span> <span class="n">screen</span>
234  <span class="go">[1, 1, 8, 2, 2, 1]</span>
235  </pre></div>
236  </div>
237  <p>There it is. The hero has moved one space to the left. We can use this
238  same  code to move him to the left again.</p>
239  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="p">[</span><span class="n">playerpos</span><span class="p">]</span> <span class="o">=</span> <span class="n">background</span><span class="p">[</span><span class="n">playerpos</span><span class="p">]</span>
240  <span class="gp">&gt;&gt;&gt; </span><span class="n">playerpos</span> <span class="o">=</span> <span class="n">playerpos</span> <span class="o">-</span> <span class="mi">1</span>

243  <span class="go">[1, 8, 2, 2, 2, 1]</span>
244  </pre></div>
245  </div>
246  <p>Excellent! This isn't exactly what you'd call smooth animation. But with
247  a couple small changes, we'll make this work directly with graphics on
248  the   screen.</p>
249  </section>
250  <section id="definition-blit">
251  <h3>Definition: &quot;blit&quot;<a class="headerlink" href="#definition-blit" title="Permalink to this headline">¶</a></h3>
252  <p>In the next sections we will transform our program from using lists to
253  using real graphics on the screen. When displaying the graphics we will
254  use the term <strong>blit</strong> frequently. If you are new to doing graphics
255  work, you are probably unfamiliar with this common term.</p>
256  <p>BLIT: Basically, blit means to copy graphics from one image
257  to another. A more formal definition is to copy an array of data
258  to a bitmapped array destination. You can think of blit as just
259  <em>&quot;assigning&quot;</em> pixels. Much like setting values in our screen-list
260  above, blitting assigns the color of pixels in our image.</p>
261  <p>Other graphics libraries will use the word <em>bitblt</em>, or just <em>blt</em>,
262  but they are talking about the same thing. It is basically copying
263  memory from one place to another. Actually, it is a bit more advanced than
264  straight copying of memory, since it needs to handle things like pixel
265  formats, clipping, and scanline pitches. Advanced blitters can also
266  handle things like transparency and other special effects.</p>
267  </section>
268  <section id="going-from-the-list-to-the-screen">
269  <h3>Going From The List To The Screen<a class="headerlink" href="#going-from-the-list-to-the-screen" title="Permalink to this headline">¶</a></h3>
270  <p>To take the code we see in the above to examples and make them work with
271  pygame is very straightforward. We'll pretend we have loaded some pretty
272  graphics and named them &quot;terrain1&quot;, &quot;terrain2&quot;, and &quot;hero&quot;. Where before
273  we assigned numbers to a list, we now blit graphics to the screen. Another
274  big change, instead of using positions as a single index (0 through 5), we
275  now need a two dimensional coordinate. We'll pretend each of the graphics
276  in our game is 10 pixels wide.</p>
277  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">background</span> <span class="o">=</span> <span class="p">[</span><span class="n">terrain1</span><span class="p">,</span> <span class="n">terrain1</span><span class="p">,</span> <span class="n">terrain2</span><span class="p">,</span> <span class="n">terrain2</span><span class="p">,</span> <span class="n">terrain2</span><span class="p">,</span> <span class="n">terrain1</span><span class="p">]</span>
278  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span> <span class="o">=</span> <span class="n">create_graphics_screen</span><span class="p">()</span>
279  <span class="gp">&gt;&gt;&gt; </span><span class="k">for</span> <span class="n">i</span> <span class="ow">in</span> <span class="nb">range</span><span class="p">(</span><span class="mi">6</span><span class="p">):</span>
280  <span class="gp">... </span>    <span class="n">screen</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="n">background</span><span class="p">[</span><span class="n">i</span><span class="p">],</span> <span class="p">(</span><span class="n">i</span><span class="o">*</span><span class="mi">10</span><span class="p">,</span> <span class="mi">0</span><span class="p">))</span>
281  <span class="gp">&gt;&gt;&gt; </span><span class="n">playerpos</span> <span class="o">=</span> <span class="mi">3</span>
282  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="n">playerimage</span><span class="p">,</span> <span class="p">(</span><span class="n">playerpos</span><span class="o">*</span><span class="mi">10</span><span class="p">,</span> <span class="mi">0</span><span class="p">))</span>
283  </pre></div>
284  </div>
285  <p>Hmm, that code should seem very familiar, and hopefully more importantly;
286  the code above should make a little sense. Hopefully my illustration of setting
287  simple values in a list shows the similarity of setting pixels on the screen
288  (with blit). The only part that's really extra work is converting the player position
289  into coordinates on the screen. For now we just use a crude <code class="code docutils literal notranslate"><span class="pre">(playerpos*10,</span> <span class="pre">0)</span></code> ,
290  but we can certainly do better than that. Now let's move the player
291  image over a space. This code should have no surprises.</p>
292  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="n">background</span><span class="p">[</span><span class="n">playerpos</span><span class="p">],</span> <span class="p">(</span><span class="n">playerpos</span><span class="o">*</span><span class="mi">10</span><span class="p">,</span> <span class="mi">0</span><span class="p">))</span>
293  <span class="gp">&gt;&gt;&gt; </span><span class="n">playerpos</span> <span class="o">=</span> <span class="n">playerpos</span> <span class="o">-</span> <span class="mi">1</span>
294  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="n">playerimage</span><span class="p">,</span> <span class="p">(</span><span class="n">playerpos</span><span class="o">*</span><span class="mi">10</span><span class="p">,</span> <span class="mi">0</span><span class="p">))</span>
295  </pre></div>
296  </div>
297  <p>There you have it. With this code we've shown how to display a simple background
298  with a hero's image on it. Then we've properly moved that hero one space
299  to the left. So where do we go from here? Well for one the code is still
300  a little awkward. First thing we'll want to do is find a cleaner way to represent
301  the background and player position. Then perhaps a bit of smoother, real
302  animation.</p>
303  </section>
304  <section id="screen-coordinates">
305  <h3>Screen Coordinates<a class="headerlink" href="#screen-coordinates" title="Permalink to this headline">¶</a></h3>
306  <p>To position an object on the screen, we need to tell the blit() function
307  where to put the image. In pygame we always pass positions as an (X,Y) coordinate.
308  This represents the number of pixels to the right, and the number of pixels
309  down to place the image. The top-left corner of a Surface is coordinate (0,
310  0). Moving to the right a little would be (10, 0), and then moving down just
311  as much would be (10, 10). When blitting, the position argument represents
312  where the topleft corner of the source should be placed on the destination.</p>
313  <p>Pygame comes with a convenient container for these coordinates, it is a
314  Rect. The Rect basically represents a rectangular area in these coordinates.
315  It has topleft corner and a size. The Rect comes with a lot of convenient
316  methods which help you move and position them. In our next examples we will
317  represent the positions of our objects with the Rects.</p>
318  <p>Also know that many functions in pygame expect Rect arguments. All of these
319  functions can also accept a simple tuple of 4 elements (left, top, width,
320  height). You aren't always required to use these Rect objects, but you will
321  mainly want to. Also, the blit() function can accept a Rect as its position
322  argument, it simply uses the topleft corner of the Rect as the real position.</p>
323  </section>
324  <section id="changing-the-background">
325  <h3>Changing The Background<a class="headerlink" href="#changing-the-background" title="Permalink to this headline">¶</a></h3>
326  <p>In all our previous sections, we've been storing the background as a list
327  of different types of ground. That is a good way to create a tile-based game,
328  but we want smooth scrolling. To make that a little easier, we're going to
329  change the background into a single image that covers the whole screen. This
330  way, when we want to &quot;erase&quot; our objects (before redrawing them) we only need
331  to blit the section of the erased background onto the screen.</p>
332  <p>By passing an optional third Rect argument to blit, we tell blit to only
333  use that subsection of the source image. You'll see that in use below as we
334  erase the player image.</p>
335  <p>Also note, now when we finish drawing to the screen, we call pygame.display.update()
336  which will show everything we've drawn onto the screen.</p>
337  </section>
338  <section id="smooth-movement">
339  <h3>Smooth Movement<a class="headerlink" href="#smooth-movement" title="Permalink to this headline">¶</a></h3>
340  <p>To make something appear to move smoothly, we only want to move it a couple
341  pixels at a time. Here is the code to make an object move smoothly across
342  the screen. Based on what we already now know, this should look pretty simple.</p>
343  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span> <span class="o">=</span> <span class="n">create_screen</span><span class="p">()</span>
344  <span class="gp">&gt;&gt;&gt; </span><span class="n">player</span> <span class="o">=</span> <span class="n">load_player_image</span><span class="p">()</span>
345  <span class="gp">&gt;&gt;&gt; </span><span class="n">background</span> <span class="o">=</span> <span class="n">load_background_image</span><span class="p">()</span>
346  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="n">background</span><span class="p">,</span> <span class="p">(</span><span class="mi">0</span><span class="p">,</span> <span class="mi">0</span><span class="p">))</span>        <span class="c1">#draw the background</span>
347  <span class="gp">&gt;&gt;&gt; </span><span class="n">position</span> <span class="o">=</span> <span class="n">player</span><span class="o">.</span><span class="n">get_rect</span><span class="p">()</span>
348  <span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="n">player</span><span class="p">,</span> <span class="n">position</span><span class="p">)</span>          <span class="c1">#draw the player</span>
349  <span class="gp">&gt;&gt;&gt; </span><span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">update</span><span class="p">()</span>                <span class="c1">#and show it all</span>
350  <span class="gp">&gt;&gt;&gt; </span><span class="k">for</span> <span class="n">x</span> <span class="ow">in</span> <span class="nb">range</span><span class="p">(</span><span class="mi">100</span><span class="p">):</span>                   <span class="c1">#animate 100 frames</span>
351  <span class="gp">... </span>    <span class="n">screen</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="n">background</span><span class="p">,</span> <span class="n">position</span><span class="p">,</span> <span class="n">position</span><span class="p">)</span> <span class="c1">#erase</span>
352  <span class="gp">... </span>    <span class="n">position</span> <span class="o">=</span> <span class="n">position</span><span class="o">.</span><span class="n">move</span><span class="p">(</span><span class="mi">2</span><span class="p">,</span> <span class="mi">0</span><span class="p">)</span>     <span class="c1">#move player</span>
353  <span class="gp">... </span>    <span class="n">screen</span><span class="o">.</span><span class="n">blit</span><span class="p">(</span><span class="n">player</span><span class="p">,</span> <span class="n">position</span><span class="p">)</span>      <span class="c1">#draw new player</span>
354  <span class="gp">... </span>    <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">update</span><span class="p">()</span>            <span class="c1">#and show it all</span>
355  <span class="gp">... </span>    <span class="n">pygame</span><span class="o">.</span><span class="n">time</span><span class="o">.</span><span class="n">delay</span><span class="p">(</span><span class="mi">100</span><span class="p">)</span>             <span class="c1">#stop the program for 1/10 second</span>
356  </pre></div>
357  </div>
358  <p>There you have it. This is all the code that is needed to smoothly animate
359  an object across the screen. We can even use a pretty background character.
360  Another benefit of doing the background this way, the image for the player
361  can have transparency or cutout sections and it will still draw correctly
362  over the background (a free bonus).</p>
363  <p>We also throw in a call to pygame.time.delay() at the end of our loop above.
364  This slows down our program a little, otherwise it might run so fast you might
365  not see it.</p>
366  </section>
367  <section id="so-what-next">
368  <h3>So, What Next?<a class="headerlink" href="#so-what-next" title="Permalink to this headline">¶</a></h3>
369  <p>Well there we have it. Hopefully this article has done everything it promised
370  to do. But, at this point the code really isn't ready for the next best-selling
371  game. How do we easily have multiple moving objects? What exactly are those
372  mysterious functions like load_player_image()? We also need a way to get simple
373  user input, and loop for more than 100 frames. We'll take the example we
374  have here, and turn it into an object oriented creation that would make momma
375  proud.</p>
376  </section>
377  <section id="first-the-mystery-functions">
378  <h3>First, The Mystery Functions<a class="headerlink" href="#first-the-mystery-functions" title="Permalink to this headline">¶</a></h3>
379  <p>Full information on these types of functions can be found in other tutorials
380  and reference. The pygame.image module has a load() function which will do
381  what we want. The lines to load the images should become this.</p>
382  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">player</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">image</span><span class="o">.</span><span class="n">load</span><span class="p">(</span><span class="s1">&#39;player.bmp&#39;</span><span class="p">)</span><span class="o">.</span><span class="n">convert</span><span class="p">()</span>
383  <span class="gp">&gt;&gt;&gt; </span><span class="n">background</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">image</span><span class="o">.</span><span class="n">load</span><span class="p">(</span><span class="s1">&#39;liquid.bmp&#39;</span><span class="p">)</span><span class="o">.</span><span class="n">convert</span><span class="p">()</span>
384  </pre></div>
385  </div>
386  <p>We can see that's pretty simple, the load function just takes a filename
387  and returns a new Surface with the loaded image. After loading we make a call
388  to the Surface method, convert(). Convert returns us a new Surface of the
389  image, but now converted to the same pixel format as our display. Since the
390  images will be the same format at the screen, they will blit very quickly.
391  If we did not convert, the blit() function is slower, since it has to convert
392  from one type of pixel to another as it goes.</p>
393  <p>You may also have noticed that both the load() and convert() return new
394  Surfaces. This means we're really creating two Surfaces on each of these
395  lines. In other programming languages, this results in a memory leak (not
396  a good thing). Fortunately Python is smart enough to handle this, and pygame
397  will properly clean up the Surface we end up not using.</p>
398  <p>The other mystery function we saw in the above example was create_screen().
399  In pygame it is simple to create a new window for graphics. The code to create
400  a 640x480 surface is below. By passing no other arguments, pygame will just
401  pick the best color depth and pixel format for us.</p>
402  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="n">screen</span> <span class="o">=</span> <span class="n">pygame</span><span class="o">.</span><span class="n">display</span><span class="o">.</span><span class="n">set_mode</span><span class="p">((</span><span class="mi">640</span><span class="p">,</span> <span class="mi">480</span><span class="p">))</span>
403  </pre></div>
404  </div>
405  </section>
406  <section id="handling-some-input">
407  <h3>Handling Some Input<a class="headerlink" href="#handling-some-input" title="Permalink to this headline">¶</a></h3>
408  <p>We desperately need to change the main loop to look for any user input, (like
409  when the user closes the window). We need to add &quot;event handling&quot; to our
410  program. All graphical programs use this Event Based design. The program
411  gets events like &quot;keyboard pressed&quot; or &quot;mouse moved&quot; from the computer. Then
412  the program responds to the different events. Here's what the code should
413  look like. Instead of looping for 100 frames, we'll keep looping until the
414  user asks us to stop.</p>
415  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="k">while</span> <span class="mi">1</span><span class="p">:</span>
416  <span class="gp">... </span>    <span class="k">for</span> <span class="n">event</span> <span class="ow">in</span> <span class="n">pygame</span><span class="o">.</span><span class="n">event</span><span class="o">.</span><span class="n">get</span><span class="p">():</span>
417  <span class="gp">... </span>        <span class="k">if</span> <span class="n">event</span><span class="o">.</span><span class="n">type</span> <span class="ow">in</span> <span class="p">(</span><span class="n">QUIT</span><span class="p">,</span> <span class="n">KEYDOWN</span><span class="p">):</span>
418  <span class="gp">... </span>            <span class="n">sys</span><span class="o">.</span><span class="n">exit</span><span class="p">()</span>
419  <span class="gp">... </span>    <span class="n">move_and_draw_all_game_objects</span><span class="p">()</span>
420  </pre></div>
421  </div>
422  <p>What this code simply does is, first loop forever, then check if there are
423  any events from the user. We exit the program if the user presses the keyboard
424  or the close button on the window. After we've checked all the events we
425  move and draw our game objects. (We'll also erase them before they move,
426  too)</p>
427  </section>
428  <section id="moving-multiple-images">
429  <h3>Moving Multiple Images<a class="headerlink" href="#moving-multiple-images" title="Permalink to this headline">¶</a></h3>
430  <p>Here's the part where we're really going to change things around. Let's
431  say we want 10 different images moving around on the screen. A good way to
432  handle this is to use python's classes. We'll create a class that represents
433  our game object. This object will have a function to move itself, and then
434  we can create as many as we like. The functions to draw and move the object
435  need to work in a way where they only move one frame (or one step) at a time.
436  Here's the python code to create our class.</p>
437  <div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="gp">&gt;&gt;&gt; </span><span class="k">class</span> <span class="nc">GameObject</span><span class="p">:</span>
438  <span class="gp">... </span>    <span class="k">def</span> <span class="fm">__init__</span><span class="p">(</span><span class="bp">self</span><span class="p">,</span> <span class="n">image</span><span class="p">,</span> <span class="n">height</span><span class="p">,</span> <span class="n">speed</span><span class="p">):</span>
439  <span class="gp">... </span>        <span class="bp">self</span><span class="o">.</span><span class="n">speed</span> <span class="o">=</span> <span class="n">speed</span>
440  <span class="gp">... </span>        <span class="bp">self</span><span class="o">.</span><span class="n">image</span> <span class="o">=</span> <span class="n">image</span>
441  <span class="gp">... </span>        <span class="bp">self</span><span class="o">.</span><span class="n">pos</span> <span class="o">=</span> <span class="n">image</span><span class="o">.</span><span class="n">get_rect</span><span class="p">()</span><span class="o">.</span><span class="n">move</span><span class="p">(</span><span class="mi">0</span><span class="p">,</span> <span class="n">height</span><span class="p">)</span>
442  <span class="gp">... </span>    <span class="k">def</span> <span class="nf">move</span><span class="p">(</span><span class="bp">self</span><span class="p">):</span>
443  <span class="gp">... </span>        <span class="bp">self</span><span class="o">.</span><span class="n">pos</span> <span class="o">=</span> <span class="bp">self</span><span class="o">.</span><span class="n">pos</span><span class="o">.</span><span class="n">move</span><span class="p">(</span><span class="mi">0</span><span class="p">,</span> <span class="bp">self</span><span class="o">.</span><span class="n">speed</span><span class="p">)</span>
444  <span class="gp">... </span>        <span class="k">if</span> <span class="bp">self</span><span class="o">.</span><span class="n">pos</span><span class="o">.</span><span class="n">right</span> <span class="o">&gt;</span> <span class="mi">600</span><span class="p">:</span>
445  <span class="gp">... </span>            <span class="bp">self</span><span class="o">.</span><span class="n">pos</span><span class="o">.</span><span class="n">left</span> <span class="o">=</span> <span class="mi">0</span>
446  </pre></div>
447  </div>
448  <p>So we have two functions in our class. The init function constructs our object.
449  It positions the object and sets its speed. The move method moves the object
450  one step. If it's gone too far, it moves the object back to the left.</p>
451  </section>
