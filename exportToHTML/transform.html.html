180  <p>A Surface transform is an operation that moves or resizes the pixels. All these
181  functions take a Surface to operate on and return a new Surface with the
182  results.</p>
183  <p>Some of the transforms are considered destructive. These means every time they
184  are performed they lose pixel data. Common examples of this are resizing and
185  rotating. For this reason, it is better to re-transform the original surface
186  than to keep transforming an image multiple times. (For example, suppose you
187  are animating a bouncing spring which expands and contracts. If you applied the
188  size changes incrementally to the previous images, you would lose detail.
189  Instead, always begin with the original image and scale to the desired size.)</p>
190  <div class="versionchanged">
191  <p><span class="versionmodified changed">Changed in pygame 2.0.2: </span>transform functions now support keyword arguments.</p>
192  </div>

207  <dt class="sig sig-object py title" id="pygame.transform.scale">
208  <span class="sig-prename descclassname"><span class="pre">pygame.transform.</span></span><span class="sig-name descname"><span class="pre">scale</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.transform.scale" title="Permalink to this definition">¶</a></dt>

210  <div class="line"><span class="summaryline">resize to new resolution</span></div>
211  <div class="line"><span class="signature">scale(surface, size, dest_surface=None) -&gt; Surface</span></div>
212  </div>
213  <p>Resizes the Surface to a new size, given as (width, height).
214  This is a fast scale operation that does not sample the results.</p>
215  <p>An optional destination surface can be used, rather than have it create a
216  new one. This is quicker if you want to repeatedly scale something. However
217  the destination must be the same size as the size (width, height) passed in. Also
218  the destination surface must be the same format.</p>

223  <span class="sig-prename descclassname"><span class="pre">pygame.transform.</span></span><span class="sig-name descname"><span class="pre">rotate</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.transform.rotate" title="Permalink to this definition">¶</a></dt>

226  <div class="line"><span class="signature">rotate(surface, angle) -&gt; Surface</span></div>

234  colorkey or the topleft pixel value.</p>

252  <dt class="sig sig-object py title" id="pygame.transform.scale2x">
253  <span class="sig-prename descclassname"><span class="pre">pygame.transform.</span></span><span class="sig-name descname"><span class="pre">scale2x</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.transform.scale2x" title="Permalink to this definition">¶</a></dt>
254  <dd><div class="line-block">
255  <div class="line"><span class="summaryline">specialized image doubler</span></div>
256  <div class="line"><span class="signature">scale2x(surface, dest_surface=None) -&gt; Surface</span></div>
257  </div>
258  <p>This will return a new image that is double the size of the original. It
259  uses the AdvanceMAME Scale2X algorithm which does a 'jaggie-less' scale of
260  bitmap graphics.</p>
261  <p>This really only has an effect on simple images with solid colors. On
262  photographic and antialiased images it will look like a regular unfiltered
263  scale.</p>
264  <p>An optional destination surface can be used, rather than have it create a
265  new one. This is quicker if you want to repeatedly scale something. However
266  the destination must be twice the size of the source surface passed in. Also
267  the destination surface must be the same format.</p>
268  </dd></dl>

271  <dt class="sig sig-object py title" id="pygame.transform.smoothscale">
272  <span class="sig-prename descclassname"><span class="pre">pygame.transform.</span></span><span class="sig-name descname"><span class="pre">smoothscale</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.transform.smoothscale" title="Permalink to this definition">¶</a></dt>
273  <dd><div class="line-block">
274  <div class="line"><span class="summaryline">scale a surface to an arbitrary size smoothly</span></div>
275  <div class="line"><span class="signature">smoothscale(surface, size, dest_surface=None) -&gt; Surface</span></div>
276  </div>
277  <p>Uses one of two different algorithms for scaling each dimension of the input
278  surface as required. For shrinkage, the output pixels are area averages of
279  the colors they cover. For expansion, a bilinear filter is used. For the
280  x86-64 and i686 architectures, optimized <code class="docutils literal notranslate"><span class="pre">MMX</span></code> routines are included and
281  will run much faster than other machine types. The size is a 2 number
282  sequence for (width, height). This function only works for 24-bit or 32-bit
283  surfaces. An exception will be thrown if the input surface bit depth is less
284  than 24.</p>
285  <div class="versionadded">
286  <p><span class="versionmodified added">New in pygame 1.8.</span></p>
287  </div>
288  </dd></dl>

303  <dl class="py function definition">
304  <dt class="sig sig-object py title" id="pygame.transform.set_smoothscale_backend">
305  <span class="sig-prename descclassname"><span class="pre">pygame.transform.</span></span><span class="sig-name descname"><span class="pre">set_smoothscale_backend</span></span><span class="sig-paren">(</span><span class="sig-paren">)</span><a class="headerlink" href="#pygame.transform.set_smoothscale_backend" title="Permalink to this definition">¶</a></dt>
306  <dd><div class="line-block">
307  <div class="line"><span class="summaryline">set smoothscale filter version to one of: 'GENERIC', 'MMX', or 'SSE'</span></div>
308  <div class="line"><span class="signature">set_smoothscale_backend(backend) -&gt; None</span></div>
309  </div>
310  <p>Sets smoothscale acceleration. Takes a string argument. A value of 'GENERIC'
311  turns off acceleration. 'MMX' uses <code class="docutils literal notranslate"><span class="pre">MMX</span></code> instructions only. 'SSE' allows

314  <p>This function is provided for pygame testing and debugging. If smoothscale
315  causes an invalid instruction error then it is a pygame/SDL bug that should
316  be reported. Use this function as a temporary fix only.</p>
317  </dd></dl>

383  <dd><div class="line-block">
384  <div class="line"><span class="summaryline">finds which, and how many pixels in a surface are within a threshold of a 'search_color' or a 'search_surf'.</span></div>
385  <div class="line"><span class="signature">threshold(dest_surface, surface, search_color, threshold=(0,0,0,0), set_color=(0,0,0,0), set_behavior=1, search_surf=None, inverse_set=False) -&gt; num_threshold_pixels</span></div>
386  </div>
387  <p>This versatile function can be used for find colors in a 'surf' close to a 'search_color'
388  or close to colors in a separate 'search_surf'.</p>
389  <p>It can also be used to transfer pixels into a 'dest_surf' that match or don't match.</p>
390  <p>By default it sets pixels in the 'dest_surf' where all of the pixels NOT within the
391  threshold are changed to set_color. If inverse_set is optionally set to True,
392  the pixels that ARE within the threshold are changed to set_color.</p>
393  <p>If the optional 'search_surf' surface is given, it is used to threshold against
394  rather than the specified 'set_color'. That is, it will find each pixel in the
395  'surf' that is within the 'threshold' of the pixel at the same coordinates
396  of the 'search_surf'.</p>
397  <dl class="field-list simple">
398  <dt class="field-odd">Parameters</dt>
399  <dd class="field-odd"><ul class="simple">
400  <li><p><strong>dest_surf</strong> (<a class="tooltip reference internal" href="surface.html#pygame.Surface" title=""><em>pygame.Surface</em><span class="tooltip-content">pygame object for representing images</span></a><em> or </em><em>None</em>) -- Surface we are changing. See 'set_behavior'.
401  Should be None if counting (set_behavior is 0).</p></li>
402  <li><p><strong>surf</strong> (<a class="tooltip reference internal" href="surface.html#pygame.Surface" title=""><em>pygame.Surface</em><span class="tooltip-content">pygame object for representing images</span></a>) -- Surface we are looking at.</p></li>
403  <li><p><strong>search_color</strong> (<a class="tooltip reference internal" href="color.html#pygame.Color" title=""><em>pygame.Color</em><span class="tooltip-content">pygame object for color representations</span></a>) -- Color we are searching for.</p></li>
404  <li><p><strong>threshold</strong> (<a class="tooltip reference internal" href="color.html#pygame.Color" title=""><em>pygame.Color</em><span class="tooltip-content">pygame object for color representations</span></a>) -- Within this distance from search_color (or search_surf).
405  You can use a threshold of (r,g,b,a) where the r,g,b can have different
406  thresholds. So you could use an r threshold of 40 and a blue threshold of 2
407  if you like.</p></li>
408  <li><p><strong>set_color</strong> (<a class="tooltip reference internal" href="color.html#pygame.Color" title=""><em>pygame.Color</em><span class="tooltip-content">pygame object for color representations</span></a><em> or </em><em>None</em>) -- Color we set in dest_surf.</p></li>
409  <li><p><strong>set_behavior</strong> (<em>int</em>) -- <ul>
410  <li><p>set_behavior=1 (default). Pixels in dest_surface will be changed to 'set_color'.</p></li>
411  <li><p>set_behavior=0 we do not change 'dest_surf', just count. Make dest_surf=None.</p></li>
412  <li><p>set_behavior=2 pixels set in 'dest_surf' will be from 'surf'.</p></li>
413  </ul>
414  </p></li>
415  <li><p><strong>search_surf</strong> (<a class="tooltip reference internal" href="surface.html#pygame.Surface" title=""><em>pygame.Surface</em><span class="tooltip-content">pygame object for representing images</span></a><em> or </em><em>None</em>) -- <ul>
416  <li><p>search_surf=None (default). Search against 'search_color' instead.</p></li>
417  <li><p>search_surf=Surface. Look at the color in 'search_surf' rather than using 'search_color'.</p></li>
418  </ul>
419  </p></li>
420  <li><p><strong>inverse_set</strong> (<em>bool</em>) -- <ul>
421  <li><p>False, default. Pixels outside of threshold are changed.</p></li>
422  <li><p>True, Pixels within threshold are changed.</p></li>
423  </ul>
424  </p></li>
425  </ul>
426  </dd>
427  <dt class="field-even">Return type</dt>
428  <dd class="field-even"><p>int</p>
429  </dd>
430  <dt class="field-odd">Returns</dt>
431  <dd class="field-odd"><p>The number of pixels that are within the 'threshold' in 'surf'
432  compared to either 'search_color' or <cite>search_surf</cite>.</p>
